## 指摘ごとの対応

### #1 must-fix: pvesh 収集失敗が `[]` に握りつぶされ false OK になる → 修正

妥当な指摘として採用。監視ツールの目的（放置スナップショットの早期検出）に対し、
収集不能を「stale なし / OK」と誤判定する false OK は致命的なため修正した。
core.md §7 の責務分離は維持（shell は失敗を観測値として返すだけ、判定は Ansible）。

- `roles/proxmox_snapshot_check/files/proxmox-snapshot-collect.sh`
  - `pvesh_json()` を `(data, error)` を返す形に変更。失敗（non-zero / 空出力 /
    JSON parse error / timeout / exec 失敗）時は `data=None` と短い error 文字列を返す。
  - `/cluster/resources` が取れない場合は `collection_ok: false` とし、`errors` に記録。
  - 個別ゲストの snapshot 取得失敗も `errors` に `scope / node / type / vmid / error` で記録し、
    そのゲストはスキップ（取得できた分は通常どおり評価）。
  - 出力 JSON に `collection_ok`（bool）と `errors`（list）を追加。判定はしない。
- `roles/proxmox_snapshot_check/tasks/main.yml`
  - `_snap_collection_failed = (not collection_ok) or (errors | length > 0)` を算出。
  - レポートに `collection_ok` / `errors` を追加。
  - `result.status` を「stale あり **または** 収集失敗」で WARNING に。
  - Slack 通知条件を `result.status == 'WARNING'` に変更（収集失敗も #alerts に通知）。
    通知本文に collection の問題（scope: error）も列挙。**fail はしない**（warn-only 維持）。
  - サマリに `collection_ok` / `errors` 件数を追加。

### #2 suggestion: 「7日以上」の境界が `<`（より古い）→ 修正

採用。要求仕様「7日（604800秒）以上前」に正確に合わせ、`selectattr('snaptime', 'lt', ...)`
を `'le'` に変更（作成からちょうど閾値秒のスナップも検出対象に含める）。

- `roles/proxmox_snapshot_check/tasks/main.yml`: stale 抽出を `le` に変更。コメントで意図を明記。

3. 実機 OK パス（Slack ミュート）:
   `ansible-playbook playbooks/proxmox_snapshot_check.yml --skip-tags notify`

4. 収集失敗 → WARNING 判定の分岐ロジック（localhost）:
   `ansible localhost -m debug -a 'msg=status={{ "WARNING" if (stale|length>0 or ((not data.collection_ok) or (data.errors|length>0))) else "OK" }}'`
   （collection_ok=false, errors=1, stale=0 を投入）→ `status=WARNING` を確認。
