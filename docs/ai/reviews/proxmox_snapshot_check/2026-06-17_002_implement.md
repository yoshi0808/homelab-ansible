# 実装報告: proxmox_snapshot_check（スナップショット放置チェック）

対象: `docs/ai/reviews/proxmox_snapshot_check/2026-06-17_001_requirement.md`

## 実装概要

pve1 / pve2 上の全 VM / CT のスナップショットを収集し、閾値（既定 7 日 = 604800 秒）を
超えて放置されているものを検出して Slack #alerts に warning 通知する read-only playbook /
role を新規実装した。

shell（収集スクリプト）は **収集と JSON 整形のみ**を行い、7 日超えの判定・通知制御は
すべて Ansible tasks 側に置いた（core.md §7）。該当なしの場合は通知せずサイレント終了する。
削除・変更操作は一切行わない。critical 段階・fail はなく、警告のみ（要求仕様どおり）。

## 変更内容の要約

### roles/proxmox_snapshot_check/files/proxmox-snapshot-collect.sh
- `hostname` で自ノード名を取得し、`python3` heredoc で収集（proxmox_healthcheck.sh と同方式）。
- `pvesh get /cluster/resources --output-format json` で全リソースを取得し、`type` が
  `qemu` / `lxc` かつ `node` が**自ノード**のゲストのみを対象にする。
- 各ゲストの `pvesh get /nodes/<node>/<type>/<vmid>/snapshot --output-format json` を読み、
  `name == "current"` の擬似エントリ（実行中状態の参照。スナップショットではない）を除外。
- 出力 JSON: `{collected_at, node, snapshots:[{vmid, guest_name, type, snapname, snaptime, description}]}`。
- **判定（7日超え）はしない**。`snaptime`（epoch）をそのまま観測値として返すのみ。

### roles/proxmox_snapshot_check/tasks/main.yml
- スクリプトを `/usr/local/sbin/` に copy（0750, root）→ `command` 実行（`changed_when: false`）。
- 空出力なら fail（スクリプト異常の検知。スナップ 0 件でも JSON は出るため通常は発火しない）。
- `from_json` で解析 → `ansible_facts.date_time.epoch` から閾値 epoch を算出。
- stale 抽出: `snaptime > 0` かつ `snaptime < (now - 閾値)` を `selectattr` で選別。
- 各 stale に `age_days`（経過日数）と `snaptime_iso`（作成日時の ISO 表記）を付与。
- レポートを組み立て、`reports_base_dir/proxmox-snapshot/` に保存（localhost、become:false）。
- `common_slack/tasks/notify.yml` を include_tasks（#alerts / warning）。`stale_count > 0` のときのみ。
- 通知本文に該当ゲスト名・VMID・スナップ名・作成日時・経過日数を列挙（要求仕様の通知内容）。

### defaults / playbook
- `proxmox_snapshot_check_threshold_seconds: 604800`（7 日）、`..._report_dir`。
- playbook は `proxmox` グループ対象、role を1つ呼ぶだけ。

## 実装上の判断（要レビュー）

1. **per-host ローカル絞り込み**: 要求仕様の「pvesh get /cluster/resources でクラスタ全
   VM/CT を列挙」を、各ノードで実行しつつ**自ノードのゲストのみ**に絞る形で実装した。
   playbook は proxmox グループ（pve1/pve2）の両方で走るため、絞らないと両ホストが全
   ゲストを処理し**通知が二重化**する。各ゲストはホームノードで一度だけ評価される。
   proxmox_healthcheck の per-host モデルとも整合する。

2. **要求仕様外の追加（2点・透明化のため明記）**:
   - Slack タスクに `tags: notify` を付与。dev 検証時の通知ミュート（`--skip-tags notify`）と
     運用時の一時ミュートに使える。既定動作は変えない。
   - `run_once` の Semaphore 向けサマリ（`SUMMARY proxmox_snapshot_check | ...`）を debug 出力。
     proxmox_healthcheck と同じ慣習で、Semaphore ログでの確認を容易にする。read-only。
   - いずれも採否はトリアージ対象。不要なら削除する。

## 実行した確認コマンドと結果

1. 収集スクリプト単体（read-only・通知なし）:
   `ansible proxmox -b -m script -a roles/proxmox_snapshot_check/files/proxmox-snapshot-collect.sh`
   → pve1: `snapshots: []` / pve2: VM 201 `ansy` の `before-install`（snaptime 1781771154）。

2. フル playbook（実閾値 7 日）:
   `ansible-playbook playbooks/proxmox_snapshot_check.yml`
   → `Result=OK | pve1=OK(stale=0/0) | pve2=OK(stale=0/1) | Threshold=7d`。
     該当なしのため Slack はスキップ（silent）。失敗 0。

3. WARNING パス検証（閾値 1 秒・Slack ミュート）:
   `ansible-playbook playbooks/proxmox_snapshot_check.yml -e proxmox_snapshot_check_threshold_seconds=1 --skip-tags notify`
   → `pve2=WARNING(stale=1/1)`。レポートの `stale_snapshots` に `age_days` /
     `snaptime_iso=2026-06-18T17:25:54+0900` 等が正しく付与されることを確認。
