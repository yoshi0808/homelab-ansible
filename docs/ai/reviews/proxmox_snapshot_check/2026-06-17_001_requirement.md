# 要求仕様: proxmox_snapshot_check（スナップショット放置チェック）

## 目的
pve1 / pve2 上の全 VM / CT に対して、7日以上前に作成されたスナップショットが
放置されていないかを日次で確認する。
パッチ前・作業前に取ったスナップショットの消し忘れを早期に検出する。

## 対象
- 対象ノード: pve1 / pve2（`proxmox` グループ）
- 実行元: quory（本番・日次 systemd timer）/ ansy（開発・CLI）
- 安全度: safe（read-only。スナップショットの削除は行わない）

## 作成・更新ファイル
- playbooks/proxmox_snapshot_check.yml（新規・read-only）
- roles/proxmox_snapshot_check/（新規）
  - tasks/main.yml
  - defaults/main.yml

## 実施内容

```text
1. pvesh get /cluster/resources でクラスタ全 VM / CT を列挙する
2. 各 VM / CT のスナップショット一覧を取得する
   （pvesh get /nodes/<node>/qemu/<vmid>/snapshot または lxc/<vmid>/snapshot）
3. 各スナップショットの作成日時と現在時刻を比較する
4. 7日（604800秒）以上前のスナップショットが1つでもあれば #alerts に通知する
5. 該当なし → 通知なし（サイレント）
```

## 判定ルール
- 閾値: **7日（604800秒）**
- 「current」スナップショット（Proxmox が自動生成する実行中状態の参照）は除外する
- 閾値超えが1件でもあれば通知する（単純判断）
- 通知内容: 該当 VM / CT 名、VMID、スナップショット名、作成日時、経過日数

## shell / Ansible tasks 責務分離（core.md §7）
- スナップショット一覧の取得と日時の収集は command/shell で行う
- 7日超えの判定と fail / 通知制御は Ansible tasks 側に置く
- shell に判定ロジックを持たせない

## 通知
`roles/common_slack/tasks/notify.yml` を include_tasks で呼ぶ（best-effort）。

| 状況 | チャンネル | status |
|---|---|---|
| 7日超えスナップショットあり | #alerts | warning |
| 該当なし | 通知なし | — |

## 制約
- read-only。スナップショットの削除・変更は行わない
- IP リテラルをファイルに書かない（core.md §3）
- 秘密情報を扱わない

## 初回実装スコープ
- 全 VM / CT のスナップショット取得
- 7日超え判定
- #alerts 通知（該当VMの一覧含む）

## 初回除外スコープ
- hacritical VM の所在チェック
- prefer* タグとの所在一致チェック
- レプリケーション状態チェック
- syslog AI 連携

## read-only / 変更系の区別
read-only。状態収集と通知のみ。
