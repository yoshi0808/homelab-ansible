## 指摘事項

### [High] 削除直前に `active=false` を確認していない

対象: `roles/cloudkey_cert_deploy/tasks/deploy.yml:76-81,147-160`

削除候補はアップロード前の一覧から `source == uploaded` だけで抽出されており、
要求仕様 §5.4 / §6 の「`source=uploaded` かつ非アクティブのものに限定する」
条件を満たしていない。

この一覧には取得時点でアクティブな既存証明書も含まれる。有効化後に配信証明書の
指紋が一致したことは確認しているが、API上で各削除候補が非アクティブになったことは
確認していないため、API挙動の変更、切替状態の反映遅延、並行操作があった場合に
アクティブな uploaded 証明書を削除し得る。

新証明書の有効化と配信指紋確認後に証明書一覧を再取得し、削除対象を
`source == uploaded`、`active == false`、`id != cloudkey_new_id` の全条件で
再構成してから DELETE すること。可能なら削除直前の assert でも条件を固定すること。

### [Medium] 配信中の証明書チェーンが3階層であることを実通信で確認していない

対象: `roles/cloudkey_cert_deploy/tasks/issue.yml:76-94`,
`roles/cloudkey_cert_deploy/tasks/deploy.yml:123-143`

発行時に連結した PEM が3証明書を含むことと、配信中のリーフ証明書の指紋一致は
確認しているが、要求仕様 §6 の
「有効化後の配信チェーンがリーフ → Home-TLS-CA → Home-RADIUS-CA の3階層」
という確認は実装されていない。

CloudKeyがアップロードされたフルチェーンの一部しか配信しない場合でも、
現状のリーフ指紋検証は成功し、そのまま旧証明書を削除して成功通知を送る。
配信先から実際のチェーンを取得し、証明書数と各 issuer/subject または指紋を
期待する3階層と照合する処理を、旧証明書削除の前提条件に追加すること。
