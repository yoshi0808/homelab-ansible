# Test Result: time_sync_check

## 結論

**FAIL（修正後に再テストが必要）。**

通常実行の連続2回テストは安定して成功した。一方、承認済み test_plan の必須項目である
`--check --diff` は rc=2 で失敗した。check mode では `chronyc tracking` の command task が
skip され、その未収集状態を quory の同期異常と誤判定する。その結果、他ホストを確認せず、
Slack 通知タスクを実行して play を中断した。

したがって、通常実行時の read-only 動作と結果の安定性は確認できたが、dry-run 対応は
test_plan の期待を満たしていない。

## T1. `--check --diff`

実行コマンド:

```bash
ansible-playbook -i inventories/homelab/hosts.yml \
  playbooks/time_sync_check.yml --check --diff
```

結果: **FAIL（rc=2）。**

確認した挙動:

- `Check chrony tracking` が `skipping: [quory]` となった。
- 後続の `Judge chrony sync status` は実行され、収集結果がないため quory を同期異常と判定した。
- `Send Slack notification (quory NTP not synchronized)` が実行され、
  `changed: [quory -> localhost]` となった。
- `Fail because reference node time is not trustworthy` が failed となった。
- Phase 2 以降は実行されず、他の対象ホストは未確認となった。
- recap は quory が `ok=8 changed=1 failed=1 skipped=1`、その他の対象は収集 task が
  skip された状態だった。

安全性上の注意:

- dry-run で Slack 通知タスクが `changed` になっており、check mode が通知を抑止していない。
  `no_log: true` のためレスポンス本文は確認できないが、モジュールは skip されず実行された。
- test_plan の「エラーなく完了」「対象ホストへの変更が一切ない」という期待を満たさない。
