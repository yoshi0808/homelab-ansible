## 確認結果

### sophos-fw の date 直接比較

`roles/time_sync_check/tasks/main.yml` の Phase 3 は、コントローラー側 before/after を秒精度で取得し、sophos-fw 側の `date +%s%3N` と中点比較している。uutils-coreutils の `%3N` 問題を避けるためコントローラー側を `date +%s` にした判断は妥当。

Advanced Shell のメニュー遷移全体が測定区間に入るため精度限界はあるが、`time_sync_check_sophos_threshold_ms` を別閾値にし、detail に往復時間を残す設計になっている。要求仕様・実装報告にある注意とも整合している。

### cloudkey の Offset 抽出

`timedatectl timesync-status` の `Offset: +675us` のような出力から数値と単位を抽出し、`us` / `ms` / `s` を ms に正規化する実装になっている。`no_log: true` も設定されており、パスワード認証情報の扱いは方針に沿っている。

### 実行元ホストの動的除外

`time_sync_check_executor_host: "{{ lookup('pipe', 'hostname -s') }}"` により、controller の short hostname をもとに chrony 比較対象から実行元を除外している。quory / ansy 実行を前提とする要求仕様とは整合している。

### read-only / 変更系の分離

`time_sync_check.yml` は read-only のまま、NTP参照追加は `time_sync_ntp_reference.yml` と `roles/time_sync_ntp_reference/` に分離されている。CloudKey と sophos-fw を変更系 playbook から外した判断も、GUI/独自設定管理との衝突回避として妥当。
