## 修正内容の確認

### check mode での read-only 収集

以下の読み取り専用収集 task に `check_mode: false` が追加されている。

- chrony ホストの `chronyc tracking`
- Sophos の ntpclient ログ・mtime 収集
- CloudKey の `timedatectl status` 収集

これにより `--check` でも実データを収集し、未収集の default 値から quory を異常とする
偽陽性を防ぐ。いずれも状態変更を行わない診断コマンドであり、check mode 中に実行する判断は
妥当である。

### check mode での Slack 副作用抑止

time_sync_check から Slack を呼ぶ2経路の双方に `not ansible_check_mode` が設定されている。

- quory 異常時の critical 通知
- 対象ホストの warning / collection error 集約通知

したがって check mode では、診断結果が異常でも外部システムへ通知しない。通常実行では
従来の通知条件を維持している。fail task は診断結果を表すため check mode でも有効なままで、
通知という副作用だけを抑止する責務分離になっている。
