## 指摘事項

### [must-fix] CloudKey の `inactive` を `active` と判定する

対象:

- `roles/time_sync_check/tasks/main.yml` の `Judge NTP sync status (CloudKey)`

`time_sync_synced` は `('active' in _service_match)` でサービス状態を判定している。
`_service_match` が `NTP service: inactive` の場合も、文字列 `inactive` は部分文字列
`active` を含むため、この条件は true になる。`System clock synchronized: yes` と
`NTP service: inactive` の組み合わせを正常扱いし、007 review で修正したはずの
timesyncd 停止検出が回帰している。

ステータス値を正規表現の capture group で取り出し、`== 'active'` と完全一致で比較すること。
`System clock synchronized` も同様に値を抽出して `== 'yes'` と比較すれば、部分一致による
誤判定を避けられる。修正後は少なくとも以下を再検証すること。

- synchronized=yes / service=active: collected=true, synced=true
- synchronized=yes / service=inactive: collected=true, synced=false
- synchronized=no / service=active: collected=true, synced=false
- 必須行欠落: collected=false

008 implement では inactive ケースを検証していたが、010 の必須行解析修正で判定式が
部分一致へ変わった後、このケースが再実行されていない。今回の回帰を防ぐため、既存の異常系
ケースも含めて確認する必要がある。

collected=true になる。片方または両方が欠落すれば collection error へ分類される。
