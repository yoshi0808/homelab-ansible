# 追加実装内容: time_sync_check(CloudKeyの部分一致バグ修正)

対象レビュー: `docs/ai/reviews/time_sync_check/2026-06-23_011_review.md`

## 対応概要

正当な指摘。`'active' in _service_match` は文字列 `inactive` も部分文字列として
`active` を含むため真になり、`NTP service: inactive`(timesyncd停止状態)を
正常と誤判定する回帰があった。

正規表現のキャプチャグループでステータス値そのもの(`yes`/`no`、`active`/`inactive`)を
抜き出し、`== 'yes'` / `== 'active'` の完全一致比較に変更した。`System clock
synchronized:` も同様に値を抽出する形に統一した。
