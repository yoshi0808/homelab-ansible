## 変更内容の要約

### 1. [高] Section 5.2 / 5.3 / 6.4 — destination node healthcheck の復元

**問題:** 前回実装で「destination node healthcheck は実装にないため削除」と誤って記録した。実際は `playbooks/proxmox_evacuate_node.yml` の Step 2 で destination node に対して `proxmox_healthcheck` role を実行し、OK でなければ停止する。

**変更内容:**

- **Section 5.3（pve2 apply 前）:** ステップ 1 に「destination node（pve1）の healthcheck が OK であることを確認する」を追加
- **Section 6.4 実行条件:** 「destination node の healthcheck が OK（playbook レベルで実行）」を追加
- **Section 6.4 処理概要:** playbook と role の2層構造を明記。Step 2 として destination healthcheck を記述
- **Section 6.4 停止条件:** 「destination node の healthcheck が OK ではない」を追加

destination_node は target_node の反対側ノードとして playbook が自動決定する（pve2 → pve1、pve1 → pve2）。

### 2. [高] Section 5.6 — local disk migration の説明修正

**問題:** 前回実装で「`false` の場合、local disk を持つ VM/CT は migration 対象から除外される」と記述したが、実装に除外処理は存在しない。

**変更内容:**

- 「対象から除外」を削除
- 「`--with-local-disks` を省いて migration を試行。local disk があれば migration コマンドが失敗し、ロールが停止する」に修正
- evacuate 用（`proxmox_evacuate_allow_local_disk_migration`）と restore 用（`proxmox_restore_allow_local_disk_migration`）の2変数をそれぞれ明記
- 変数の意味を表形式で整理（`true`: `--with-local-disks` で実行 / `false`: オプション省略で試行・失敗時停止）

### 3. [中] Section 5.2 — タグなし VM/CT と force stop の説明修正

**問題:** 「タグなし」を「対象外（テンプレート等）・退避しない」とだけ記述し、force stop の可能性が伝わっていなかった。

**変更内容:**

- 「対象外（テンプレート等）・退避しない」→「明示 migration の対象外。ただし running のまま残れば force stop の対象になる」に変更
- `hacritical` タグは HA 登録状態を pvesh で照会するのではなく、タグの有無だけを判断基準とする旨を本文に追記

### 4. [中] Section 5.5 — 各ノード単位の restore に修正

**問題:** 「pve1 の patch 完了後に VM/CT を home node へ戻す」という記述が最後の pve1 restore だけを示しており、pve2 restore の存在が抜けていた。

**変更内容:**

- 「pve1 の patch / reboot / post-healthcheck が完了した後に home node へ戻す」→「各ノードの patch / post-patch healthcheck が完了した後、そのノードを restore する」に変更
- weekly_full では pve2 restore → pve1 evacuate → pve1 patch → pve1 restore の順であることを明記

### 5. [中] Section 6.6 — weekly_full の実行元制限を具体化

**問題:** 「quory など外部 control node から実行」という緩い記述で、許可リスト制限が伝わっていなかった。

**変更内容:**

- 「デフォルトは `quory` のみ。別のホストを使う場合は `proxmox_patch_weekly_full_allowed_controllers` を extra vars で明示 override が必要」を追記
- override のコマンド例を追加
