## 変更内容の要約

### 1. [低] Section 6.1 — evacuate の説明を Section 6.4 に合わせて修正

**問題:** Playbook 一覧の `proxmox_evacuate_node.yml` 欄に「patch対象ノード上のVM/CTを反対側ノードへ退避」と記述されており、Section 6.4 で修正済みの目的と一致していなかった。

**変更内容:**

```
変更前: patch対象ノード上のVM/CTを反対側ノードへ退避
変更後: patch対象ノードを reboot 可能な状態にする（migration / HA退避 / force stop）
```

### 2. [低] Section 11.1 Mode A フロー図 — evacuate を追加

**問題:** Mode A の短縮フロー図が `pve2 patch` から始まっており、patch 前に必須の evacuate が省略されていた。Section 11.3 や実装には evacuate が明記されているが、この図だけを見ると patch 前の必須処理がないように読めた。

**変更内容:**

- `pve2 patch` の前に `pve2 evacuate (non-HA migrate / HA退避 / force stop)` を追加
- `pve1 patch` の前に `pve1 evacuate (non-HA migrate / HA退避 / force stop)` を追加
