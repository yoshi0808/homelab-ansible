## 変更内容の要約

### Section 5.2: `hacritical` タグ定義の追加

**変更前:** `preferpve*` タグのみ定義。`hacritical` タグへの言及なし。
**変更後:** `hacritical` タグを追加定義し、HA/non-HA の分類とそれぞれの evacuate/restore 挙動を明記した。

追加内容:

- `hacritical` タグの意味（HA マネージャー管理対象の明示）
- タグ構成による3分類（non-HA migrate 対象 / HA管理対象 / 対象外）
- evacuate 時: non-HA は `qm migrate`/`pct migrate`、HA は `ha-manager crm-command node-maintenance enable` で自動退避
- restore 時: non-HA は `qm migrate`/`pct migrate` で手動復帰、HA は `ha-manager crm-command node-maintenance disable` で自動復帰し running になるまで待機

### Section 5.3: pve2 apply 前の処理ステップ更新

**変更前:** 一般的な記述（一覧化・自動退避確認・migrate/shutdown/skip）
**変更後:** 実装の処理順序に合わせて更新

- healthcheck → クラスタリソース取得 → タグ分類 → non-HA migrate → maintenance enable → HA退避待機 → 残存running を force stop

### Section 5.6: local disk migration の変数制御を明記

**変更前:** 「local-only storage を使っている場合は自動 migration を行わず停止する」と一律記述
**変更後:** `proxmox_evacuate_allow_local_disk_migration` 変数で制御することを明記

- デフォルト（role defaults）: `false`（安全側）
- 本 homelab（group_vars/proxmox.yml）: `true`（local-zfs 環境で明示的に承認済み）
- HA 管理 VM/CT はこのフラグの影響を受けないことを明記

### Section 6.4: evacuate 処理概要の全面修正

**変更前:** destination node healthcheck 確認・リソース余力確認・local storage確認・Sophos確認などが記述されていたが、実装に存在しない。
**変更後:** 実装の処理フローを正確に記述

主な差分:
- destination node のhealthcheck / リソース余力確認は実装にないため削除
- 実装のフローを明記: validate → healthcheck(target) → リソース取得 → タグ分類 → non-HA migrate → maintenance enable → HA待機 → 残存 force stop → report 保存
- 出力から「skipped VM/CT 一覧」「failed VM/CT 一覧」を削除（実装に存在しない）
- 「force-stopped VM/CT 一覧」を追加

### Section 6.6: weekly_full の目的・処理概要修正

**変更前:** 「最後に VM/CT を home node へ戻す」という一括 restore 記述。`NO_UPDATES` の扱い不明確。final healthcheck ステップが存在。full flow report が出力に含まれていた。
**変更後:** 各ノード単位の restore フローを反映

主な差分:
- 目的の記述: 「最後に一括 restore」→「各ノードのパッチ完了後、そのノードの VM/CT を restore する」
- `NO_UPDATES` の扱いを追記: skip フラグをセットして正常終了（dry-run ロールが通知済み）
- `MAINTENANCE_REQUIRED`/`BLOCKED`/`MAJOR_UPGRADE_DETECTED` は fail で停止と明記
- 処理概要: pve2 evacuate → patch → post-patch healthcheck → restore → pve1 evacuate → patch → post-patch healthcheck → restore → summary mail
- 「final healthcheck を実行する」ステップを削除（restore ロール内に post-restore healthcheck が含まれるため）
- 出力から「full flow report」「final healthcheck report」を削除

### Section 6.7: restore 処理概要の全面修正

**変更前:** 「home node 定義を読む」から始まる YAML ファイルを前提とした記述。maintenance mode への言及なし。
**変更後:** タグを正本とした実装フローを記述

主な差分:
- 「home node 定義を読む」を削除（YAML ファイルは使用しない、タグが正本）
- 実装のフローを明記: クラスタリソース取得 → タグで migrate 対象を特定 → non-HA migrate → 停止VM起動 → maintenance disable → HA復帰待機 → post-restore healthcheck → report 保存
- 停止条件から「home node 定義がない」を削除
- post-restore healthcheck が restore ロール内に含まれることを明記

### Section 11.1: Mode A フロー図の修正

**変更前:** フロー図の末尾が「VM/CT を home node へ戻す」で終わっていた
**変更後:** 「pve2 restore（VM/CT を home node へ戻す + post-restore healthcheck）→ pve1 patch → pve1 restore → summary mail」の構成に修正

### Section 11.2: Mode A スケジュール例の修正

**変更前:** 06:30 VM/CT 復帰 → 06:40 final healthcheck → 06:45 summary mail
**変更後:** 各ノードでの restore を個別スロットで表現し、final healthcheck を削除。`NO_UPDATES` 時の正常終了も明記。

### Section 11.3: Mode A 実行フローの修正

**変更前:** 21 ステップ（step 19: VM/CT 復帰、step 20: final healthcheck、step 21: summary mail）
**変更後:** 17 ステップ

主な差分:
- `NO_UPDATES` 時の動作を追記（skip フラグ・正常終了）
- evacuate の内部処理（non-HA migrate + maintenance enable + HA退避待機 + force stop）を簡潔に注記
- restore の内部処理（non-HA migrate back + 停止VM起動 + maintenance disable + HA復帰待機 + post-restore healthcheck）を簡潔に注記
- final healthcheck ステップを削除

### Section 17.1: Mode A 標準手順の修正

**変更前:** 17 ステップ（step 15: VM/CT 復帰、step 16: final healthcheck、step 17: summary mail）
**変更後:** 15 ステップ

主な差分:
- `NO_UPDATES` 時の正常終了を注記
- final healthcheck ステップを削除
- pve2/pve1 それぞれに evacuate と restore を明記
