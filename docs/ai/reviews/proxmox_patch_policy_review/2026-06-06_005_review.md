### 2. [高] local disk migration を無効にした場合の説明が実装と異なる

Policy は次のように記述しています。

```text
false の場合、local disk を持つ VM/CT は migration 対象から除外される。
```

- `docs/ai/prompts/proxmox_patch_policy.md:297`
- `docs/ai/prompts/proxmox_patch_policy.md:308`

実装には local disk の検出処理も、対象から除外する filter もありません。変数が `false` の場合は `qm migrate` から `--with-local-disks` を省略するだけで、タグ条件を満たす QEMU VM は引き続き migration コマンドの実行対象です。local disk が原因でコマンドが失敗すれば、role 全体が失敗します。

- `roles/proxmox_evacuate_node/tasks/main.yml:99`
- `roles/proxmox_evacuate_node/tasks/main.yml:126`
- `roles/proxmox_restore_vm_placement/tasks/main.yml:60`
- `roles/proxmox_restore_vm_placement/tasks/main.yml:79`

また restore では別変数 `proxmox_restore_allow_local_disk_migration` を使用しますが、Section 5.6 は evacuate 用変数しか定義していません。

- `roles/proxmox_restore_vm_placement/defaults/main.yml:19`
- `inventories/homelab/group_vars/proxmox.yml:7`

次の内容へ修正してください。

- evacuate は `proxmox_evacuate_allow_local_disk_migration` で `--with-local-disks` の付与を制御する
- restore は `proxmox_restore_allow_local_disk_migration` で制御する
- `false` は対象除外ではなく、オプションを付けずに migration を試行する
- migration 失敗時は停止する

### 3. [中] タグなし VM / CT を「退避しない」とだけ記載しており、force stop を表していない

Section 5.2 の表では、タグなし VM / CT を「対象外（テンプレート等）」「退避しない」としています。

- `docs/ai/prompts/proxmox_patch_policy.md:238`

実装上、タグなしまたは `preferpve*` / `hacritical` のどちらにも該当しない VM / CT は migration 対象にはなりません。ただし running のまま target node に残れば、最終確認ですべて `qm stop` / `pct stop` の対象になります。

- `roles/proxmox_evacuate_node/tasks/main.yml:95`
- `roles/proxmox_evacuate_node/tasks/main.yml:210`
- `roles/proxmox_evacuate_node/tasks/main.yml:230`

「明示 migration の対象外。ただし running で残存した場合は force stop」と記載してください。`hacritical` も実際の HA 登録状態を照会して分類しているのではなく、実装はタグを HA 管理対象の印として扱っていることを明確にすると誤解がありません。
