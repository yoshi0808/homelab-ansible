## 変更内容の要約

### 1. [中] Section 11.3 / 17.1 — dry-run の対象を "クラスタ固定ペア" に修正

**問題:** Section 11.3 は `5. pve2 patch dry-run`、Section 17.1 は `4. pve2 dry-run status PATCH_READY` と記述しており、pve2 単独実行のように見えた。

実装では `proxmox_patch_dryrun.yml` を import しており、Section 6.3 の記述通り pve1 / pve2 の固定ペアを対象とする。

**変更内容:**

- Section 11.3 ステップ 5: `pve2 patch dry-run` → `pve1 / pve2 dry-run（クラスタ固定ペア対象）`
- Section 17.1 ステップ 4: `pve2 dry-run status PATCH_READY` → `pve1 / pve2 dry-run status PATCH_READY（クラスタ固定ペア対象）`

### 2. [中] Section 6.4 目的 — 退避方針を正確に記述

**問題:** 「patch 対象ノード上の VM / CT をすべて反対側ノードへ退避する」という記述は、タグなし VM / CT が force stop されること（反対側へ移動しない）を表していなかった。

**変更内容:**

目的を以下に置き換えた:

> patch 対象ノードを reboot 可能な状態にする。対象 VM / CT は分類に応じて migration、HA 退避、または force stop する。non-HA VM / CT は反対側ノードへ migrate し、HA 管理 VM / CT は maintenance mode により HA が自動退避する。それ以外の running VM / CT は反対側へ移動せず対象ノード上で force stop する。

### 3. [中] Section 5.2 — report/mail 表示例を「将来の推奨表示」として区別

**問題:** evacuation / restore report は VM ID 配列と HA 数を JSON で保存するのみで、VM 名・current node・home node・action を一覧表示する実装は存在しない。Section 5.2 の表示例が実装と乖離していた。

**変更内容:**

- 「report / mail では、VM ID だけでなく以下のように表示する」という記述を削除
- 現在の実装（VM ID 配列と HA 数の JSON 保存）を説明する文を追加
- 元の表示例を「将来の推奨表示（未実装）」として明示的に区別

### 4. [低] Section 6.4 対象 — `destination_node` の指定例を削除

**問題:** `target_node: pve2 / destination_node: pve1` という YAML 例が、`destination_node` を外部から指定できるように見えた。実際には playbook が `target_node` のみを検証し、`destination_node` は常に反対側ノードとして内部決定する。

**変更内容:**

- `destination_node: pve1` を含む YAML 例を削除
- `destination_node` は playbook が自動決定することを明記
- `target_node` のみを指定する実際のコマンド例に置き換えた
