# 要求仕様: proxmox_patch_policy.md 整合性レビュー

## 目的

proxmox_patch_policy.md の記述と、以下の実装済みPlaybook / Roleの実装内容との整合性を確認する。
コードは動作確認済みのためgroundtruthとし、Policy側の記述を実装に合わせて修正する。

## 対象

- ドキュメント: docs/ai/prompts/proxmox_patch_policy.md
- Playbook / Role（以下の3本）:
  - playbooks/proxmox_evacuate_node.yml および対応role
  - playbooks/proxmox_restore_vm_placement.yml および対応role
  - playbooks/proxmox_patch_weekly_full.yml および対応role

## 作成・更新ファイル

- docs/ai/reviews/proxmox_patch_policy_review/2026-06-06_001_requirement.md（本ファイル）
- レビュー・修正工程で proxmox_patch_policy.md を更新する

## 実施内容

- 実装コードを読み、Policy記述との乖離を洗い出す
- 特に以下の観点を重点チェックする:
  - Proxmoxタグ（preferpve1 / preferpve2 / hacritical）を参照してVM home nodeおよびHA登録を判断している実装が、Policyに反映されているか
  - HA登録VMに対するevacuate / restore挙動（ha-manager crm-commandの利用など）がPolicyに十分記述されているか
  - non-HA VMのmigration方法（qm migrate --with-local-disks --online）がPolicyに記述されているか
  - proxmox_vm_home_nodes.ymlの実際の構造・フィールドがPolicyの定義例と一致しているか
  - weekly_fullフローの実装ステップがPolicyのSection 11 / Section 6.6と一致しているか
- 乖離箇所を列挙し、Policy側の修正案を提示する
- Policy修正はdocs/ai/prompts/proxmox_patch_policy.mdに直接反映する

## 制約

- コード側は変更しない
- Policy側のみ修正する
- 実装の動作を変えるような提案はしない
- 秘密情報を扱わない

- requirement.mdの保存のみ（実装・修正はCodexのレビュー結果を受けてから）

## 初回除外スコープ

- Policy以外のドキュメント修正
- weekly_full以外のPlaybookとの整合性チェック
