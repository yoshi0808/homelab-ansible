## 変更内容

### 1. roles/proxmox_patch_apply_node/defaults/main.yml

`proxmox_patch_apply_manual_confirm: ""` を追加。

- manual モード + MAINTENANCE_REQUIRED 検出時、この変数が `"MAINTENANCE_REQUIRED"` でなければ abort する
- 空文字列がデフォルトなので、`-e proxmox_patch_apply_mode=manual` だけでは続行できない

### 2. roles/proxmox_patch_apply_node/tasks/main.yml

**① target_node 許可リスト検証（block の外、最初のタスク）**

```yaml
- name: Validate target_node is pve1 or pve2
  ansible.builtin.assert:
    that:
      - target_node is defined
      - target_node in ['pve1', 'pve2']
    fail_msg: "target_node must be 'pve1' or 'pve2', got: ..."
  delegate_to: localhost
  become: false
```

**② 初期化に `_running_on_target: []` を追加**

**③ VM/CT 残存確認（block 内、control node チェックの直後）**

`pvesh get /cluster/resources --type vm` の結果 `_pvesh_raw` を再利用。target_node 上で `status == 'running'` の VM/CT を検出し、存在すれば abort。

```
abort_type: vms_not_evacuated
```

**④ manual モード強化**

変更前:
```
MAINTENANCE_REQUIRED 検出時 + manual → debug 警告して続行
```

変更後:
```
MAINTENANCE_REQUIRED 検出時 + manual + confirm != 'MAINTENANCE_REQUIRED' → abort
MAINTENANCE_REQUIRED 検出時 + manual + confirm == 'MAINTENANCE_REQUIRED' → debug 警告して続行
```

新しい abort_type: `manual_not_confirmed`

**⑤ メール件名・本文の分岐に新 abort_type を追加**

`vms_not_evacuated` / `manual_not_confirmed` を「前提チェック失敗」カテゴリに追加。

### 3. playbooks/proxmox_patch_apply_node.yml

Usage コメントに manual モードの実行例を追加:

```bash
# MAINTENANCE_REQUIRED 手動適用
ansible-playbook ... \
  -e target_node=pve2 \
  -e proxmox_patch_apply_mode=manual \
  -e proxmox_patch_apply_manual_confirm=MAINTENANCE_REQUIRED
```

## abort_type 一覧（更新後）

| abort_type | 発生条件 |
|---|---|
| `control_node` | ansy が target_node 上で稼働中 |
| `vms_not_evacuated` | 対象ノード上に running VM/CT が残存 |
| `healthcheck` | pre-healthcheck が WARNING / CRITICAL |
| `apt_update` | apt-get update 失敗 |
| `apt_check` | apt-get check 失敗 |
| `sim_failed` | apt-get -s dist-upgrade 失敗 |
| `maintenance_required` | MR 検出 + auto モード |
| `manual_not_confirmed` | MR 検出 + manual モード + 確認変数未設定 |
| `apply_failed` | apt-get dist-upgrade 失敗 |
| `reboot_timeout` | reboot 後 15 分で応答なし |
| `none` | 正常完了 |
