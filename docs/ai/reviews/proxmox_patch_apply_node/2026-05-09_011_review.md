## 改善確認

### 1. main play の hosts に default が追加された

`playbooks/proxmox_patch_apply_node.yml:29` が以下になっています。

```yaml
hosts: "{{ target_node | default('__invalid_target__') }}"
```

これにより、`target_node` 未指定時でも playbook parser が main play の `hosts` 評価で落ちなくなりました。

### 2. target_node 未指定・不正値ともに pre-play assert で止まる

`playbooks/proxmox_patch_apply_node.yml:17` の localhost pre-play が先に実行され、`target_node` 未指定・不正値のどちらも `playbooks/proxmox_patch_apply_node.yml:21` の assert で停止します。

このため、誤指定時に Proxmox ノードへ接続しに行く前に止められます。

## 残っている既知リスク

以下は今回の 010_implement では対象外であり、引き続き既知リスクです。

- 通知保証は best-effort のまま
- control node チェックの name 固定・部分一致の脆弱性
- post-healthcheck 判定が healthcheck role と一致していない
- `MAJOR_UPGRADE_DETECTED` / `BLOCKED` 区別が粗い
- manual confirm 文字列が `MAINTENANCE_REQUIRED` のみで、target node を含まない
