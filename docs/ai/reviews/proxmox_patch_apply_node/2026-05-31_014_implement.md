### 指摘2（低）: 初回 post-healthcheck が UNKNOWN でも retry 対象に → 修正済み

**問題**

retry 条件が `_post_hc_status == 'CRITICAL'` のみだった。
reboot 直後にコマンド失敗・不正JSON が発生して `UNKNOWN` になった場合は、
retry 対象外のままだった。

retry task 内部はコマンド失敗・不正JSON を CRITICAL 扱いで再試行する設計に
なっているため、初回 post-healthcheck でも同じ一貫性を持たせるべき。

**修正**

retry タスクの `when:` と `Re-determine` タスクの `when:` を以下に変更した。

```yaml
when:
  - _post_hc_status in ['CRITICAL', 'UNKNOWN']
  - _apply_reboot_done | bool
```

- CRITICAL: JSON 取得成功だが health 判定 NG
- UNKNOWN: healthcheck スクリプト自体が失敗（コマンドエラー・不正JSON など）

どちらも reboot 直後の一時不安定として再試行する。
reboot を伴わない CRITICAL / UNKNOWN はリトライしない（本物の障害として扱う）。

**あわせてポリシー文書も更新**

`proxmox_patch_policy.md` の retry 説明を UNKNOWN も対象に含む表現に修正した。
