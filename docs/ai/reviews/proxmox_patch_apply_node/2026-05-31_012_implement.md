## 実施内容

### 指摘1（中）: mail vars assert が空値を通す → 修正済み

**問題**

`is defined` のみの確認では以下のような空値が通過し、
apply 後のメール送信で失敗する。

```yaml
smtp_host: ""
smtp_port: ""
```

**修正**

各変数に長さ・値チェックを追加した。

```yaml
that:
  - smtp_host is defined
  - smtp_host | string | length > 0
  - smtp_port is defined
  - smtp_port | int > 0
  - smtp_user is defined
  - smtp_user | string | length > 0
  - smtp_password is defined
  - smtp_password | string | length > 0
  - mail_to is defined
  - mail_to | string | length > 0
```

- `smtp_port`: 数値として `> 0` を確認（"" や 0 を弾く）
- その他: `| string | length > 0` で空文字列を弾く
- fail_msg を「未定義または空値」の表現に更新

### 指摘2（低）: ポリシー文書の動作例に「即時実行」「delay」が残っている → 修正済み

**問題**

現在の実装は各 retry 試行の冒頭で `pause` するが、文書には：

```
→ post-healthcheck 2回目: CRITICAL   （リトライタスク 試行1、即時実行）
→ 60秒待機                            （delay）
```

と古い `retries/delay` 方式の表現が残っていた。

**修正**

各試行の冒頭 pause を明示する表現に統一した。

```text
→ post-healthcheck 1回目: CRITICAL   （初回）
→ 60秒待機                            （リトライ試行1の冒頭 pause）
→ post-healthcheck 2回目: CRITICAL   （リトライ試行1）
→ 60秒待機                            （リトライ試行2の冒頭 pause）
→ post-healthcheck 3回目: OK         （リトライ試行2）
```
