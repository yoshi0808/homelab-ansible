## 指摘

### 1. 中: 新規 task file がまだ untracked のまま

該当箇所:

- `roles/proxmox_patch_apply_node/tasks/main.yml:416`
- `roles/proxmox_patch_apply_node/tasks/retry_healthcheck_once.yml`

`main.yml` は `retry_healthcheck_once.yml` を include する。

```yaml
ansible.builtin.include_tasks: retry_healthcheck_once.yml
```

しかし `git status --short` では、対象ファイルはまだ untracked。

```text
?? roles/proxmox_patch_apply_node/tasks/retry_healthcheck_once.yml
```

実装メモでは「commit 時に含める」と整理されているが、現時点では git diff にも出ないため、レビューや commit で見落としやすい。

このファイルが commit されないと、別環境で `include_tasks` が失敗する。

対応:

```bash
git add roles/proxmox_patch_apply_node/tasks/retry_healthcheck_once.yml
```

commit 前に `git status --short` で `?? roles/proxmox_patch_apply_node/tasks/retry_healthcheck_once.yml` が消えていることを確認する。

### 2. 低: defaults のコメントが CRITICAL のみのまま

該当箇所:

- `roles/proxmox_patch_apply_node/defaults/main.yml:47`
- `roles/proxmox_patch_apply_node/defaults/main.yml:48`

実装と policy は `CRITICAL or UNKNOWN` を retry 対象にしているが、defaults のコメントはまだ以下のまま。

```yaml
# Post-healthcheck retry on CRITICAL after reboot.
# Retried only when status is CRITICAL and reboot was performed.
```

動作には影響しないが、設定値を読む人には「UNKNOWN は対象外」と誤解されやすい。

対応案:

```yaml
# Post-healthcheck retry on CRITICAL/UNKNOWN after reboot.
# Retried only when status is CRITICAL or UNKNOWN and reboot was performed.
```
