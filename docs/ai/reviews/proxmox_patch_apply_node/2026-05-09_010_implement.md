## 対応概要

指摘 1（target_node 未指定時に pre-play assert まで到達しない）を修正した。

---

## 変更内容

### 指摘 1: main play の hosts に default を追加

**変更ファイル:** `playbooks/proxmox_patch_apply_node.yml`

変更前:
```yaml
hosts: "{{ target_node }}"
```

変更後:
```yaml
hosts: "{{ target_node | default('__invalid_target__') }}"
```

**効果:**

`target_node` 未指定時、playbook parser が `hosts` 評価で落ちなくなった。
代わりに play 1（localhost pre-play）の assert が先に実行され、意図したエラーメッセージを出す。

```text
TASK [Assert target_node is pve1 or pve2]
FAILED! => target_node must be 'pve1' or 'pve2', got: 'undefined'
```

play 1 が失敗すると Ansible は play 2 を開始しないため、`__invalid_target__` に接続しようとすることはない。

---

## 指摘 2 について（修正なし）

syntax-check は task 内の assert を実行しない。そのため、`target_node=foo` などの不正値を syntax-check で検出することはできない。

これは Ansible の仕様であり、通常実行では pre-play の assert が正しく動作する。
syntax-check OK は「YAML 構文が正しい」を示すものであり、target allowlist の動作確認ではない。
