## 対応内容

### 指摘 1: target_node 検証を role 到達前に効かせる

**変更ファイル:** `playbooks/proxmox_patch_apply_node.yml`

playbook に pre-play（localhost）を追加し、main play の `hosts: "{{ target_node }}"` が評価される前に assert を実行する。

```yaml
- name: Validate target_node before main play
  hosts: localhost
  gather_facts: false
  tasks:
    - name: Assert target_node is pve1 or pve2
      ansible.builtin.assert:
        that:
          - target_node is defined
          - target_node in ['pve1', 'pve2']
        fail_msg: "target_node must be 'pve1' or 'pve2', got: ..."
```

role 内の assert（`tasks/main.yml:6`）はそのまま残す（二重ガード）。

### 指摘 2: pvesh で CT も取得する

**変更ファイル:** `roles/proxmox_patch_apply_node/tasks/main.yml`

`pvesh get /cluster/resources --type vm` から `pvesh get /cluster/resources`（type フィルターなし）に変更し、JSON 解析時に type フィルターをかける。

- control node チェック: `selectattr('type', 'equalto', 'qemu')` を追加（VM のみ対象）
- VM/CT 残存チェック: `selectattr('type', 'in', ['qemu', 'lxc'])` を追加（VM + CT の両方を対象）

### 指摘 3: メール本文の apply 未実施条件に新 abort_type を追加

**変更ファイル:** `roles/proxmox_patch_apply_node/tasks/main.yml`

```yaml
# 変更前
{% if not (_apply_abort_type in ['control_node', 'healthcheck', 'apt_update', 'apt_check', 'sim_failed', 'maintenance_required']) %}

# 変更後
{% if not (_apply_abort_type in ['control_node', 'healthcheck', 'vms_not_evacuated', 'apt_update', 'apt_check', 'sim_failed', 'maintenance_required', 'manual_not_confirmed']) %}
```

`vms_not_evacuated` / `manual_not_confirmed` が abort した場合、apply セクションには「(前提チェック失敗のため未実施)」が表示されるようになった。
