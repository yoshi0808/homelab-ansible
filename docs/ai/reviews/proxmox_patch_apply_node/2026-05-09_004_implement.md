# proxmox_patch_apply_node 追加実装（apply_mode 追加）

実装日: 2026-05-09
対応要求仕様: 2026-05-09_003_requirement.md

---

## 対応概要

`proxmox_patch_apply_mode` 変数を追加し、MAINTENANCE_REQUIRED 検出時の動作を分岐した。

---

## 変更内容

### 1. roles/proxmox_patch_apply_node/defaults/main.yml

以下を追加した（`proxmox_patch_apply_reboot_timeout` の直後）:

```yaml
# Apply mode: controls behavior when MAINTENANCE_REQUIRED is detected in re-dry-run
# auto:   abort with email (default, for automated PATCH_READY apply)
# manual: warn and continue (for intentional MAINTENANCE_REQUIRED apply)
proxmox_patch_apply_mode: "auto"
```

### 2. roles/proxmox_patch_apply_node/tasks/main.yml — Phase 2 判定修正

変更前:

```yaml
- name: Abort if MAINTENANCE_REQUIRED detected in re-dry-run
  when: _redryrun_is_mr | bool
  block:
    - ansible.builtin.set_fact:
        _apply_abort_type: "maintenance_required"
        _apply_error_msg: "MAINTENANCE_REQUIRED: ..."
    - ansible.builtin.fail:
        msg: "..."
```

変更後:

```yaml
- name: Abort if MAINTENANCE_REQUIRED detected (auto mode)
  when: (_redryrun_is_mr | bool) and proxmox_patch_apply_mode == 'auto'
  block:
    - ansible.builtin.set_fact:
        _apply_abort_type: "maintenance_required"
        _apply_error_msg: "MAINTENANCE_REQUIRED: ..."
    - ansible.builtin.fail:
        msg: "..."

- name: Warn and continue if MAINTENANCE_REQUIRED detected (manual mode)
  ansible.builtin.debug:
    msg: "[WARNING] apply_mode=manual: MAINTENANCE_REQUIRED detected but continuing: ..."
  when: (_redryrun_is_mr | bool) and proxmox_patch_apply_mode == 'manual'
```

---

## 動作まとめ

| proxmox_patch_apply_mode | MAINTENANCE_REQUIRED 検出時 |
|---|---|
| `auto`（デフォルト） | abort + メール送信 |
| `manual` | `[WARNING]` デバッグ出力 + 続行 |

- `auto` の場合: `_apply_abort_type: "maintenance_required"` がセットされ、rescue → always でメール送信 → fail
- `manual` の場合: `_apply_abort_type` は `none` のまま。メール件名は post-healthcheck 結果に応じて `[SUCCESS]` / `[WARNING]` となる
