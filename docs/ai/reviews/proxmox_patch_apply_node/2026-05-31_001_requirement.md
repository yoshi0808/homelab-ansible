# 要求仕様: post-healthcheck リトライとメール通知修正

作成日: 2026-05-31

## 目的
reboot直後のpost-healthcheckがCRITICALになる問題を改善する。
サービスが起動しきる前にhealthcheckが走ることが原因と思われるため、
CRITICALの場合に待機してからリトライする仕組みを追加する。
あわせて、apply成功時のメール通知が届いていない問題も修正する。

## 対象
- グループ: proxmox
- ホスト: pve1, pve2

## 作成・更新ファイル
- roles/proxmox_patch_apply_node/tasks/main.yml
- roles/proxmox_patch_apply_node/defaults/main.yml

## 実施内容

### 1. post-healthcheck リトライ

post-healthcheckがCRITICALの場合、一定時間待機してから再度healthcheckを実行する。
再確認でOKになった場合はSUCCESSとして扱う。
待ち時間・リトライ回数はdefaults/main.ymlで設定可能にする。

### 2. メール通知修正

apply成功時も必ずsummary mailが送信されることを確認・修正する。

## 制約
- 既存の処理フロー・変数名は壊さない
- 秘密情報を扱わない
- 変更系（変更あり: tasks/main.yml, defaults/main.yml）

## 初回実装スコープ
- post-healthcheck CRITICAL時のリトライロジック追加
- defaults/main.ymlにリトライ設定追加
- メール送信タスクの確認・修正

## 初回除外スコープ
- control node検出ロジックの修正（ansy-old削除により解消済み）

## shell / Ansible 責務分離
- shell: 対象外（今回は触らない）
- Ansible tasks: リトライロジック・メール送信の修正を担当
