# 要求仕様: apply後のreboot判定改善

## 目的
カーネル更新が適用されても /var/run/reboot-required が作られない場合がある。
実際に今回pve2でカーネルパッチ適用後にrebootがスキップされた。
uname -r とインストール済みカーネルバージョンの比較を追加し、
差分がある場合はreboot_requiredと判定する。

## 対象
- グループ: proxmox
- ホスト: pve1, pve2

## 作成・更新ファイル
- roles/proxmox_patch_apply_node/files/proxmox-patch-apply-collect.sh
  （apply後のreboot判定スクリプト）

## 実施内容
/var/run/reboot-required の確認に加えて、以下を追加する。

```bash
running_ver=$(uname -r | sed 's/-pve//')
for pkg in proxmox-kernel-6.17 proxmox-kernel-7.0; do
  installed_ver=$(dpkg-query -W -f='${Version}' "$pkg" 2>/dev/null)
  if [ -n "$installed_ver" ]; then
    if dpkg --compare-versions "$installed_ver" gt "$running_ver"; then
      reboot_required="true"
      break
    fi
  fi
done
```

reboot_required の判定は以下のいずれかで true とする：
- /var/run/reboot-required が存在する
- インストール済みカーネルバージョンが現在の稼働カーネルより新しい

## 参照ファイル
- roles/proxmox_patch_apply_node/files/（既存スクリプトを確認して修正）

## 制約
- shell 側は収集と判定のみ
- 既存の動作を壊さない
- 秘密情報を扱わない

## 初回実装スコープ
- 上記スクリプトの修正のみ

## 初回除外スコープ
- pve1/pve2以外のノード対応

## shell / Ansible 責務分離
- shell: reboot_required の判定結果をJSONに含める
- Ansible tasks: その結果を受けてrebootを実行する（既存の仕組みを流用）
