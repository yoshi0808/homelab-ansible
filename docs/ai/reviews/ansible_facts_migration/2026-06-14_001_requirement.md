# 要求仕様: ansible_facts 移行（INJECT_FACTS_AS_VARS deprecation 対応）

作成日: 2026-06-14
版: 1.0（要求仕様）
対象: リポジトリ全体の fact 由来 `ansible_*` トップレベル注入変数

---

## 1. 目的

ansible-core 2.24 で `INJECT_FACTS_AS_VARS` のデフォルトが `False` になる予告
（DEPRECATION WARNING）に先行対応する。fact 由来の `ansible_*` トップレベル
注入変数を `ansible_facts['...']` 辞書参照へ移行し、警告を解消する。

---

## 2. 調査（実装前に必ず実施）

機械的な文字列置換の前に、リポジトリ全体で以下を調査・報告する。
表面的な grep ヒットだけでなく、依存関係・参照経路を確認する。

1. fact 由来 `ansible_*` 注入変数の全使用箇所の棚卸し
   - tasks / templates(.j2) / vars / defaults / when / assert / loop /
     set_fact / debug / フィルタ式の中まで含めて洗う
   - 特に `templates/*.j2` 内の参照は grep から漏れやすいので重点確認
2. 各使用箇所が属するプレイ/ロールの `gather_facts` 状態を確認し、
   fact が収集済みかを検証する（収集前参照があれば別途対応が必要）
3. ロールが複数 playbook から呼ばれる場合、呼び出し元すべての
   `gather_facts` 前提を確認する（cloudkey_cert_deploy role など）
4. 接続予約変数（`ansible_host` / `ansible_user` /
   `ansible_ssh_private_key_file` / `ansible_connection`）、マジック変数
   （`ansible_play_hosts` / `ansible_failed_result` 等）、ユーザー定義変数
   （`systemd_ansible_timers` 等）は対象外として明確に除外する

---

## 3. 変換ルール

- `ansible_<key>` → `ansible_facts['<key>']` のみ。
- 比較・代入の右辺（例: `== 'quory'`）は変更しない。
- 既知の対象（調査で追加が出る可能性あり）:
    - `ansible_hostname`  → `ansible_facts['hostname']`
    - `ansible_date_time` → `ansible_facts['date_time']`

---

## 4. 制約

- 実行ホスト制限の assert（安全装置）の挙動を変えないこと。
- 接続予約変数・マジック変数・ユーザー定義変数は絶対に触らない。

---

## 5. 確認項目

- 全 playbook の `--syntax-check` が PASS。
- cert_renew / cert_renew_quory / cloudkey_cert_deploy の assert が
  従来通り機能（quory 以外で fail、quory/ansy で通過）。
- DEPRECATION WARNING (`INJECT_FACTS_AS_VARS`) が実行時に消えること。
- 調査で判明した全使用箇所が漏れなく移行されていること。
