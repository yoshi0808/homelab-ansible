## 2. 指摘内容と原因

### 指摘（003_review.md / Low）

`2026-06-14_002_implement.md:96` の「`INJECT_FACTS_AS_VARS` には環境変数バインドが
無い」という記述が誤り。この環境（ansible-core 2.20.1）には env バインド
`ANSIBLE_INJECT_FACT_VARS` が存在し、`False` が反映される。

### 原因（なぜ矛盾が生じたか）

意図的に残したものではなく、初回検証時の単純な誤りである。

- env 名を config 名 `INJECT_FACTS_AS_VARS` から推測し、
  `ANSIBLE_INJECT_FACTS_AS_VARS`（**FACTS** 複数形）で試した。
- それが `ansible-config dump` に反映されなかったため、「env バインドが無い」と
  誤って断定し、一時 ansible.cfg にフォールバックした。
- 実際の env バインドは `ANSIBLE_INJECT_FACT_VARS`（**FACT** 単数形）。
  ansible 自身の命名が config 名（FACTS）と env 名（FACT）で不一致のため、
  推測が外れた。
- なお、一時 ansible.cfg を用いた検証「結論」（注入 OFF 時に旧 `ansible_*` が
  未定義になり、新 `ansible_facts[...]` が解決し assert が設計どおり動く）自体は
  有効で、コードの正当性評価には影響しない。

## 4. 再検証（正しい env バインドで実施）

検証ホストは ansy（開発機）。

### 4.1 env バインドの存在確認

```bash
ansible-config list | grep -A12 '^INJECT_FACTS_AS_VARS:'
```

```text
INJECT_FACTS_AS_VARS:
  env:
  - name: ANSIBLE_INJECT_FACT_VARS
  - key: inject_facts_as_vars
```

```bash
ANSIBLE_INJECT_FACT_VARS=False ansible-config dump --only-changed | grep -i inject
# INJECT_FACTS_AS_VARS(env: ANSIBLE_INJECT_FACT_VARS) = False
```

（参考: 当初使った `ANSIBLE_INJECT_FACTS_AS_VARS`（FACTS 複数形）は反映されない＝
誤った env 名であったことを再確認した。）

### 4.2 注入 OFF 時の解決（env バインドで再現）

```bash
ANSIBLE_INJECT_FACT_VARS=False ansible-playbook -i inventories/homelab/hosts.yml <tmp play>
# msg: "old=<<UNDEFINED>> new=ansy"
```

- 旧形式 `ansible_hostname` → `<<UNDEFINED>>`（旧コードなら 2.24 で破綻を再現）。
- 新形式 `ansible_facts['hostname']` → `ansy`（正常解決）。

### 4.3 安全装置 assert（env バインドで再確認）

```bash
ANSIBLE_INJECT_FACT_VARS=False ansible-playbook -i inventories/homelab/hosts.yml \
  playbooks/cert_renew_quory.yml
```

```text
TASK [Ensure this playbook is running on quory]
fatal: [localhost]: FAILED! =>
  "assertion": "ansible_facts['hostname'] == 'quory'",
  "evaluated_to": false,
  "msg": "cert_renew_quory.yml must be run on quory only (current host: ansy)."
```

- quory 以外（ansy）で assert が fail し、fail_msg の
  `{{ ansible_facts['hostname'] }}` が `ansy` に解決。安全装置は従来どおり機能。
- 実行中に `INJECT_FACTS_AS_VARS` の DEPRECATION WARNING は出ない。
