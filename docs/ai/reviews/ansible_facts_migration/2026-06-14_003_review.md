# Review: ansible_facts 移行

Review date: 2026-06-14

## 結論

fact 由来のトップレベル注入変数は、要求仕様どおり
`ansible_facts[...]` 参照へ移行されている。

実行ホスト制限の比較条件、CloudKey 証明書名の形式、接続予約変数・
マジック変数・ユーザー定義変数には不要な変更がなく、コードに
commit を止める問題は確認されなかった。

実装報告に検証方法の軽微な事実誤認が1件あるが、実装の正当性には影響しない。
コード差分はこの内容で commit してよい。

## Findings

### Low: INJECT_FACTS_AS_VARS の環境変数バインドに関する説明が誤っている

- File: `docs/ai/reviews/ansible_facts_migration/2026-06-14_002_implement.md:96`

実装報告には「`INJECT_FACTS_AS_VARS` には環境変数バインドが無い」とあるが、
この環境の ansible-core 2.20.1 では `ANSIBLE_INJECT_FACT_VARS` が定義されている。

実際に以下で `False` が反映されることを確認した。

```bash
ANSIBLE_INJECT_FACT_VARS=False ansible-config dump --only-changed
```

```text
INJECT_FACTS_AS_VARS(env: ANSIBLE_INJECT_FACT_VARS) = False
```

一時 `ansible.cfg` を使った検証結果自体が無効になるわけではないが、
レビュー履歴の正確性のため、追加実装時または final 前に説明を修正するとよい。

## 要求仕様との整合

- `ansible_hostname` の実参照はすべて
  `ansible_facts['hostname']` へ移行されている。
- `ansible_date_time.iso8601_basic_short` は
  `ansible_facts['date_time']['iso8601_basic_short']` へ移行されている。
- assert の右辺 `quory`、許可ホスト `['quory', 'ansy']` は変更されていない。
- 対象3 playbook は参照前に `gather_facts: true` を実行している。
- `cloudkey_cert_deploy` role の呼び出し元は
  `playbooks/cloudkey_cert_deploy.yml` のみで、同じ fact 収集済み play 内から
  `include_role` されるため、`date_time` の参照条件を満たす。
- `ansible_host`、`ansible_user`、`ansible_ssh_private_key_file`、
  `ansible_connection` は接続予約変数として変更されていない。
- `ansible_failed_result`、`ansible_play_hosts` はマジック変数として
  変更されていない。
- `systemd_ansible_timers` はユーザー定義変数として変更されていない。
- `proxmox_patch_weekly_full.yml` に残る `ansible_hostname` はコメントだけであり、
  実行時参照ではない。
- `core.md` の秘密情報、名前解決、変更系 playbook 分離の方針に影響する変更はない。
- `ubuntu_vm_patch_policy.md` が定義する Ubuntu patch/reboot/通知動作への変更はない。

## Verification

リポジトリ全体の `*.yml`、`*.yaml`、`*.j2`、`*.cfg` を再検索し、
未移行の fact 由来実参照がないことを確認した。

全 playbook に対して syntax-check を実行し、すべて PASS した。
`target_node` が必要な2 playbookには `-e target_node=pve2` を指定した。

```bash
ANSIBLE_LOCAL_TEMP=/tmp/ansible-local \
ANSIBLE_REMOTE_TEMP=/tmp/ansible-remote \
ansible-playbook --syntax-check \
  -i inventories/homelab/hosts.yml playbooks/<playbook>.yml
```

`cert_renew_quory.yml` は ansy 上で、注入設定がデフォルトの場合と
`ANSIBLE_INJECT_FACT_VARS=False` の場合の両方を実行した。
いずれも `ansible_facts['hostname']` が `ansy` に解決され、
quory 限定 assert が設計どおり失敗した。実行中に
`INJECT_FACTS_AS_VARS` の DEPRECATION WARNING は出なかった。

`git diff --check` も PASS した。
