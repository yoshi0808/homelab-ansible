# 追加実装内容 Sophos Firewall SSD Trim

## 対応レビュー

- docs/ai/reviews/sophos_trim/2026-06-07_003_review.md

## 修正概要

レビュー指摘 [High] 「`fstrim` の失敗を検出できず成功通知になる」に対応した。

SSH 終了コード（`trim_result.rc`）だけでは各 `fstrim` コマンドの成否を判定できないため、コマンドの終了コードをマーカー文字列として標準出力へ出力し、Ansible 側でそのマーカーを検査する方式に変更した。

---

## 変更ファイル

| ファイル | 操作 |
|---|---|
| `roles/sophos_trim/tasks/main.yml` | 更新（fstrim コマンドとステータス判定の変更） |

他のファイル（playbook・inventory・defaults・requirement）は変更なし。

---

## 変更内容の詳細

### 1. fstrim コマンドへのマーカー追加

**変更前:**

```yaml
'# ':
  - "fstrim -v /var"
  - "fstrim -v /content"
  - "exit"
```

**変更後:**

```yaml
'# ':
  - "fstrim -v /var; echo __SOPHOS_TRIM_VAR_RC__:$?"
  - "fstrim -v /content; echo __SOPHOS_TRIM_CONTENT_RC__:$?"
  - "exit"
```

各 `fstrim` の直後に `echo __SOPHOS_TRIM_*_RC__:$?` を実行し、終了コードを固有マーカーとして標準出力へ出力する。マーカー名は `fstrim` 自体の出力（バイト数等）と衝突しない固有文字列とした。

### 2. ステータス判定の変更

**変更前:**

```yaml
trim_status: "{{ 'SUCCESS' if (trim_result.rc is defined and trim_result.rc == 0) else 'FAILED' }}"
```

**変更後:**

```yaml
trim_status: >-
  {{
    'SUCCESS'
    if (
      trim_result.rc is defined
      and trim_result.rc == 0
      and '__SOPHOS_TRIM_VAR_RC__:0' in (trim_result.stdout | default(''))
      and '__SOPHOS_TRIM_CONTENT_RC__:0' in (trim_result.stdout | default(''))
    )
    else 'FAILED'
  }}
```

SUCCESS となる条件を以下の 3 つすべての AND に変更した。

| 条件 | 意味 |
|---|---|
| `trim_result.rc == 0` | SSH セッションが正常終了した |
| `__SOPHOS_TRIM_VAR_RC__:0` が stdout に含まれる | `/var` の fstrim が終了コード 0 で完了した |
| `__SOPHOS_TRIM_CONTENT_RC__:0` が stdout に含まれる | `/content` の fstrim が終了コード 0 で完了した |

いずれか一つでも満たさない場合は FAILED となり、メール件名に `FAILED` が入り、最終タスクで playbook が失敗終了する。
