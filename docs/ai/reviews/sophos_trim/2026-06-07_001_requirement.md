# 要求仕様: Sophos Firewall SSD Trim

## 目的
Sophos Firewall VMに対して月次でSSD Trimを実行する。

## 対象
- ホスト: sophos.internal（/etc/hostsに登録済み）
- SSHユーザー: admin
- 秘密鍵: ~/.ssh/id_rsa_sophos
- 実行ユーザー: yoshi（become不要）

## 接続方式
SSHで接続するとインタラクティブなCLIメニューが表示される。
ansible.builtin.expectモジュールを使って操作する。
pexpect 4.9.0はquoryにインストール済み。

## 操作フロー
1. SSH接続（admin@sophos.internal）
2. メインメニューで 5 + Enter（Device Management）
3. Device Managementメニューで 3 + Enter（Advanced Shell）
4. fstrim -v /var を実行
5. fstrim -v /content を実行
6. exit + Enter（Advanced Shellを終了）
7. 0 + Enter（Device Managementを終了）
8. 0 + Enter（メインメニューを終了、SSH切断）

## 期待するプロンプト文字列
- メインメニュー: "Select Menu Number [0-7]:"
- Device Managementメニュー: "Select Menu Number [0-4]:"
- Advanced Shell: "#"（rootシェルのプロンプト）
- Advanced Shell終了後: "Select Menu Number [0-4]:"
- メインメニュー戻り: "Select Menu Number [0-7]:"

## 実行結果の扱い
- fstrimの出力（trimされたバイト数等）をAnsibleのログに記録する
- 成功/失敗をメール通知する

## 作成ファイル
- playbooks/sophos_trim.yml（新規）
- roles/sophos_trim/tasks/main.yml（新規）
- roles/sophos_trim/defaults/main.yml（新規）

## inventoryへの追加
inventories/homelab/hosts.ymlにsophosグループを追加する。

```yaml
sophos:
  hosts:
    sophos:
      ansible_host: sophos.internal
      ansible_user: admin
      ansible_ssh_private_key_file: ~/.ssh/id_rsa_sophos
      ansible_connection: ssh
```

## 制約
- become不要
- 秘密鍵はGit管理しない
- 変更系（SSD Trimあり）
- Semaphoreから手動実行を想定（月次スケジュール）

## 初回除外スコープ
- Sophosのhealthcheck
- Trim実行前後のディスク使用量比較
