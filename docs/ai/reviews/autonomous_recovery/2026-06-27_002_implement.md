## 各実装の設計ポイント

### 共通設計

- 全 playbook は `hosts: pve1` (変更系 Proxmox 操作の実行元)
- タグ再検証は必ず Phase 1 で実行 (policy §5: Ansible tasks 側で判定、shell 側では判定しない)
- `block: / rescue: / always:` 構造により、失敗時も必ず JSON レポート + Slack 通知を実行
- target が不適格な場合は assert で即 fail (playbook レベルで制御)
- Slack 通知は `common_slack/tasks/notify.yml` の既存 Incoming Webhook 方式を流用 (policy §10)
- JSON レポートは `reports/recovery_investigations/<target>/<timestamp>.json` に保存 (policy §11)
- IPリテラルなし (core.md §3)

### A: recovery_service_restart.yml

- authy: `freeradius.service`、monnie: `grafana-server / prometheus / loki / unpoller`
- 対応サービスは `recovery_service_restart_units` dict で管理 (defaults/main.yml)
- サービス restart は `delegate_to: "{{ target }}"` で対象 VM に SSH して実行
- Phase 3 で systemd の `ActiveState` をポーリング (12回 × 5秒 = max 60s)
- VM が running でない場合は assert で fail し "recovery_vm_reboot.yml を検討してください" を示唆

### B: recovery_vm_reboot.yml

- QEMU guest agent 有無 (policy §⑥確認済み):
  - authy / monnie: あり → `pvesh .../agent/shutdown` を第1選択
  - sophos-fw: なし → `pvesh .../status/shutdown` (ACPI) のみ
- agent shutdown が失敗した場合も ACPI shutdown にフォールバック (ignore_errors)
- ソフト停止タイムアウト(既定 60s)後に `pvesh .../status/stop` (強制電源断) にフォールバック
- 強制電源断使用の有無を `_rvr_used_force_stop` で追跡し、レポート + Slack 通知に反映
- 全シャットダウン・起動の pvesh コマンドはノード URL に current_node を明示
  (`/nodes/{{ _rvr_vm_node }}/qemu/{{ _rvr_vmid }}/...`)
  これにより pve1 から実行しながら pve2 上の VM も操作可能

### C: recovery_ha_failover.yml

- 対象: authy / sophos-fw (hacritical タグ、HA 登録済み) のみ
  monnie は ops タグのみで hacritical なし → assert で fail
- HA 登録確認: `pvesh get /cluster/ha/resources` で `vm:<vmid>` エントリを確認
- 移行先ノード決定: `/cluster/status` から「オンライン、かつ現在ノードではない」メンバーを選択
  (prefer* タグではなく動的に別ノードを選択: フェイルオーバーの目的は「別ノードへ逃がす」ため)
- HA 移行指示: `ha-manager crm-command relocate vm:{{ vmid }} {{ target_node }}`
  既存 `proxmox_restore_vm_placement.yml` Phase 3 と同じコマンドを流用 (policy §13)
- 移行完了待機: max 300s (HA 移行は VM リブートより時間がかかる)
- 失敗時の Slack 通知 status は `critical` (最終手段の失敗 = 即人間エスカレーション)

### ⑤ skip_notifications

3 healthcheck role の Slack notify タスクの `when:` 条件に
`not (skip_notifications | default(false) | bool)` を追加。
使用例: `ansible-playbook playbooks/radius_healthcheck.yml -e skip_notifications=true`
