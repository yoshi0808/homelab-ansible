## 概要

Slackメンション経由でhomelabサーバー群（authy/monnie）の調査・復旧を行う
パイプラインを実装した。当初v0.4として実装したアカウント構成・LLM実行基盤
（Claude Code + Python ACTIONS dict）を、実装後の振り返りでv0.5として
全面改訂した。改訂の経緯はpolicy §0/§14を参照。本ファイルは最終構成
（v0.5）の実装内容を記録する。

### LLM実行基盤

- Codex CLI採用（既存のProxmox patch advisor運用と同一の信頼パターン）
- `sandbox: workspace-write`、`writable_roots`はworkspace自身を明示指定
  （`[]`にすると§14.12の衝突が発生するため修正済み）
- `network_access: true`、`approval_policy: never`
- execpolicy `default_policy="deny"`、`homelab-investigate-*` /
  `homelab-recover-*` のみ許可
- 認証: ChatGPTサブスクリプション経由のOAuth（既存のpatch advisor運用と
  同一の認証方式を流用、API従量課金は発生しない）。`xoxb-`/`xapp-`
  トークン自体はVault管理、git管理下には説明コメントのみ存在することを
  確認済み
- AppArmor unprivileged user namespace制限への対応として、Codexの
  vendored bwrapバイナリのみを対象にした個別AppArmorプロファイルを適用

### wrapper構成

| wrapper | 対象 | 種別 | 備考 |
|---|---|---|---|
| `homelab-investigate-authy` | authy | 調査 | status / freeradius / sshd / journal-* / failed / disk / memory / load / network / ports / journal-system / radius-port / radius-config-check |
| `homelab-investigate-monnie` | monnie | 調査 | status / 4サービス個別 / journal-* / failed / disk / memory / load / network / ports / journal-system / prometheus-targets / grafana-health / loki-health / unpoller-config |
| `homelab-recover-authy` | authy | 復旧 | freeradius restart（引数なし、固定、1回のみ） |
| `homelab-recover-monnie` | monnie | 復旧 | grafana-server/prometheus/loki/unpoller 一括restart（引数なし、固定、1回のみ。個別指定はトレードオフとして非対応） |

調査系wrapperはパラメータを受け取るが、対象ホスト側dispatchスクリプトが
許可リスト照合する（policy §4.4）。復旧系wrapperは引数を受け取らない
（policy §4.3）。同一対象への復旧は1回のみ、再実行は行わない（policy §4.3.1）。

### 内部TLS

monnieのGrafana/Prometheus/Loki向けHTTPSエンドポイント確認のため、
既存の自前ECDSA中間CAをansy/quory/monnie/authy/pve1/pve2の証明書ストア
に配備（`home-tls-ca.crt`、`update-ca-certificates`済み）。証明書検証
スキップ（`-k`）は使用しない。アクセスは`https://monnie.internal:3000`
（証明書SANに合わせる）。

### Slack連携

- App名: Homelab。Bot Token Scopes: `app_mentions:read`, `chat:write`
  のみ。`incoming-webhook`スコープは不要（既存`slack_webhook_alerts`を
  流用するため未使用、設定からも削除済み）
- Socket Mode（App-Level Token: `connections:write`のみ）
- 認可: 特定Slack User IDのみに反応
- botを招待した任意のチャンネルからメンション可能（特定チャンネルに限定
  されない仕様。認可はチャンネルではなくUser IDで制御）

### 起動方法

- ansy: systemd登録済み、手動起動・手動停止（開発用）
- quory: systemd自動起動（本番稼働中）
- 同時稼働はしない運用（policy §7.1）
