## 対応内容

### 指摘: mute 中に lock が残り続ける

**原因:** atomic lock（mkdir）取得 → mute チェックで exit 0 した場合、cleanup が pipeline.sh の EXIT trap にあるため、pipeline を起動しないと lock が解放されなかった。

**修正:** lock 取得をすべての前提チェック（mute/flapping）の後に移動。

```
変更前の順序:
  1. validate
  2. lock 取得 (mkdir)   ← ここで取得
  3. mute チェック → exit 0 でロック残留
  4. flapping チェック
  5. pipeline 起動

変更後の順序:
  1. validate
  2. mute チェック       ← lock なしで exit できる
  3. flapping チェック   ← lock なしで exit できる
  4. lock 取得 (mkdir)   ← すべてのチェック通過後に取得
  5. pipeline 起動
```

mute/flapping で exit する場合はロックを取得していないため、残留しない。
pipeline 起動時のみロックが取得され、pipeline.sh の EXIT trap で確実に解放される。
