# 実装: Playbook分離 + CA鍵ファイルコピー方式

日付: 2026-06-05
担当: Claude (claude-sonnet-4-6)

## 概要

cert_renew.ymlをSemaphore実行用とsystemd timer実行用に分離し、
CA鍵の展開方式を環境変数+fix_pem.py方式からファイルコピー方式に変更した。

## 変更ファイル

### playbooks/cert_renew.yml（更新）
- quoryのsemaphore証明書発行・デプロイplayを削除
- monnie（Grafana）playを追加（issue taskのみ、deploy_grafanaは未実装）
- 通知サマリーのホストリストを `['ansy', 'quory', 'pve1', 'pve2']` → `['ansy', 'pve1', 'pve2', 'monnie']` に変更
- コメントにquory除外の理由を追記

### playbooks/cert_renew_quory.yml（新規）
- quoryのsemaphore証明書専用playbook
- systemd timerから実行する想定（semaphore再起動を含むためsemaphoreからは実行不可）
- 構成: verify → prepare_ca → issue → deploy_semaphore → cleanup
- notificationブロックなし（systemd timerのため）
- cleanupの失敗はfailタスクで即座に検出

### roles/homelab_cert_renew/tasks/prepare_ca.yml（更新）
- 環境変数チェック（homelab-ca-crt / homelab-ca-key）を削除
- fix_pem.pyコピー・実行タスクを削除
- `/home/yoshi/.cert/radius_ca.crt` → `/run/semaphore-ca/ca.crt` へのコピーに変更
- `/home/yoshi/.cert/radius_ca.key` → `/run/semaphore-ca/ca.key` へのコピーに変更
- remote_src: true を使用（quory上でローカルコピー）
- no_log: trueはCA秘密鍵コピータスクに維持

### roles/homelab_cert_renew/defaults/main.yml（更新）
- `cert_renew_ca_src_crt: /home/yoshi/.cert/radius_ca.crt` 追加
- `cert_renew_ca_src_key: /home/yoshi/.cert/radius_ca.key` 追加
- monnieのhosts_config追加（deploy_type: grafana, san_ip: []）

### roles/homelab_cert_renew/files/fix_pem.py（削除）
- 環境変数方式廃止に伴い不要になったため削除

### .gitignore（更新）
- `/home/yoshi/.cert` を追加（CA秘密鍵ディレクトリのGit管理除外）

## 設計判断

### prepare_ca.yml の remote_src: true
quory上で `connection: local` で実行するため、「リモート」ホストはquory自身。
`/home/yoshi/.cert/` はquory上に存在するため、remote_src: true でローカルコピーとして機能する。

### monnieの扱い
- `cert_renew_hosts_config` にmonnieを追加（deploy_type: grafana, san_ip: []）
- cert_renew.ymlではissueタスクのみ実行し、deploy_grafana.ymlは未実装
- 通知サマリーにmonnieを含めるが「renewed but deploy FAILED」が表示される予定
- deploy_grafanaの実装は別タスクで対応

### cert_renew_quory.ymlのcleanup
通知playがないため、cleanup失敗の検出はcleanup playの末尾にfailタスクを直接追加。
