## 確認内容

### 実行ホスト preflight

- 該当箇所:
  - `playbooks/cert_renew.yml:13`
  - `playbooks/cert_renew.yml:16`
  - `playbooks/cert_renew.yml:18`
  - `playbooks/cert_renew.yml:21`

Playbook 先頭に `hosts: localhost` / `connection: local` / `gather_facts: true` の preflight play が追加され、`ansible_hostname == 'quory'` を assert するようになっている。

これにより、誤って ansy など quory 以外から実行した場合は CA 秘密鍵展開前に停止する。要求仕様の「CA秘密鍵はansyに置かない（quoryのみ）」に対する実行時ガードとして妥当。

失敗メッセージにも「quory only」と「CA private key must never be placed」が含まれており、運用者にも理由が伝わる。

### quory local execution

- 該当箇所:
  - `playbooks/cert_renew.yml:27`
  - `playbooks/cert_renew.yml:31`
  - `playbooks/cert_renew.yml:55`
  - `playbooks/cert_renew.yml:59`
  - `playbooks/cert_renew.yml:78`
  - `playbooks/cert_renew.yml:95`
  - `playbooks/cert_renew.yml:107`

prepare / quory deploy は `connection: local` になっており、ansy / proxmox / quory / cleanup / notification で `cert_renew_ca_host: localhost` が明示されている。

quory から実行する前提では、CA 発行・生成物の `slurp`・cleanup は controller local、対象ホストへの配布は通常の SSH という整理になり、前回指摘した quory への SSH 自己接続は解消されている。

### cleanup failure message

- 該当箇所:
  - `playbooks/cert_renew.yml:103`
  - `playbooks/cert_renew.yml:107`
  - `playbooks/cert_renew.yml:154`

通知 play にも `cert_renew_ca_host: localhost` が追加されているため、cleanup 失敗後の fail message で `cert_renew_ca_host` が未定義になる前回 P3 は解消されている。

### 維持されている既存対策

- proxmox play の `serial: 1` が維持され、CA serial file 競合対策が残っている。
- `cert_cleanup_status` による `OK` / `FAILED` / `UNREACHABLE` 通知が維持されている。
- `cert_needs_renewal is not defined` による `check FAILED` 表示が維持されている。
- PVE 一時ファイル削除は `block` / `always` のまま維持されている。
