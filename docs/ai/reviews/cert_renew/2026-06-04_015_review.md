## Findings

### P1: ansy / proxmox 処理では CA ホストへの委譲がまだ `quory` のため、quory への SSH 自己接続が残る

- 該当箇所:
  - `playbooks/cert_renew.yml:24`
  - `playbooks/cert_renew.yml:57`
  - `roles/homelab_cert_renew/tasks/issue.yml:36`
  - `roles/homelab_cert_renew/tasks/issue.yml:44`
  - `roles/homelab_cert_renew/tasks/issue.yml:61`
  - `roles/homelab_cert_renew/tasks/issue.yml:72`
  - `roles/homelab_cert_renew/tasks/deploy_semaphore.yml:2`
  - `roles/homelab_cert_renew/tasks/deploy_proxmox.yml:4`

今回の実装メモでは「quory 上の処理はすべて `connection: local` で実行し SSH 経由の自己接続を避ける」とされている。

しかし、`cert_renew_ca_host: localhost` を上書きしているのは quory 自身の play と cleanup play だけで、ansy play と proxmox play では defaults の `cert_renew_ca_host: quory` がそのまま使われる。

そのため、以下の CA ホスト上で動くべき処理は、quory から実行していても `delegate_to: quory` となり、inventory の `quory.internal` へ SSH 自己接続する可能性がある。

- OpenSSL config 作成
- EC 秘密鍵生成
- CSR 生成
- CA 署名
- quory 上で生成済みの crt/key の `slurp`

これは今回の修正目的である「SSH 自己接続を避ける」に反しており、環境によっては ansy / pve1 / pve2 の証明書更新が失敗する。

推奨:

- quory からのみ実行する設計に寄せるなら、playbook 全体または ansy/proxmox play にも `cert_renew_ca_host: localhost` を設定する。
- その場合、CA 発行・slurp は controller local、対象ホストへの配布は通常の SSH 接続、という意図した形になる。
- 逆に ansy からの実行も残したいなら、`cert_renew_ca_host` と接続方式を実行元に応じて明示的に切り替える設計が必要。

例:

```yaml
- name: cert_renew - issue and deploy ansy cert
  hosts: ansy
  vars:
    cert_renew_ca_host: localhost
```

```yaml
- name: cert_renew - issue and deploy proxmox certs
  hosts: proxmox
  serial: 1
  vars:
    cert_renew_ca_host: localhost
```

## Notes

- prepare play は `hosts: quory` + `connection: local` になっており、CA 鍵の初期展開については SSH 自己接続を避ける形になっている。
- quory 自身の証明書更新 play は `connection: local` + `cert_renew_ca_host: localhost` になっており、quory の stat / issue / deploy / restart はローカル実行される。
- cleanup play も `cert_renew_ca_host: localhost` に上書きされており、quory から実行する前提では `/run/semaphore-ca/` をローカル削除する形になっている。
- proxmox play の `serial: 1` は維持されている。
- `ansible-playbook -i inventories/homelab/hosts.yml playbooks/cert_renew.yml --syntax-check` は、`ANSIBLE_LOCAL_TEMP=/tmp/ansible-local` と `ANSIBLE_REMOTE_TEMP=/tmp/ansible-remote` を指定して成功した。

## Overall

quory 自身の play 分離は方向性としてよいが、ansy / proxmox 向けの CA 発行・成果物取得に `delegate_to: quory` が残っているため、今回の目的はまだ完全には満たせていない。

マージ前に、quory からのみ実行する前提で `cert_renew_ca_host: localhost` を ansy / proxmox play にも適用するのがよい。
