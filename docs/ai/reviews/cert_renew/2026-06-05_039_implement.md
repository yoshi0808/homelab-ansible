## 変更内容の要約

### High 対応: deploy_grafana.yml に grafana.ini 設定タスクと TLS 検証を追加

レビュー指摘: 証明書を配置しても `grafana.ini` が新パスを参照していなければ、再起動後も旧証明書のままになる。

対応:

**1. grafana.ini TLS 設定タスクを cert deploy と restart の間に追加**

```yaml
- name: Configure Grafana TLS settings in grafana.ini
  ansible.builtin.lineinfile:
    path: /etc/grafana/grafana.ini
    regexp: "{{ item.regexp }}"
    line: "{{ item.line }}"
    insertafter: '^\[server\]'
  loop:
    - regexp: '^\s*;?\s*protocol\s*='
      line: "protocol = https"
    - regexp: '^\s*;?\s*cert_file\s*='
      line: "cert_file = {{ cert_host_config.cert_path }}"
    - regexp: '^\s*;?\s*cert_key\s*='
      line: "cert_key = {{ cert_host_config.key_path }}"
```

- `lineinfile` の `regexp` はコメントアウト状態（`; option = value`）と有効状態（`option = value`）の両方にマッチする
- コメントアウト行があれば置き換え、なければ `[server]` セクションの直後に挿入する（`insertafter`）
- `cert_host_config.cert_path` / `key_path` は `defaults/main.yml` の定義と同期するため、パス変更時に一か所の修正で反映される

**2. restart 後の TLS 検証タスクを追加**

```yaml
- name: Wait for Grafana to become available on port 3000
  ansible.builtin.wait_for:
    port: 3000
    host: localhost
    timeout: 30

- name: Verify Grafana is serving TLS
  ansible.builtin.shell:
    cmd: "openssl s_client -connect localhost:3000 < /dev/null"
  register: grafana_tls_check
  changed_when: false
  failed_when: grafana_tls_check.rc != 0
```

- `wait_for` で Grafana が port 3000 をバインドするまで最大 30 秒待機する
- `openssl s_client` で TLS ハンドシェイクが成功することを確認する（rc = 0 なら成功）
- `< /dev/null` で stdin を空にして openssl が自動終了するようにする
- これにより `cert_deployed: true` は TLS 検証通過後にのみ記録される

`deploy_grafana.yml` のタスク実行順序:

```
1. slurp crt / key from CA host
2. Ensure directory exists
3. Deploy certificate
4. Deploy private key
5. Configure grafana.ini TLS settings  ← NEW
6. Restart grafana-server
7. Wait for port 3000                   ← NEW
8. Verify Grafana TLS                   ← NEW
9. Record deploy success
```

## 注意点

- `openssl s_client` は TLS ハンドシェイク成功（rc = 0）を確認するが、証明書の CN / SAN が期待値と一致するかは確認しない。期待 CN の検証が必要な場合は `-verify_hostname monnie.internal` を追加する。
- `grafana.ini` の lineinfile は冪等だが、`; protocol = http` と `protocol = https` が両方存在するケースでは前者が `protocol = https` に置き換わり、後者はそのまま残る（重複行）。通常の Grafana 初期設定ではこのようなケースは起きない。
- `wait_for` の `timeout: 30` は Grafana 起動が遅延する場合に調整が必要。
