# 実装内容: homelab サーバー証明書自動更新

## 実装概要

`homelab_cert_renew` ロールと `playbooks/cert_renew.yml` を新規作成した。

4ホスト (ansy/quory/pve1/pve2) の TLS 証明書を EC secp384r1 で発行し、
quory 上の tmpfs (`/run/semaphore-ca/`) に CA 鍵を一時展開して処理後に削除する。

## 変更内容の要約

### playbook 構成 (cert_renew.yml)

5つのプレイで構成。

```
Play 1: hosts: localhost         → prepare_ca (CA 鍵を tmpfs に展開)
Play 2: hosts: semaphore_servers → issue + deploy_semaphore
Play 3: hosts: proxmox           → issue + deploy_proxmox
Play 4: hosts: localhost         → cleanup (CA 鍵を削除)
Play 5: hosts: localhost         → メール通知
```

Play 2/3 が失敗しても Play 4 の cleanup が実行される（Ansible のデフォルト動作）。

### role 構成

**prepare_ca.yml**
- `homelab-ca-crt` / `homelab-ca-key` 環境変数の存在確認
- `/run/semaphore-ca/certs/` ディレクトリを root:root mode 0700 で作成
- CA 証明書・鍵を書き込み（鍵は `no_log: true`）

**issue.yml** (各ホストに対して実行)
- リモートホスト上の証明書ファイルの存在確認と残日数計算
- `force_renew: false` (デフォルト) かつ残日数 > 15 の場合はスキップ
- 更新対象の場合: openssl-san.cnf.j2 をテンプレート展開し、EC 鍵生成・CSR 生成・CA 署名を `delegate_to: localhost` で実行
- `cert_renewed` ファクトを set_fact (通知 play で hostvars 経由参照)

**deploy_semaphore.yml**
- `/etc/semaphore/tls/` ディレクトリを作成
- crt/key をリモートにコピー (鍵は `no_log: true`)
- `notify: restart semaphore`

**deploy_proxmox.yml**
- crt/key を `/tmp/` に転送 (鍵は `no_log: true`)
- `openssl pkcs8 -topk8 -nocrypt` で PKCS8 変換
- `pvenode cert set /tmp/pve_cert.pem /tmp/pve_key_pkcs8.pem --force`
- `/tmp/` の一時ファイルを削除
- `notify: restart pveproxy`

**cleanup.yml**
- `/run/semaphore-ca/` を `state: absent` で削除

**openssl-san.cnf.j2**
- `[ req ]`, `[ dn ]`, `[ req_ext ]`, `[ v3_req ]`, `[ alt_names ]` セクション
- `san_dns` / `san_ip` リストから DNS.N / IP.N エントリを生成

### inventory 変更

`semaphore_servers` グループを追加 (ansy, quory)。  
`group_vars/semaphore_servers.yml` に `ansible_user: ann` を設定。

### .gitignore 変更

`*.crt`, `*.key`, `*.csr`, `*.pem` を追加し、誤コミットを防止。

## cert 残日数チェックの動作

| 状態 | 動作 |
|---|---|
| ファイルなし | renewal=true (初回実行) |
| 残日数 <= 15 | renewal=true |
| 残日数 > 15 かつ force_renew=false | skipped |
| force_renew=true | 常に renewal=true |

PVE の確認パス: `/etc/pve/local/pveproxy-ssl.pem`  
Semaphore の確認パス: `/etc/semaphore/tls/<host>.internal.crt`

## 未対応事項

- **Phase 2**: monnie (Grafana) は対象外（要件通り）
- **証明書残日数チェックの PVE 初回**: `/etc/pve/local/pveproxy-ssl.pem` は
  `pvenode cert set` 実行前は存在しない場合がある。その場合 `stat` が false を返し
  正しく renewal=true になる。
- **serial file**: `-CAcreateserial` を使用するため、quory 上に `ca.srl` が生成される。
  cleanup で `/run/semaphore-ca/` ごと削除されるため問題なし。

## 注意点

- CA 秘密鍵は quory の tmpfs のみに展開される。ansy には一切書き込まない。
- プレイ 1〜4 は `become: true`。`/run/` 配下への書き込みに root 権限が必要。
- handlers は各プレイに直接定義。role の handlers/main.yml は role as tasks_from で
  呼び出した場合に利用される（playbook の handlers が優先）。
- `no_log: true` を CA 鍵書き込み・EC 鍵生成・CSR 署名・鍵配布タスクに付与済み。
- Semaphore の変数グループ `CA_Info` に `homelab-ca-crt` / `homelab-ca-key` が
  SECRETS として設定されていること（実行前提条件）。
