## P1 対応詳細: CA署名の直列化

### 問題

Play 2 (`semaphore_servers`: ansy/quory) と Play 3 (`proxmox`: pve1/pve2) では、複数ホスト分の
`openssl x509 -CAcreateserial` が quory 上の同一 CA ディレクトリに対して並列実行される可能性があった。

`-CAcreateserial` は `/run/semaphore-ca/ca.srl` を読み書きするため、競合・重複・破損のリスクがあった。

### 対応

Play 2 と Play 3 に `serial: 1` を追加した。

```yaml
- name: cert_renew - issue and deploy semaphore certs
  hosts: semaphore_servers
  serial: 1
  ...

- name: cert_renew - issue and deploy proxmox certs
  hosts: proxmox
  serial: 1
  ...
```

`serial: 1` により、Ansible は 1 ホストずつ順番に処理する。

実行順:
1. Play 1: prepare CA (quory)
2. Play 2: ansy (issue → deploy) 完了後 → quory (issue → deploy)
3. Play 3: pve1 (issue → deploy) 完了後 → pve2 (issue → deploy)
4. Play 4: cleanup (quory)
5. Play 5: notification (localhost)

CA 署名処理は常に 1 プロセスのみが quory 上で走るため、`.srl` 競合は発生しない。

## P2 対応詳細: サービス再起動を明示実行・成功後に `cert_deployed` を設定

### 問題

従来の実装では:
1. cert/key を配置
2. `cert_deployed: true` を設定 ← ここで cert_deployed が true に
3. handler が後から restart を実行 ← 失敗しても cert_deployed は true のまま

handler の再起動失敗が通知に反映されず、`renewed and deployed` と誤表示される可能性があった。

### 対応

handler を廃止し、deploy タスクファイル内で restart を明示実行。
`cert_deployed: true` を restart の後に移動した。

**deploy_semaphore.yml**:

```yaml
- name: Deploy certificate
  ansible.builtin.copy: ...
  when: cert_needs_renewal

- name: Deploy private key
  ansible.builtin.copy: ...
  when: cert_needs_renewal
  no_log: true

- name: Restart semaphore            # ← explicit restart
  ansible.builtin.systemd:
    name: semaphore
    state: restarted
  when: cert_needs_renewal

- name: Record deploy success        # ← restart 成功後にのみ設定
  ansible.builtin.set_fact:
    cert_deployed: true
  when: cert_needs_renewal
```

**deploy_proxmox.yml** (block 内):

```yaml
- name: Apply certificate with pvenode
  ansible.builtin.command:
    cmd: pvenode cert set ...
  when: cert_needs_renewal
  changed_when: true

- name: Restart pveproxy             # ← explicit restart
  ansible.builtin.systemd:
    name: pveproxy
    state: restarted
  when: cert_needs_renewal

- name: Record deploy success        # ← restart 成功後にのみ設定
  ansible.builtin.set_fact:
    cert_deployed: true
  when: cert_needs_renewal
```

`cert_deployed: true` は deploy + restart の両方が成功した場合のみ設定される。
どちらかが失敗すると未設定のまま残り、通知では `renewed but deploy FAILED` が表示される。

### handlers 削除

`notify:` が不要になったため、cert_renew.yml の Play 2/3 から `handlers:` セクションを削除した。
`roles/homelab_cert_renew/handlers/main.yml` は main.yml 経由で呼ぶ場合の互換として残す。
