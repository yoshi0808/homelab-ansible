## 対応内容

### High: cert_renew_quory.yml のgit追跡

**指摘**: `cert_renew_quory.yml` が未追跡のため、追跡済み差分だけcommitすると quory の証明書更新経路がリポジトリに入らない。

**対応**: `git add` でstaged状態にした。

```
git add playbooks/cert_renew_quory.yml
```

合わせて、今回の実装で作成された review/implement ドキュメントも追跡済みにした。

```
git add docs/ai/reviews/cert_renew/2026-06-05_024_implement.md
git add docs/ai/reviews/cert_renew/2026-06-05_025_review.md
git add docs/ai/reviews/cert_renew/2026-06-05_026_implement.md
```

`git status --short` で `A` (Added) になっていることを確認済み。

### Medium: prepare_ca.yml にソースファイルのstatと権限チェックを追加

**指摘**: CA秘密鍵のソースを `/home/yoshi/.cert/` からコピーする方式に変わっているが、
ソースファイルの存在確認や権限チェックがなかった。

**対応**: `roles/homelab_cert_renew/tasks/prepare_ca.yml` に以下のチェックを追加した。

1. `stat` でCAcert・CAkeyのソースファイル存在確認
2. ソースファイルが存在しない場合は `fail`
3. CA秘密鍵のmode が `0600` 以外の場合は `fail`

```yaml
- name: Fail if CA private key permissions are too open
  ansible.builtin.fail:
    msg: "CA private key {{ cert_renew_ca_src_key }} must be mode 0600, got {{ ca_key_stat.stat.mode }}"
  when: ca_key_stat.stat.mode != '0600'
  no_log: true
```

stat / 存在確認 / 権限チェックのタスクにはすべて `no_log: true` を付けた。

**CA秘密鍵管理の運用前提（コードに反映できない部分）**:
- 保管場所: `/home/yoshi/.cert/radius_ca.key` (quory上)
- 権限: `chmod 600`、所有者: `yoshi`
- バックアップ: quory の OS 再インストール時に再配置が必要
- Git管理: しない（.gitignore で `.cert/` を除外）
- 権限が0600でない場合はPlaybookがfailするため、誤設定を早期検知できる

### Low: .gitignoreのCA entryを修正

**指摘**: `.gitignore` の `/home/yoshi/.cert` はリポジトリルート基準のパターンであり、
実際のホームディレクトリの保護としては機能しない。

**対応**: `.gitignore` のエントリを以下に変更した。

変更前:
```gitignore
# CA source files (contains private key - never commit)
/home/yoshi/.cert
```

変更後:
```gitignore
# CA source files (誤ってリポジトリ内に配置された場合の保護)
# 実際の /home/yoshi/.cert はリポジトリ外であり、ファイル権限(0600)で保護する
.cert/
```

- パターンを `.cert/` に変更（リポジトリ内に誤って作成された場合に除外される）
- コメントを「誤配置の保護」として意図を明確化
- 実際のホーム配下の保護はファイル権限（0600）で担保する旨を明記
