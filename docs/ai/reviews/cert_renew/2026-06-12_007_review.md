## Findings

### Medium: 訂正後のチェーン検証コマンドでも中間CAとルートCAを正しく指定できていない

- File: `docs/ai/reviews/cert_renew/2026-06-12_006_implement.md`
- Lines: 36-98

追加実装メモは、リーフ証明書と中間CA証明書を連結した fullchain ファイルを次のように直接検証できるとしている。

```text
openssl verify -CAfile home_radius_ca.crt <fullchain-path>
```

しかし、`openssl verify` で検証対象として渡したファイル内の後続証明書が、自動的に未信頼の中間CAチェーンとして使われることは保証されない。中間CA証明書は `-untrusted` で明示し、検証対象にはリーフ証明書を指定する必要がある。

配布済み fullchain を使う場合は、一時ファイルへリーフ証明書と中間CA証明書を分離した上で、次の形で検証するのが確実である。

```text
openssl verify \
  -CAfile home_radius_ca.crt \
  -untrusted home_tls_ca.crt \
  leaf.crt
```

また、`s_client` の各コマンドには `-CAfile home_radius_ca.crt` がない。Home-RADIUS-CA が quory のシステム trust store に登録済みとは限らないため、サーバーが正しいリーフ証明書と中間CA証明書を提示していても、通常は `Verify return code: 0 (ok)` にならない。

各 `s_client` コマンドにもルートCAを明示する必要がある。

```text
openssl s_client \
  -connect <host>:<port> \
  -servername <host> \
  -showcerts \
  -CAfile home_radius_ca.crt
```

この修正後に、次の3点を運用者確認項目とするのが適切である。

- 提示された cert 0 の issuer が Home-TLS-CA
- 提示された cert 1 の subject が Home-TLS-CA
- 明示した Home-RADIUS-CA に対して `Verify return code: 0 (ok)`
