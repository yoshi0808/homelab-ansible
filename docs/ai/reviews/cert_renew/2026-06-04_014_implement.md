# 実装内容: quory local connection 対応

## 問題

`cert_renew.yml` は quory からのみ実行する前提だが、`hosts: semaphore_servers` に quory 自身が含まれており、quory の証明書配布が SSH 経由 (quory → quory.internal) になっていた。SSH 自己接続は環境によって失敗する。

prepare CA play (`hosts: quory`) も同様に SSH 経由だった。

## 修正方針

- `cert_renew.yml` を quory からのみ実行するplaybookとして設計し直す
- quory 上の処理はすべて `connection: local` で実行し SSH 経由の自己接続を避ける
- semaphore_servers play を ansy (SSH) と quory (local) に分離する

## 変更ファイル

| ファイル | 変更内容 |
|---|---|
| `playbooks/cert_renew.yml` | prepare play に `connection: local` 追加、semaphore_servers play を分離、cleanup play に `vars: cert_renew_ca_host: localhost` 追加 |

ロールタスクファイルに変更なし。`delegate_to: cert_renew_ca_host` が play vars で上書きされることで対応。

---

## 変更詳細

### Play 1: prepare CA

```yaml
# Before
- name: cert_renew - prepare CA key on quory
  hosts: quory
  become: true

# After
- name: cert_renew - prepare CA key on quory
  hosts: quory
  become: true
  connection: local      # ← 追加
```

`connection: local` により `/run/semaphore-ca/` 作成と CA 鍵書き込みがローカル実行になる。Semaphore の環境変数 (`homelab-ca-crt`, `homelab-ca-key`) はローカルプロセスから直接 `lookup('env', ...)` で参照できる。

### Play 2: semaphore_servers を分離

**旧 Play（削除）:**
```yaml
- name: cert_renew - issue and deploy semaphore certs
  hosts: semaphore_servers
  serial: 1
```

**新 Play 2a — ansy (SSH):**
```yaml
- name: cert_renew - issue and deploy ansy cert
  hosts: ansy
  become: true
```

ansy は quory から SSH 経由で接続するため従来通り。

**新 Play 2b — quory (local):**
```yaml
- name: cert_renew - issue and deploy quory cert (local)
  hosts: quory
  become: true
  connection: local
  vars:
    cert_renew_ca_host: localhost
```

`connection: local` により issue/deploy タスクがローカル実行になる。

`vars: cert_renew_ca_host: localhost` により `issue.yml` と `deploy_semaphore.yml` 内の `delegate_to: "{{ cert_renew_ca_host }}"` が `delegate_to: localhost` に置き換わる。`localhost` は inventory で `ansible_connection: local` が設定されているため SSH 接続なしでローカル実行される。

**タスクの実行場所（quory play）:**

| タスク | ファイル | delegate | 実行場所 |
|---|---|---|---|
| stat (cert check) | issue.yml | なし | quory (local) |
| shell (days check) | issue.yml | なし | quory (local) |
| template (openssl.cnf) | issue.yml | localhost | quory (local) |
| command (ecparam) | issue.yml | localhost | quory (local) |
| command (openssl req) | issue.yml | localhost | quory (local) |
| command (openssl x509) | issue.yml | localhost | quory (local) |
| slurp (crt) | deploy_semaphore.yml | localhost | quory (local) |
| slurp (key) | deploy_semaphore.yml | localhost | quory (local) |
| file (mkdir) | deploy_semaphore.yml | なし | quory (local) |
| copy (crt) | deploy_semaphore.yml | なし | quory (local) |
| copy (key) | deploy_semaphore.yml | なし | quory (local) |
| systemd restart | deploy_semaphore.yml | なし | quory (local) |

すべてローカル実行。SSH 自己接続なし。

### Play 4: cleanup

```yaml
# Before
- name: cert_renew - cleanup CA key from quory
  hosts: localhost
  become: false

# After
- name: cert_renew - cleanup CA key from quory
  hosts: localhost
  become: false
  vars:
    cert_renew_ca_host: localhost    # ← 追加
```

cleanup.yml 内の `delegate_to: "{{ cert_renew_ca_host }}"` が `delegate_to: localhost` になる。`localhost` は `ansible_connection: local` のためローカル実行。SSH 自己接続なし。

### serial: 1 の扱い

旧 semaphore_servers play の `serial: 1` は ansy/quory を1ホストずつ処理するためのものだった。分離後は各 play が単一ホストのため `serial: 1` は不要（削除）。

proxmox play (pve1/pve2) は引き続き `serial: 1` を維持（CA serial file 競合防止）。

## 実行フロー（修正後）

```
Play 1: hosts: quory, connection: local
  → CA鍵・証明書を /run/semaphore-ca/ にローカル展開

Play 2: hosts: ansy
  → ansyの証明書発行(delegate→localhost)・SSH配布

Play 3: hosts: quory, connection: local, cert_renew_ca_host: localhost
  → quoryの証明書発行(delegate→localhost)・ローカル配布

Play 4: hosts: proxmox, serial: 1
  → pve1, pve2 の証明書発行・SSH配布（順次）

Play 5: hosts: localhost, cert_renew_ca_host: localhost
  → /run/semaphore-ca/ をローカル削除

Play 6: hosts: localhost
  → メール通知、cleanup失敗時にfail
```
