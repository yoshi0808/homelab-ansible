## 対応内容

### Medium: cert_renew_policy.md が要求仕様・共通プロンプトから参照されていない問題

**指摘**: `cert_renew_policy.md` を運用正本としているが、既存の `001_requirement.md` は
Semaphore Secrets方式のままで、`cert_renew_policy.md` への参照がない。
次回以降のAI/作業者が `001_requirement.md` を正として読み誤る可能性がある。

**対応**: `docs/ai/reviews/cert_renew/2026-06-03_001_requirement.md` の冒頭に注記を追加した。

```markdown
> **注意**: この要求仕様はPhase 1初回実装時点の内容です。
> CA鍵取得方式・Playbook構成・運用方針の現在の正本は以下を参照してください。
> `docs/ai/prompts/cert_renew_policy.md`
```

`core.md` は全体共通のプロンプトファイルであるため、cert_renew固有の参照は追加しない。
個別のPlaybook/Roleの正本ポインタは要求仕様ファイル側に持たせる。

### Low: 029_review.md / 030_implement.md が未追跡

**指摘**: 今回の対応根拠である `029_review.md` と `030_implement.md` が `??` のまま。

**対応**: `git add` で追跡対象に追加した（`001_requirement.md` と `cert_renew_policy.md` の変更も同時にstage）。

### Low: cert_renew_policy.md の失敗検知説明が実装と不一致

**指摘**: policy では「完了メール（失敗時も `ignore_errors` なしでfail）」と書いているが、
実際の `Send completion notification` タスクには `ignore_errors: true` が付いており、
SMTP障害時もPlaybookが失敗する、と読み違える可能性がある。

**対応**: `docs/ai/prompts/cert_renew_policy.md` の「6. 失敗検知」表を修正した。

変更前:
```
| cert_renew.yml | 完了メール（失敗時も ignore_errors なしでfail） |
```

変更後:
```
| cert_renew.yml | 完了メールを送る（メール送信失敗は ignore_errors: true で無視）。cleanup失敗などPlaybook本体の失敗は fail タスクで検知する。 |
```
