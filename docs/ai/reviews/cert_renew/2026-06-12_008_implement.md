## 指摘対応の詳細

### [Medium] チェーン検証コマンドの再訂正

**指摘内容**:
- `openssl verify -CAfile root_ca.crt <fullchain-path>` の形式では、ファイル内の後続証明書が中間CAとして自動使用されることは保証されない。中間CAは `-untrusted` で明示する必要がある。
- `s_client` コマンドに `-CAfile` がなく、システムトラストストアに Home-RADIUS-CA が登録されていない環境では `Verify return code: 0 (ok)` が得られない。

**再訂正後の確認手順（運用者確認待ち）**:

#### 前提

| ファイル | 場所 | 備考 |
|---|---|---|
| ルートCA証明書 | `<home_radius_ca.crt>`（任意のパス） | 秘密鍵はオフライン。証明書のみ quory 上に配置しておくこと |
| 中間CA証明書 | `/home/yoshi/.cert/ca/home_tls_ca.crt` | quory 上に既存 |

ルートCA証明書（Home-RADIUS-CA の `.crt`）は秘密鍵とは別物であり、検証にのみ必要。
適切なパスに配置していない場合は事前に配置すること（例: `/home/yoshi/.cert/ca/home_radius_ca.crt`）。

---

#### (1) チェーン検証

配布済み fullchain 証明書からリーフ証明書を取り出し、中間CAを `-untrusted` で明示して検証する。
`openssl x509` は PEM ファイルの先頭証明書のみを読み込むため、そのままリーフ抽出に使える。

```sh
# ルートCA証明書のパス（事前に配置しておくこと）
ROOT_CA=/home/yoshi/.cert/ca/home_radius_ca.crt
INT_CA=/home/yoshi/.cert/ca/home_tls_ca.crt

# ansy の Semaphore 証明書（ansy 上で実行）
openssl x509 -in /etc/semaphore/tls/ansy.internal.crt -out /tmp/leaf_ansy.crt
openssl verify -CAfile "$ROOT_CA" -untrusted "$INT_CA" /tmp/leaf_ansy.crt
rm -f /tmp/leaf_ansy.crt

# quory の Semaphore 証明書（quory 上で実行）
openssl x509 -in /etc/semaphore/tls/quory.internal.crt -out /tmp/leaf_quory.crt
openssl verify -CAfile "$ROOT_CA" -untrusted "$INT_CA" /tmp/leaf_quory.crt
rm -f /tmp/leaf_quory.crt

# monnie の Grafana 証明書（monnie 上で実行）
openssl x509 -in /etc/grafana/certs/monnie.crt -out /tmp/leaf_monnie.crt
openssl verify -CAfile "$ROOT_CA" -untrusted "$INT_CA" /tmp/leaf_monnie.crt
rm -f /tmp/leaf_monnie.crt
```

期待結果（各コマンド）: `<path>: OK`

pve1 / pve2 は pveproxy の証明書ファイルの直接アクセスが困難なため、s_client で代替確認（下記 (2)）。

---

#### (2) 各サービスのフルチェーン提示確認（s_client）

`-CAfile` でルートCA証明書を明示することで、`Verify return code: 0 (ok)` が正確に判定される。

```sh
ROOT_CA=/home/yoshi/.cert/ca/home_radius_ca.crt

# Semaphore: ansy（quory から実行）
echo | openssl s_client \
  -connect ansy.internal:3000 \
  -servername ansy.internal \
  -showcerts \
  -CAfile "$ROOT_CA" \
  2>&1 | grep -E "( s:| i:|Verify return code)"

# Semaphore: quory（quory 上で実行）
echo | openssl s_client \
  -connect quory.internal:3000 \
  -servername quory.internal \
  -showcerts \
  -CAfile "$ROOT_CA" \
  2>&1 | grep -E "( s:| i:|Verify return code)"

# Grafana: monnie（quory から実行）
echo | openssl s_client \
  -connect monnie.internal:3000 \
  -servername monnie.internal \
  -showcerts \
  -CAfile "$ROOT_CA" \
  2>&1 | grep -E "( s:| i:|Verify return code)"

# pveproxy: pve1（quory から実行）
echo | openssl s_client \
  -connect pve1.internal:8006 \
  -servername pve1.internal \
  -showcerts \
  -CAfile "$ROOT_CA" \
  2>&1 | grep -E "( s:| i:|Verify return code)"

# pveproxy: pve2（quory から実行）
echo | openssl s_client \
  -connect pve2.internal:8006 \
  -servername pve2.internal \
  -showcerts \
  -CAfile "$ROOT_CA" \
  2>&1 | grep -E "( s:| i:|Verify return code)"
```

期待結果の形式（例: quory）:

```text
 0 s:CN = quory.internal
   i:CN = Home-TLS-CA
 1 s:CN = Home-TLS-CA
   i:CN = Home-RADIUS-CA
Verify return code: 0 (ok)
```

確認ポイント:
- cert 0（depth=0）の issuer が `Home-TLS-CA` であること（中間CA署名）
- cert 1（depth=1）の subject が `Home-TLS-CA` であること（中間CA証明書が提示されている）
- `-CAfile` で指定した Home-RADIUS-CA に対して `Verify return code: 0 (ok)` であること
