# implement: cert_renew_quory.yml に force_renew: true を追加

## 実装概要

`cert_renew_quory.yml` は systemd timer から引数なしで実行される。
role のデフォルト値 `force_renew: false` のままでは有効期限が十分残っている場合に証明書が更新されない。
playbook の vars に `force_renew: true` を追加し、毎回証明書を発行するよう変更した。

## 作成・更新したファイル

| ファイル | 変更種別 |
|---|---|
| `playbooks/cert_renew_quory.yml` | 更新 |

## 変更内容の要約

`cert_renew_quory - issue and deploy quory cert` play の `vars` ブロックに `force_renew: true` を追加。

```diff
   vars:
     cert_renew_ca_host: localhost
+    force_renew: true
```

role の `defaults/main.yml` は `force_renew: false` のままとし、role 単体を他 playbook から使う場合はデフォルトで条件チェックが働く。
このファイルのみを対象とした上書きのため、他 playbook への影響はない。

## 注意点

- `force_renew: true` は play レベルの vars に設定したため、role defaults より優先される。
- 他の cert_renew 系 playbook（cert_renew_grafana.yml 等）には影響しない。
- systemd timer から引数なしで実行されるこの playbook に限定した変更である。
