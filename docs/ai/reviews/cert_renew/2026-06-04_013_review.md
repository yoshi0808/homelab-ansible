## 確認内容

### cleanup unreachable 対応

- 該当箇所:
  - `roles/homelab_cert_renew/tasks/cleanup.yml:2`
  - `roles/homelab_cert_renew/tasks/cleanup.yml:10`
  - `roles/homelab_cert_renew/tasks/cleanup.yml:12`
  - `roles/homelab_cert_renew/tasks/cleanup.yml:17`
  - `playbooks/cert_renew.yml:93`
  - `playbooks/cert_renew.yml:112`

前回レビューで指摘した、cleanup の delegated unreachable が通知へ進まない可能性については、`ignore_unreachable: true` が追加されている。

また、cleanup 結果は boolean ではなく `cert_cleanup_status` として扱われるようになり、通知上も以下の3状態を区別できる。

- `OK`
- `FAILED`
- `UNREACHABLE`

通知送信後に `cert_cleanup_status is defined` で `fail` するため、Semaphore / CLI 上の実行結果も失敗として残る。

### 通知のホスト状態

- 該当箇所:
  - `playbooks/cert_renew.yml:75`
  - `playbooks/cert_renew.yml:76`
  - `playbooks/cert_renew.yml:82`
  - `playbooks/cert_renew.yml:84`
  - `playbooks/cert_renew.yml:86`

`cert_needs_renewal is not defined` が先に判定されるため、remote check 前に失敗したホストを `skipped` と誤表示する問題は解消されている。

状態表示も以下に分離されており、月次自動実行後のメール確認に必要な粒度は確保されている。

- `check FAILED`
- `skipped`
- `issue FAILED`
- `renewed and deployed`
- `renewed but deploy FAILED`

### 既存の改善点

- Play 2 / Play 3 に `serial: 1` があり、quory 上の `-CAcreateserial` 競合リスクは抑えられている。
- CA 生成・署名処理は `cert_renew_ca_host: quory` へ委譲され、CA秘密鍵を ansy に置かない方針に沿っている。
- PVE の一時ファイル削除は `block` / `always` で維持されており、通常の失敗経路でも `/tmp/pve_*_{{ inventory_hostname }}.pem` を削除する構造になっている。
- `no_log: true` は CA 秘密鍵・ホスト秘密鍵を扱う主要タスクに付いている。
