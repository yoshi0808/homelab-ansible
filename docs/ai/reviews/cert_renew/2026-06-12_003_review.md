# Review: cert_renew role Home-TLS-CA 対応

Review date: 2026-06-12

## Summary

CA参照パスの変更、リーフ証明書と中間CA証明書の連結、中間CA残存日数の警告、Slack通知への反映は、要求仕様に沿って実装されている。

また、`cert_renew.yml` と `cert_renew_quory.yml` はともに Ansible の構文チェックを通過した。

ただし、要求仕様の確認項目であるチェーン検証と各サービスのフルチェーン提示確認は、実施結果が記録されていない。さらに、ansy 除外挙動に対する実装メモの回答は、要求文と現在の Playbook 構成を正しく対応付けられていない。

このため、実機確認と文書修正を行ってから確定するのが妥当である。

## Findings

### Medium: チェーン検証と各サービスのフルチェーン提示確認が未実施

- File: `docs/ai/reviews/cert_renew/2026-06-12_001_requirement.md`
- Lines: 51-53
- File: `docs/ai/reviews/cert_renew/2026-06-12_002_implement.md`
- Lines: 111-129

要求仕様は、発行証明書のチェーン検証と、Semaphore / Grafana / pveproxy がフルチェーンを提示することの確認を求めている。

実装メモには実行予定のコマンドだけが記載されており、実行結果はない。また、記載された Grafana 確認コマンドは `s_client` の出力を `openssl x509` に渡して先頭の証明書の subject / issuer を表示するだけなので、サーバーが中間CA証明書まで提示したことを確認できない。

少なくとも以下を実機で確認し、結果を追加実装メモに記録する必要がある。

- 発行した各リーフ証明書について、ルートCAと中間CAを用いた `openssl verify` が成功すること
- 各サービスに対する `openssl s_client -showcerts` の出力に、リーフ証明書と Home-TLS-CA 証明書の両方が含まれること
- `Verify return code: 0 (ok)` になること

実機確認ができない場合は、「未対応事項なし」ではなく、運用者による確認待ちとして明記する必要がある。

### Medium: ansy 除外挙動の確認回答が要求項目と一致していない

- File: `docs/ai/reviews/cert_renew/2026-06-12_001_requirement.md`
- Lines: 49-50
- File: `docs/ai/reviews/cert_renew/2026-06-12_002_implement.md`
- Lines: 101-109
- File: `playbooks/cert_renew.yml`
- Lines: 13-53
- File: `playbooks/cert_renew_quory.yml`
- Lines: 12-53

要求仕様は「ansy実行時に ansy 自身の証明書配布が除外される既存挙動」の特定を求めているが、実装メモは `cert_renew_quory.yml` が ansy を対象にしないことを回答としている。

現在の実装では、両 Playbook とも冒頭で実行ホストが quory であることを assert している。さらに `cert_renew.yml` は `hosts: ansy` の play で ansy 証明書を明示的に更新する。したがって、現行コードからは要求文にある「ansy実行時の ansy 自身の除外」という挙動を確認できない。

要求仕様が意図している挙動を再確認し、次のいずれかに揃える必要がある。

- 意図が「`cert_renew_quory.yml` は quory のみを更新し、ansy を含めない」であれば、requirement の表現を修正する
- ansy 上から実行する別の既存運用を指しているのであれば、その Playbook / 条件分岐を特定し、改修後も維持されていることを確認する

### Low: CAを ansy に配置するという文書記載と実行制約が不整合

- File: `docs/ai/prompts/cert_renew_policy.md`
- Lines: 61-88, 200-204
- File: `playbooks/cert_renew.yml`
- Lines: 13-36

Policy は中間CA証明書・秘密鍵を quory / ansy に配置するとしている一方、保管場所の表と復旧手順は主に quory を対象としており、Playbook 自体も quory 以外での実行を禁止している。

現行実装だけを見ると ansy 上の中間CA秘密鍵は使用されない。秘密鍵の不要な複製を避けるためにも、ansy 配置が必要な別運用があるのかを明記し、不要なら requirement / Policy から ansy 配置の記述を削除した方がよい。
