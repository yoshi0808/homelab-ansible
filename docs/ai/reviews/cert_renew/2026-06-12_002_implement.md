# 実装内容: cert_renew role の Home-TLS-CA(中間CA)対応

作成日: 2026-06-12

---

## 実装概要

cert_renew role の署名CA参照を、ルートCA(Home-RADIUS-CA)直接署名から
中間CA(Home-TLS-CA)署名に移行した。あわせてフルチェーン配布と中間CA有効期限監視を追加した。

## 変更内容の要約

### 1. CA参照パスの変更（defaults/main.yml）

```yaml
# 変更前
cert_renew_ca_src_crt: /home/yoshi/.cert/radius_ca.crt
cert_renew_ca_src_key: /home/yoshi/.cert/radius_ca.key

# 変更後
cert_renew_ca_src_crt: /home/yoshi/.cert/ca/home_tls_ca.crt
cert_renew_ca_src_key: /home/yoshi/.cert/ca/home_tls_ca.key
cert_renew_intermediate_ca_warn_days: 90
```

tmpfs への一時展開・cleanup・mode 0600 検証の既存方式は変更なし。

### 2. 中間CA有効期限監視（prepare_ca.yml）

CA源ファイルの stat 確認直後（tmpfs コピー前）に有効期限チェックタスクを追加。

- openssl で残存日数を計算
- 90日未満の場合: debug warning 出力 + `cert_intermediate_ca_warn: true` + `cert_intermediate_ca_days_left` をホストファクトとして設定
- fail はしない（警告のみ）

### 3. フルチェーン生成（issue.yml）

署名タスク（`openssl x509 -req`）の直後に以下の2タスクを追加。

```
リーフ証明書 + 中間CA証明書 → fullchain.crt
```

- `ansible.builtin.shell` で `cat leaf.crt ca.crt > fullchain.crt` を実行
- `delegate_to: cert_renew_ca_host` で quory（tmpfs）上で生成
- パーミッション `0600` を設定

### 4. フルチェーン配布（deploy_*.yml 3ファイル）

- `deploy_semaphore.yml` / `deploy_proxmox.yml` / `deploy_grafana.yml` の slurp タスクを `.crt` から `.fullchain.crt` に変更
- slurp 変数名 `slurp_crt` は維持（配布先へのコピー・pvenode への引き渡しロジックは変更なし）

### 5. 通知への中間CA警告追加（cert_renew.yml / cert_renew_quory.yml）

cert_renew_summary の末尾に条件付きブロックを追加。

```jinja2
{% if hostvars['quory'].cert_intermediate_ca_warn | default(false) | bool %}
WARNING: Intermediate CA expires in {{ hostvars['quory'].cert_intermediate_ca_days_left }} days!
{% endif %}
```

Slack チャンネル・ステータスの選択ロジックも更新。
- WARNING を含む場合: `alerts` チャンネル、`warning` ステータス

### 6. cert_renew_policy.md v2.0 改版

- 変更履歴セクション追加

## 確認項目への回答

### ansy実行時に ansy 自身の証明書配布が除外される既存挙動

cert_renew_quory.yml の対象は `hosts: quory` のみであり、ansy は含まれていない。
これが「cert_renew_quory.yml 実行時に ansy の証明書配布が除外される」挙動の実体である。

cert_renew.yml では ansy は `hosts: ansy` の play で明示的に対象になっているが、
cert_renew_quory.yml にはそのような play が存在しない（意図的な分離）。

改修後もこの構造は変えていない。cert_renew_quory.yml に ansy を追加していない。✓

### チェーン検証確認コマンド（運用者による手動確認）

発行後に以下で検証する。

```sh
# チェーン検証（ルートCA証明書が必要）
openssl verify -CAfile <root_ca.crt> -untrusted /home/yoshi/.cert/ca/home_tls_ca.crt \
  /path/to/leaf.crt

# フルチェーン提示確認（Grafana例）
echo | openssl s_client -connect monnie.internal:3000 -servername monnie.internal 2>/dev/null \
  | openssl x509 -noout -subject -issuer
```

## 注意点

- CA配置の前提: quory の `/home/yoshi/.cert/ca/home_tls_ca.{crt,key}` は手動配置が必要（本role管轄外）
- cert_renew_quory.yml が ansy から実行された場合は冒頭の assert で fail する（既存挙動）
- Proxmox の pvenode cert set にフルチェーン PEM を渡すことは正常に動作する（leaf + intermediate の PEM 連結を受け付ける）
