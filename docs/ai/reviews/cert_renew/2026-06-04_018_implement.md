## P1 対応詳細: preflight play の追加

### 問題

`connection: local` と `cert_renew_ca_host: localhost` はコントローラーのローカルを指す設計のため、ansy から実行した場合、CA 秘密鍵が ansy の `/run/semaphore-ca/ca.key` に展開される。要求仕様の「CA秘密鍵はansyに置かない（quoryのみ）」に違反するが、コメントによる運用注意だけでは不十分。

### 対応

Playbook の先頭に `verify execution host` play を追加した。

```yaml
- name: cert_renew - verify execution host
  hosts: localhost
  gather_facts: true
  connection: local
  tasks:
    - name: Ensure this playbook is running on quory
      ansible.builtin.assert:
        that:
          - ansible_hostname == 'quory'
        fail_msg: >-
          cert_renew.yml must be run on quory only
          (current host: {{ ansible_hostname }}).
          CA private key must never be placed on {{ ansible_hostname }}.
```

`gather_facts: true` で `ansible_hostname` を取得し、`quory` でなければ即座に失敗する。

- quory から実行: `ansible_hostname == 'quory'` → assert 成功 → 処理継続
- ansy から実行: `ansible_hostname == 'ansy'` → assert 失敗 → Playbook 停止
- その他ホストから実行: 同様に即座に停止

失敗メッセージに「CA秘密鍵を置いてはならない」理由を明記することで、誰が見ても実行制約の意図が分かる。

### playbook 全体のフロー（変更後）

```
Play 0: verify execution host (localhost, assert)
Play 1: prepare CA key on quory (quory, local)
Play 2: issue and deploy ansy cert (ansy, SSH)
Play 3: issue and deploy quory cert (quory, local)
Play 4: issue and deploy proxmox certs (proxmox, SSH, serial: 1)
Play 5: cleanup CA key (localhost, delegate→localhost)
Play 6: notification + fail on cleanup error (localhost)
```

## P3 対応詳細: 通知 play の `cert_renew_ca_host` 定義追加

### 問題

cleanup play には `vars: cert_renew_ca_host: localhost` があるが、play 変数は次 play に引き継がれない。通知 play の fail タスク:

```yaml
msg: "CA cleanup {{ cert_cleanup_status }}: /run/semaphore-ca/ may still exist on {{ cert_renew_ca_host }}"
```

`cert_renew_ca_host` が未定義の場合、`AnsibleUndefinedVariable` により fail メッセージ自体が壊れ、Semaphore / CLI 上での失敗理由が読みにくくなる。

### 対応

通知 play に `vars: cert_renew_ca_host: localhost` を追加した。

```yaml
- name: cert_renew - send completion notification
  hosts: localhost
  gather_facts: false
  become: false
  vars:
    cert_renew_ca_host: localhost    # ← 追加
  tasks:
    ...
    - name: Fail if CA cleanup was not successful
      ansible.builtin.fail:
        msg: "CA cleanup {{ cert_cleanup_status }}: /run/semaphore-ca/ may still exist on {{ cert_renew_ca_host }}"
      when: cert_cleanup_status is defined
```

cleanup 失敗時の fail メッセージが正しく展開される。
