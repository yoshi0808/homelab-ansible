## 対応内容

### High: monnieをcert_renew.ymlから除外

**指摘**: monnieがcert_renew.ymlに追加されているが`deploy_grafana`が未実装。
毎回「renewed but deploy FAILED」が通知サマリーに出る。

**対応**:
- `playbooks/cert_renew.yml` から monnie の issue play を削除
- 通知サマリーのホストリストを `['ansy', 'pve1', 'pve2', 'monnie']` → `['ansy', 'pve1', 'pve2']` に戻す
- コメントに「monnieはPhase2。deploy_grafana.yml実装後に追加する。」を明記

`roles/homelab_cert_renew/defaults/main.yml` の monnie 設定は残す。
Phase 2 実装時に cert_renew.yml へ追加するための準備として有効。

### High: cert_renew_quory.ymlのgit追跡

**指摘**: `cert_renew_quory.yml`が未追跡のため、追跡済み差分だけcommitするとリポジトリに入らない。

**対応**: コードとしての変更は不要。ファイルは作成済みであり、commit時に`git add`する。
本ドキュメントにて`git status`で`??`になっていることを確認・記録済み。

### Medium: cert_renew_quory.ymlの失敗通知経路

**指摘**: cleanup失敗時にfailするがメール通知がなく、障害に気づきにくい。

**対応**: `playbooks/cert_renew_quory.yml` のコメントに以下を明記した。
```
# 通知: このPlaybookはメール通知を行わない。
# 失敗検知はsystemdのExitCode / OnFailure= または journalctl で行う。
# cleanup失敗時はPlaybook自体がfailするため、systemd unitのstateで検知できる。
```
systemd timer 設定時に `OnFailure=` を追加することを運用前提とする。
