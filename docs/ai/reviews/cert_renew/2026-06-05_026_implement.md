## 実装概要

`roles/systemd_timers/defaults/main.yml` の `systemd_ansible_timers` リストから、Semaphore UI スケジュールに移行済みのエントリをコメントアウトした。  
`cert-renew-quory` エントリは systemd timer で運用継続のためそのまま維持した。

## 変更内容の要約

### コメントアウトしたエントリ（Semaphore UI スケジュールに移行済み）

| エントリ名 | playbook | schedule |
|---|---|---|
| `authy-healthcheck` | `playbooks/radius_healthcheck.yml` | `*-*-* 05:30:00` |
| `proxmox-healthcheck` | `playbooks/proxmox_healthcheck.yml` | `*-*-* 05:40:00` |
| `proxmox-patch-dryrun` | `playbooks/proxmox_patch_dryrun.yml` | `Fri *-*-* 17:30:00` |
| `authy-reboot-if-required` | `playbooks/ubuntu_nightly.yml` | `*-*-* 03:30:00` |
| `proxmox-hw-check` | `playbooks/proxmox_hw_check.yml` | `*-*-* 05:45:00` |
| `monitoring-healthcheck` | `playbooks/monitoring_healthcheck.yml` | `*-*-* 05:35:00` |

各エントリの先頭に `# 以下のエントリは Semaphore UI スケジュールに移行済み` コメントを追加した上でコメントアウトした。  
設定値は将来の参照のために残している。

### 維持したエントリ

| エントリ名 | playbook | schedule |
|---|---|---|
| `cert-renew-quory` | `playbooks/cert_renew_quory.yml` | `Sat *-*-01..07 00:35:00` |

`cert-renew-quory` は Semaphore の外から実行するため systemd timer で運用継続。変更なし。

## 注意点

- quory 上で `systemd_timers` role を実行すると、コメントアウトされたエントリに対応する systemd timer が削除される（role の実装次第）。移行済みの timer unit ファイルが quory 上に残っている場合は、手動で `systemctl disable --now <timer>` して unit ファイルを削除すること。
- コメントアウトした設定値は Semaphore UI のスケジュール設定を確認する際の参照として残している。
