## 変更内容の要約

### Medium 対応: cert_renew_policy.md の SAN 説明を更新

変更前:
```
| SAN | DNS + IP（hosts_config で定義） |
```

変更後:
```
| SAN | DNS + IPv4（発行時に CA ホストで `getent ahostsv4` により動的取得） |
```

`defaults/main.yml` に IP を直書きせず、発行時に CA ホスト上の `/etc/hosts` / DNS から動的に IPv4 を取得する現在の方式を正本ドキュメントに反映した。「hosts_config で定義」という記述を削除し、今後の実装・レビューで誤解が生じないようにした。

### Low 対応 1: `getent hosts` → `getent ahostsv4` で IPv4 を厳密に保証

変更前:
```yaml
- name: Resolve hostname IPv4 address from CA host
  ansible.builtin.command:
    cmd: getent hosts {{ cert_host_config.cn }}
```

変更後:
```yaml
- name: Resolve hostname IPv4 address from CA host
  ansible.builtin.command:
    cmd: getent ahostsv4 {{ cert_host_config.cn }}
```

- `getent hosts` は IPv6 行が先に返る可能性があったが、`getent ahostsv4` は IPv4 エントリのみを返す
- `stdout.split()[0]` で先頭フィールドを取得する処理は変わらない（`getent ahostsv4` の出力フォーマットは `<IP>    <STREAM|DGRAM|RAW>    <hostname>` なので split()[0] が IPv4 アドレスになる）
- 名前解決できない場合は `failed_when: rc != 0` で fail する点も変わらない
- fact をセットするタスク名も `Set resolved IPv4 fact` に修正（`IP` → `IPv4` と明確化）

### Low 対応 2: Grafana TLS 検証に IP SAN 確認を追加

変更前:
```yaml
- name: Verify Grafana TLS certificate SAN contains expected hostname
  ...
  failed_when: cert_host_config.cn not in grafana_cert_check.stdout
```

変更後:
```yaml
- name: Verify Grafana TLS certificate SAN contains expected hostname and IP
  ...
  failed_when: >-
    cert_host_config.cn not in grafana_cert_check.stdout or
    (cert_resolved_san_ip is defined and ('IP Address:' + cert_resolved_san_ip) not in grafana_cert_check.stdout)
```

- `openssl x509 -noout -ext subjectAltName` の出力には `IP Address:monnie.internal` のような形式で IP SAN が含まれる
- `cert_resolved_san_ip is defined` ガードにより、IP 解決が実行されなかった場合は IP チェックをスキップする（後方互換性を維持）
- `cert_host_config.cn` の DNS SAN チェック（前条件）と IP SAN チェック（後条件）を OR で組み合わせ、どちらか一方でも失敗したら fail する
- タスク名も「and IP」を追加して内容を明示した

## 注意点

- `getent ahostsv4` は glibc ベースの Linux（Debian / Ubuntu）で利用できる。quory は Ubuntu であるため問題なし。
- IP SAN チェックの文字列 `'IP Address:' + cert_resolved_san_ip` は OpenSSL の `openssl x509 -ext subjectAltName` 出力形式に依存する。バージョンによっては `IP:` と出力される場合があるが、Ubuntu の OpenSSL 3.x は `IP Address:` 形式で出力する。
- `cert_resolved_san_ip is defined` ガードは `cert_needs_renewal: false` の際にタスク自体が `when: cert_needs_renewal` でスキップされるため、実際には常に defined のまま到達する。安全のためガードを残している。
