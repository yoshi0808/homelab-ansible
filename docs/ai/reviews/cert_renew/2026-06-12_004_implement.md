```sh

  -CAfile <home_radius_ca.crt> \
  -untrusted /home/yoshi/.cert/ca/home_tls_ca.crt \
  /run/semaphore-ca/certs/<hostname>.crt
```

### [Medium] ansy 除外挙動の確認回答の訂正

**指摘内容**: 前回の implement メモの回答が要求仕様の記述と対応していなかった。

**訂正内容**:

要求仕様の「ansy実行時に ansy 自身の証明書配布が除外される既存挙動」という表現は、
以下の実態を意図したものであった。

> **cert_renew_quory.yml（systemd timer から実行されるPlaybook）は quory のみを対象とし、ansy を含まない。**

現行実装の正確な整理:

| Playbook | 実行元 | ansy の扱い |
|---|---|---|
| `cert_renew.yml` | quory の Semaphore | **ansy を対象に含む** (`hosts: ansy` の play が存在する) |
| `cert_renew_quory.yml` | quory の systemd timer | **ansy を含まない** (`hosts: quory` のみ) |

両 Playbook とも冒頭の assert で quory 以外からの実行を禁止している。
したがって「ansy から実行する」という状況は現行実装では発生しない。

要求仕様の「ansy実行時」という表現は、「ansy の証明書を扱う操作時」ではなく
「cert_renew_quory.yml 実行時（= quory のみを更新するPlaybook実行時）に
ansy が対象外になっている」という意味であったと解釈する。

この挙動は v2.0 実装でも維持されている:
- `cert_renew_quory.yml` に ansy を追加する変更は行っていない ✓
- `cert_renew.yml` の ansy 対象 play は変更していない ✓
