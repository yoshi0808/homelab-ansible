# 実装内容: CA証明書・CA秘密鍵の改行コード修正

## 問題

Semaphore 変数グループ (CA_Info) の SECRETS に登録した `homelab-ca-crt` / `homelab-ca-key` は、
環境変数として渡される際に改行が `\n` のリテラル文字列になる。

`prepare_ca.yml` で `lookup('env', 'homelab-ca-crt')` を `content:` にそのまま渡すと、
1行に連結されたファイルが `/run/semaphore-ca/ca.crt` に書かれ、
openssl が PEM として認識できずエラーになる。

## 変更ファイル

| ファイル | 変更内容 |
|---|---|
| `roles/homelab_cert_renew/tasks/prepare_ca.yml` | CA証明書・CA秘密鍵の `content:` に `replace('\\n', '\n')` を追加 |

## 変更内容

```yaml
# Before
content: "{{ lookup('env', 'homelab-ca-crt') }}"

# After
content: "{{ lookup('env', 'homelab-ca-crt') | replace('\\n', '\n') }}"
```

CA秘密鍵も同様:

```yaml
# Before
content: "{{ lookup('env', 'homelab-ca-key') }}"

# After
content: "{{ lookup('env', 'homelab-ca-key') | replace('\\n', '\n') }}"
```

`replace('\\n', '\n')` は Jinja2 フィルター呼び出しであり:
- 第1引数 `'\\n'` は Python 文字列の `\n` 2文字 (バックスラッシュ + n)
- 第2引数 `'\n'` は Python 文字列の改行文字 (LF)

これにより `\n` リテラルを実際の改行に置換し、正しい PEM フォーマットのファイルが生成される。

## 確認

```bash
ansible-playbook playbooks/cert_renew.yml --syntax-check
# → playbook: playbooks/cert_renew.yml (エラーなし)
```
