## 指摘対応の詳細

### [Medium] 実機確認手順の訂正

**指摘内容**:
- 前回の確認コマンドは `/run/semaphore-ca/certs/<hostname>.crt`（tmpfs）を対象にしていたが、cleanup play で削除済みのため「次回実行後」には存在しない
- grep パターンに `Verify return code` が含まれておらず、期待結果として記載した内容が表示されない

**訂正**: cleanup 後も存在する各サービスの**配布済み証明書**を対象とする正確なコマンドに差し替える。

#### 訂正後の確認手順（運用者確認待ち）

**(1) チェーン検証（cleanup 後に配布済みパスを使用）**

配布済み fullchain 証明書（リーフ + Home-TLS-CA）はそのまま `openssl verify` に渡せる。
OpenSSL がファイル内の中間CA証明書を使ってルートCA までのチェーンを構築する。

```sh
# ルートCA証明書（Home-RADIUS-CA）を安全なメディアから取得済みであること

# ansy の Semaphore 証明書（ansy 上で実行）
openssl verify -CAfile home_radius_ca.crt /etc/semaphore/tls/ansy.internal.crt

# quory の Semaphore 証明書（quory 上で実行）
openssl verify -CAfile home_radius_ca.crt /etc/semaphore/tls/quory.internal.crt

# monnie の Grafana 証明書（monnie 上で実行）
openssl verify -CAfile home_radius_ca.crt /etc/grafana/certs/monnie.crt

# pve1 / pve2 は配布先パスがアクセスしにくいため s_client で代替（下記 (2) を使用）
```

期待結果（各コマンド）: `<path>: OK`

**(2) 各サービスのフルチェーン提示確認（s_client）**

`-showcerts` の出力から subject (s:) と issuer (i:) を抽出し、
中間CA証明書まで提示されていること・`Verify return code: 0 (ok)` を確認する。

```sh
# Semaphore: ansy（quory から実行）
echo | openssl s_client -connect ansy.internal:3000 -servername ansy.internal \
  -showcerts 2>&1 | grep -E "( s:| i:|Verify return code)"

# Semaphore: quory（quory 上で実行）
echo | openssl s_client -connect quory.internal:3000 -servername quory.internal \
  -showcerts 2>&1 | grep -E "( s:| i:|Verify return code)"

# Grafana: monnie（quory から実行）
echo | openssl s_client -connect monnie.internal:3000 -servername monnie.internal \
  -showcerts 2>&1 | grep -E "( s:| i:|Verify return code)"

# pveproxy: pve1（quory から実行）
echo | openssl s_client -connect pve1.internal:8006 -servername pve1.internal \
  -showcerts 2>&1 | grep -E "( s:| i:|Verify return code)"

# pveproxy: pve2（quory から実行）
echo | openssl s_client -connect pve2.internal:8006 -servername pve2.internal \
  -showcerts 2>&1 | grep -E "( s:| i:|Verify return code)"
```

期待結果の形式（例: quory）:

```text
 0 s:CN = quory.internal
   i:CN = Home-TLS-CA
 1 s:CN = Home-TLS-CA
   i:CN = Home-RADIUS-CA
Verify return code: 0 (ok)
```

確認ポイント:
- cert 0（depth=0）の issuer が `Home-TLS-CA` であること（中間CA署名）
- cert 1（depth=1）が `Home-TLS-CA` であること（中間CA証明書が提示されている）
- `Verify return code: 0 (ok)` であること
