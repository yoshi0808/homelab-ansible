## 対応詳細

### 問題

`ignore_errors: true` は Ansible モジュールの失敗 (non-zero rc、module error) を吸収するが、**unreachable は吸収しない**。

`delegate_to: quory` で quory に SSH できない場合 (unreachable):
- `ignore_errors: true` では吸収されず、cleanup Play の `localhost` が failed 状態になる
- 通知 Play (`hosts: localhost`) が実行されず、CA 秘密鍵削除未確認の状態がメールに届かない
- `cert_cleanup_failed` も設定されない

### 対応

**cleanup.yml**: `ignore_unreachable: true` を追加し、unreachable も吸収して通知 Play まで確実に進む。`cert_cleanup_failed` (bool) を廃止し `cert_cleanup_status` (文字列: `FAILED` / `UNREACHABLE`) に変更。

```yaml
- name: Remove CA staging directory from tmpfs
  ansible.builtin.file:
    path: "{{ cert_renew_ca_dir }}"
    state: absent
  delegate_to: "{{ cert_renew_ca_host }}"
  become: true
  register: cleanup_result
  ignore_errors: true
  ignore_unreachable: true      # ← 追加

- name: Record cleanup unreachable
  ansible.builtin.set_fact:
    cert_cleanup_status: UNREACHABLE
  when: cleanup_result.unreachable | default(false) | bool

- name: Record cleanup failure
  ansible.builtin.set_fact:
    cert_cleanup_status: FAILED
  when:
    - not (cleanup_result.unreachable | default(false) | bool)
    - cleanup_result is failed
```

`cert_cleanup_status` の設定ロジック:
- quory が unreachable → `UNREACHABLE` (unreachable check を先に評価)
- quory に到達できたがファイル削除が失敗 → `FAILED`
- 削除成功 → 未設定 (通知側で `default('OK')` を使用)

**cert_renew.yml** 通知:

```jinja2
CA cleanup: {{ cert_cleanup_status | default('OK') }}
```

3 状態を明示:
- `OK` → 正常削除
- `FAILED` → 削除失敗 (ファイルシステム/権限エラー等)
- `UNREACHABLE` → quory に SSH 接続できず削除未確認

**cert_renew.yml** 通知後の fail:

```yaml
- name: Fail if CA cleanup was not successful
  ansible.builtin.fail:
    msg: "CA cleanup {{ cert_cleanup_status }}: /run/semaphore-ca/ may still exist on {{ cert_renew_ca_host }}"
  when: cert_cleanup_status is defined
```

`cert_cleanup_status is defined` により FAILED・UNREACHABLE の両方で Playbook を失敗させる。

### 動作フロー

| cleanup 結果 | `cert_cleanup_status` | 通知に届くか | 通知の CA cleanup 行 | Playbook 終了状態 |
|---|---|---|---|---|
| 削除成功 | 未設定 | YES | `OK` | SUCCESS |
| 削除失敗 (module error) | `FAILED` | YES | `FAILED` | FAILED (通知後) |
| quory unreachable | `UNREACHABLE` | YES | `UNREACHABLE` | FAILED (通知後) |

### 注意点

`ignore_unreachable: true` は Ansible 2.7 以降で利用可能。quory が unreachable の場合、`/run/semaphore-ca/` は tmpfs のため quory の次回再起動で自動消去される。Playbook は FAILED で終了するため Semaphore UI でエラーが記録される。
