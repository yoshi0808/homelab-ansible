# 実装内容: レビュー指摘対応 (2026-06-04_015_review.md)

## ユーザーコメントの確認

> 実行ホストがquoryの場合、ansyの場合で挙動が変えられるのではないかと思います。

レビュー P1 の指摘（ansy / proxmox play で `cert_renew_ca_host: quory` が残り SSH 自己接続が発生する）については、Yoshinobu コメントの通り、**このPlaybookはquoryからのみ実行する前提**で `cert_renew_ca_host: localhost` を全playに揃えることで解決できる。対応はシンプルに実施した。

| ansy / proxmox play で CA 委譲先が `quory` のため SSH 自己接続が残る | P1 | ansy play と proxmox play に `vars: cert_renew_ca_host: localhost` を追加 |

## 変更詳細

### ansy play

```yaml
# Before
- name: cert_renew - issue and deploy ansy cert
  hosts: ansy
  become: true

# After
- name: cert_renew - issue and deploy ansy cert
  hosts: ansy
  become: true
  vars:
    cert_renew_ca_host: localhost    # ← 追加
```

### proxmox play

```yaml
# Before
- name: cert_renew - issue and deploy proxmox certs
  hosts: proxmox
  serial: 1
  become: true

# After
- name: cert_renew - issue and deploy proxmox certs
  hosts: proxmox
  serial: 1
  become: true
  vars:
    cert_renew_ca_host: localhost    # ← 追加
```

---

## cert_renew_ca_host の設定一覧（修正後）

| play | hosts | connection | cert_renew_ca_host |
|---|---|---|---|
| prepare CA | quory | local | (not used) |
| issue/deploy ansy | ansy | SSH | **localhost** ← 今回追加 |
| issue/deploy quory | quory | local | localhost |
| issue/deploy proxmox | proxmox | SSH + serial:1 | **localhost** ← 今回追加 |
| cleanup | localhost | local | localhost |
| notification | localhost | local | localhost |

全 play で `cert_renew_ca_host: localhost` が揃い、quory から実行した場合に `delegate_to: localhost` = local connection となる。SSH 自己接続 (quory → quory.internal) が発生する箇所がなくなった。

---

## 設計上の注意: このPlaybookはquoryからのみ実行すること

`cert_renew_ca_host: localhost` を全 play に設定したため、ansy から実行した場合:
- CA 鍵・証明書が ansy の `/run/semaphore-ca/` に展開される
- 要求仕様「CA秘密鍵はansyに置かない（quoryのみ）」に違反する

このPlaybook はquoryからのみ実行すること。ファイル冒頭に記載済み:

```
# このPlaybookはquoryからのみ実行すること。
# quory上の処理はすべてlocal connectionで実行し、SSH経由の自己接続を避ける。
```
