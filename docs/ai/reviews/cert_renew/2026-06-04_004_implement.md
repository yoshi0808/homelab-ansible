## P1 対応詳細: CA鍵展開先の quory 固定

### defaults/main.yml

```yaml
cert_renew_ca_host: quory
```

`cert_renew_ca_host` を変数として定義し、prepare/issue/cleanup の処理先を明示的に quory に固定。

### playbooks/cert_renew.yml

```yaml
# Before
- name: cert_renew - prepare CA key on quory
  hosts: localhost

# After
- name: cert_renew - prepare CA key on quory
  hosts: quory
```

Play 1 (prepare) と Play 4 (cleanup) の `hosts: localhost` を `hosts: quory` に変更。  
これにより ansible-playbook を ansy から実行しても CA 秘密鍵は quory 上の `/run/semaphore-ca/` にのみ展開される。

### issue.yml

```yaml
# Before
  delegate_to: localhost

# After
  delegate_to: "{{ cert_renew_ca_host }}"
```

openssl 操作（テンプレート展開・鍵生成・CSR生成・CA署名）の委譲先を変数化。  
`cert_renew_ca_host: quory` により実質的に quory で処理される。

### deploy_semaphore.yml / deploy_proxmox.yml

`copy src: /run/semaphore-ca/...` はコントローラーのローカルファイルを参照する。  
ansy から実行した場合は ansy 上のファイルを探すため、quory で生成したファイルが見つからない。

`slurp delegate_to: cert_renew_ca_host` に切り替えることで、常に quory 上の生成物を各ホストへ配布する設計にした。

```yaml
- name: Read certificate from CA host
  ansible.builtin.slurp:
    src: "{{ cert_renew_ca_dir }}/certs/{{ inventory_hostname }}.crt"
  register: slurp_crt
  delegate_to: "{{ cert_renew_ca_host }}"
  when: cert_needs_renewal

- name: Deploy certificate
  ansible.builtin.copy:
    content: "{{ slurp_crt.content | b64decode }}"
    dest: "{{ cert_host_config.cert_path }}"
    ...
```

## P2 対応詳細: deploy 成功追跡と通知改善

### deploy 状態の追跡

`deploy_semaphore.yml` と `deploy_proxmox.yml` の末尾に `cert_deployed: true` の set_fact を追加。  
deploy タスクが失敗するとこの行には到達しないため、`cert_deployed` が未設定になる。

```yaml
- name: Record deploy success
  ansible.builtin.set_fact:
    cert_deployed: true
  when: cert_needs_renewal
```

### 通知文面の改善 (playbooks/cert_renew.yml)

```yaml
{% set renewed = hostvars[h].cert_renewed | default(false) | bool %}
{% set deployed = hostvars[h].cert_deployed | default(false) | bool %}
{% if not renewed %}
{{ h }}: skipped
{% elif deployed %}
{{ h }}: renewed and deployed
{% else %}
{{ h }}: renewed but deploy FAILED
{% endif %}
```

3パターンを明示的に区別:
- `skipped` = 証明書の更新が不要だった
- `renewed and deployed` = 発行・配布・再起動まで完了
- `renewed but deploy FAILED` = 発行はしたが配布または再起動が失敗した

## P2 対応詳細: PVE 一時秘密鍵の確実削除

### block/always 構造

```yaml
- name: Deploy certificate to PVE node
  block:
    - slurp crt/key from quory
    - copy to /tmp/pve_cert_{{ inventory_hostname }}.pem
    - copy to /tmp/pve_key_{{ inventory_hostname }}.pem
    - pkcs8 変換 → /tmp/pve_key_pkcs8_{{ inventory_hostname }}.pem
    - pvenode cert set
    - set_fact cert_deployed: true
  always:
    - /tmp/pve_cert_{{ inventory_hostname }}.pem を削除
    - /tmp/pve_key_{{ inventory_hostname }}.pem を削除
    - /tmp/pve_key_pkcs8_{{ inventory_hostname }}.pem を削除
```

- `pvenode cert set` が失敗しても `always` セクションが実行され `/tmp` 上の一時ファイルが削除される
- `failed_when: false` を cleanup タスクに付与し、削除タスク自体の失敗でプレイが止まらないようにした

### ホスト固有ファイル名

`/tmp/pve_cert.pem` → `/tmp/pve_cert_{{ inventory_hostname }}.pem`  
`/tmp/pve_key.pem` → `/tmp/pve_key_{{ inventory_hostname }}.pem`  
`/tmp/pve_key_pkcs8.pem` → `/tmp/pve_key_pkcs8_{{ inventory_hostname }}.pem`

pve1/pve2 を並列実行または途中再実行した場合のファイル衝突を防ぐ。
