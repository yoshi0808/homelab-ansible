# 要求仕様: homelab サーバー証明書自動更新

> **注意**: この要求仕様はPhase 1初回実装時点の内容です。
> CA鍵取得方式・Playbook構成・運用方針の現在の正本は以下を参照してください。
> `docs/ai/prompts/cert_renew_policy.md`

## 目的
ホームラボ内の管理系Web UIのTLS証明書を月1回自動更新する。
ブラウザの証明書警告を解消し、短命証明書運用に移行する。

## 対象サービス（Phase 1）
| ホスト | サービス | 証明書パス | 鍵パス | 反映方法 |
|---|---|---|---|---|
| ansy | semaphore | /etc/semaphore/tls/ansy.internal.crt | /etc/semaphore/tls/ansy.internal.key | systemctl restart semaphore |
| quory | semaphore | /etc/semaphore/tls/quory.internal.crt | /etc/semaphore/tls/quory.internal.key | systemctl restart semaphore |
| pve1 | pveproxy | pvenode cert set で適用 | pkcs8変換後に適用 | systemctl restart pveproxy |
| pve2 | pveproxy | pvenode cert set で適用 | pkcs8変換後に適用 | systemctl restart pveproxy |

Phase 2: monnie（Grafana）は初回除外スコープ。

## CA情報
- CA証明書: 環境変数 homelab-ca-crt（Semaphore変数グループ CA_Info のSECRETSで管理）
- CA秘密鍵: 環境変数 homelab-ca-key（同上）
- アルゴリズム: EC secp384r1
- CA証明書はPlaybook実行確認済み（lookup('env', 'homelab-ca-crt')で取得可能）

## 証明書仕様
- 有効期間: 45日
- 更新条件: 残日数が15日以下の場合に再発行（force_renewオプションで強制更新も可）
- 鍵アルゴリズム: EC secp384r1
- SAN必須

## SAN定義
| ホスト | SAN |
|---|---|
| ansy | DNS:ansy.internal, DNS:ansy, Resolved host: ansy.internal |
| quory | DNS:quory.internal, DNS:quory, Resolved host: quory.internal |
| pve1 | DNS:pve1.internal, DNS:pve1, Resolved host: pve1.internal |
| pve2 | DNS:pve2.internal, DNS:pve2, Resolved host: pve2.internal |

## CA鍵の展開方式
1. quory上の/run/semaphore-ca/に一時展開（tmpfsのため再起動後に残らない）
2. chmod 600
3. 処理後に必ず削除
4. no_logを適切に使用

## 実行フロー
1. CA鍵・証明書を/run/semaphore-ca/に一時展開
2. 各ホストの現在の証明書残日数を確認
3. 更新対象の場合のみ証明書を発行
4. 各ホストへ配布
5. サービス再起動
6. 一時CA鍵を削除
7. メール通知

## PVE固有の処理
- 秘密鍵はPKCS8変換が必要: openssl pkcs8 -topk8 -nocrypt
- 証明書適用: pvenode cert set /tmp/pve_cert.pem /tmp/pve_key_pkcs8.pem --force
- /tmpの一時ファイルは処理後に削除
- pvenode cert setはpveproxy再起動が別途必要

## 制約
- CA秘密鍵はansyに置かない（quoryのみ）
- CA秘密鍵はGit管理しない
- 証明書・秘密鍵・CSRはGit管理しない（.gitignoreに追加）
- no_logを秘密鍵操作タスクに必ず付ける
- 変更系（変更あり: 証明書配置・サービス再起動）

## 初回実装スコープ
- Phase 1の4ホスト（ansy/quory/pve1/pve2）
- CA鍵の一時展開と削除
- 証明書発行・配布・再起動
- メール通知

## 初回除外スコープ
- monnie（Grafana）Phase 2
- 証明書残日数チェックによる条件付き更新（初回は常に発行でもよい）

## shell / Ansible 責務分離
- shell: opensslコマンド実行
- Ansible tasks: フロー制御・配布・再起動・通知
