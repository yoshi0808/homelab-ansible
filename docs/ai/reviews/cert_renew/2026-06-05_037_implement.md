## 変更内容の要約

### 1. deploy_grafana.yml（新規）

`deploy_semaphore.yml` と同一パターンで実装。

- CA ホスト (localhost/quory) から証明書・秘密鍵を slurp して monnie に配置
- 対象ディレクトリ: `{{ cert_host_config.cert_path | dirname }}` （`/etc/grafana/certs`）
  - owner: grafana, group: grafana, mode: 0750
- 証明書: owner: grafana, group: grafana, mode: 0644
- 秘密鍵: owner: grafana, group: grafana, mode: 0600、`no_log: true`
- 再起動: `systemctl restart grafana-server`（ansible.builtin.systemd）
- `cert_deployed: true` を set_fact して完了記録

### 2. defaults/main.yml（monnie パス修正）

| 項目 | 変更前 | 変更後 |
|---|---|---|
| cert_path | `/etc/grafana/tls/monnie.internal.crt` | `/etc/grafana/certs/monnie.crt` |
| key_path | `/etc/grafana/tls/monnie.internal.key` | `/etc/grafana/certs/monnie.key` |
| cert_check_path | `/etc/grafana/tls/monnie.internal.crt` | `/etc/grafana/certs/monnie.crt` |

### 3. cert_renew.yml（monnie play 追加）

- ヘッダコメントの `# 対象:` に `monnie (grafana)` を追加
- Phase2 コメント（`# monnie (grafana) はPhase2...`）を削除
- proxmox play の直後に monnie play を追加（issue → deploy_grafana）
- cert_renew_summary のループ対象に `'monnie'` を追加

### 4. deploy_semaphore.yml（owner/group 修正）

| タスク | 変更前 | 変更後 |
|---|---|---|
| ディレクトリ | owner/group: root, mode: 0750 | owner/group: yoshi, mode: 0755 |
| 証明書 copy | owner/group: root | owner/group: yoshi |
| 秘密鍵 copy | owner/group: root | owner/group: yoshi |

Semaphore は yoshi ユーザーで起動するため、証明書ファイルの所有者を yoshi に変更。

### 5. deploy_proxmox.yml（group: root 追加）

- `/tmp/pve_cert_*.pem` copy タスクに `group: root` を追加
- `/tmp/pve_key_*.pem` copy タスクに `group: root` を追加
- /tmp 一時ファイルのそれ以外の設定は既存のまま維持

## 注意点

- monnie の `/etc/grafana/certs/` ディレクトリは grafana ユーザーが読める必要があるため mode: 0750 (grafana:grafana) とした
