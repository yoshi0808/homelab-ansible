# implement: cert_renew_quory.yml にメール通知を追加

## 実装概要

`cert_renew_quory.yml` に `cert_renew.yml` と同パターンのメール通知 play を追加した。
証明書更新結果（renewed/skipped/failed）と実行日時をメールで通知する。
cleanup 失敗時もメール送信後に playbook が fail するよう、fail タスクを notification play へ移動した。

## 作成・更新したファイル

| ファイル | 変更種別 |
|---|---|
| `playbooks/cert_renew_quory.yml` | 更新 |

## 変更内容の要約

### ヘッダーコメント更新

```diff
-# 通知: このPlaybookはメール通知を行わない。
-# 失敗検知はsystemdのExitCode / OnFailure= または journalctl で行う。
-# cleanup失敗時はPlaybook自体がfailするため、systemd unitのstateで検知できる。
+# 通知: 証明書更新結果をメールで通知する（renewed/skipped/failed）。
+# cleanup失敗時もメール通知後にPlaybook自体がfailするため、systemd unitのstateで検知できる。
```

### cleanup play の変更

`Fail if CA cleanup was not successful` タスクを cleanup play から削除し、notification play へ移動。
これにより cleanup 失敗時もメールが送信されてから fail する。

### notification play を追加

```yaml
- name: cert_renew_quory - send completion notification
  hosts: localhost
  gather_facts: false
  become: false
  vars:
    cert_renew_ca_host: localhost
  tasks:
    - name: Load mail variables
    - name: Build renewal summary
    - name: Send completion notification   # ignore_errors: true
    - name: Fail if CA cleanup was not successful
```

通知内容：
- `hostvars['quory']` から `cert_needs_renewal` / `cert_renewed` / `cert_deployed` を参照してサマリーを生成
- subject: `[cert_renew_quory] Certificate renewal completed`
- `inventory_dir/../vars/mail.yml` を `include_vars` で読み込む（cert_renew.yml と同じパス）
- `ignore_errors: true` でメール送信失敗時も続行
- CA cleanup 失敗時は通知後に `ansible.builtin.fail` で終了

## 注意点

- `mail.yml` は Vault 暗号化済み。playbook 実行時に `--ask-vault-pass` または Vault パスワードファイルが必要。
- `force_renew: true` のため `quory: skipped` は通常発生しないが、ロジックは `cert_renew.yml` と統一した。
- hostvars['quory'] は `connection: local` で実行された play のホスト変数を参照するため、問題なくアクセス可能。
