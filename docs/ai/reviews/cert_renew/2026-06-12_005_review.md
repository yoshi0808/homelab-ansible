# Review: cert_renew role Home-TLS-CA 対応 再レビュー

## Summary

前回レビュー後の追加実装により、以下は改善されている。

ただし、追加された実機確認コマンドは期待する確認を行えず、Policy にも ansy への CA 配置記述が残っている。追加のドキュメント修正が必要である。

## Findings

### Medium: 実機確認手順が cleanup 後には実行できず、提示チェーンの内容も判定できない

- File: `docs/ai/reviews/cert_renew/2026-06-12_004_implement.md`
- Lines: 37-82

チェーン検証コマンドは `/run/semaphore-ca/certs/<hostname>.crt` を対象にしているが、`/run/semaphore-ca` は Playbook の cleanup play で削除される。追加実装メモにある「次回の cert_renew 実行後」にこのパスを指定しても、ファイルは既に存在しない。

実行後に検証する手順とするなら、各サービスへ配布済みの証明書からリーフ証明書を取り出して検証するか、cleanup 前に検証するタスクを Playbook に追加する必要がある。

また、`s_client -showcerts` の出力を次の grep に通している。

```text
grep -E "^(subject|issuer|-----BEGIN)"
```

この出力から確認できるのは主に PEM ブロックの開始数であり、PEM 内の証明書が各ホストのリーフ証明書と Home-TLS-CA であることは判定できない。さらに `Verify return code` は grep 条件に含まれていないため、期待結果として記載された `Verify return code: 0 (ok)` も表示されない。

検証手順は、少なくとも以下を実際に確認できる形へ修正する必要がある。

- `s_client -showcerts` から各 PEM ブロックを分離する
- 各証明書を `openssl x509 -noout -subject -issuer` で表示する
- ルートCAを `-CAfile` に指定して `Verify return code: 0 (ok)` を確認する
- cleanup 後にも存在する配布先証明書、または一時退避した検証用ファイルを使用する

### Low: Policy に ansy への CA 配置記述が2か所残っている

- File: `docs/ai/prompts/cert_renew_policy.md`
- Lines: 81-83, 201-205
- File: `docs/ai/reviews/cert_renew/2026-06-12_004_implement.md`
- Lines: 117-126

追加実装メモは「CA配置から ansy を削除」「CA配置を quory のみに限定」としているが、Policy には次の記述が残っている。

```text
中間CA(Home-TLS-CA)の証明書・秘密鍵が quory / ansy 上に配置される
新しい home_tls_ca.{crt,key} を quory / ansy ... に配置する
```

同じ Policy 内の「ansy への CA 配置は不要」「quory のみ」という記述と矛盾する。両方を quory のみに統一する必要がある。

### Low: requirement の CA 配置前提と訂正後の方針が不一致のまま

- File: `docs/ai/reviews/cert_renew/2026-06-12_001_requirement.md`
- Lines: 60-62
- File: `docs/ai/reviews/cert_renew/2026-06-12_004_implement.md`
- Lines: 117-126

requirement は CA を quory と ansy の両方へ配置済みと記載している一方、追加実装では ansy への配置は不要と整理している。

requirement を履歴として変更しない運用であれば、追加実装メモに「要求仕様の当該前提を訂正し、Policy v2.0 を正とする」と明記すると、最終判断時の基準が明確になる。

## Resolved Findings

前回の「ansy 除外挙動の確認回答が要求項目と一致していない」という指摘は、追加実装メモの説明により解消している。

現在の実装は次の構成であり、追加実装メモの整理と一致する。

- `cert_renew.yml`: quory の Semaphore から実行し、ansy を更新対象に含む
- `cert_renew_quory.yml`: quory の systemd timer から実行し、quory のみを更新する
- 両 Playbook とも quory 以外からの実行を assert で禁止する
