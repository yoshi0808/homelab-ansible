## 指摘事項

### 1. `connection: local` が残っており、`target_hosts` の意味が弱くなる

**重要度**: High

該当箇所:

- `playbooks/systemd_timers.yml:3`
- `playbooks/systemd_timers.yml:6`

現在の内容:

```yaml
hosts: "{{ target_hosts | default('control_nodes') }}"
...
connection: local
```

要求仕様では、デフォルトの実行対象を `control_nodes`、ansy で実行する場合は
`-e "target_hosts=dev_nodes"` で切り替える方針になっている。

しかし playbook レベルで `connection: local` を固定すると、Ansible は対象ホストへ SSH 接続せず、
playbook を起動したローカルマシン上で task を実行する。

そのため、たとえば ansy から以下を実行した場合:

```bash
ansible-playbook playbooks/systemd_timers.yml
```

`hosts` は `control_nodes` を指していても、実際の systemd unit 配置先は quory ではなく ansy になる。

これは「quory（control_nodes）の到着に伴い、どのホストグループでも再利用できる role にする」という
今回の目的とずれる。

推奨修正:

```yaml
# connection: local を削除する
```

そのうえで、ローカル実行したいホストについては inventory 側で `ansible_connection: local` を指定するか、
実行時に明示する方が、`target_hosts` の意味と整合する。

もし「この playbook は必ず対象ホスト自身の上で実行する」という設計であれば、
`target_hosts` による対象ホスト切り替えは実行場所とセットで運用する必要があるため、
実装メモまたは運用手順にその前提を明記した方がよい。
