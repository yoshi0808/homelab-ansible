# 要求仕様: systemd_timers リネーム・変数化

## 目的

quory（control_nodes）の到着に伴い、ansy 固有の命名 `ansy_systemd_timers` を
汎用的な `systemd_timers` にリネームする。
また、実行対象ホストおよび WorkingDirectory・実行ユーザーを変数化し、
どのホストグループでも再利用できる role にする。

## 対象

- playbook: `playbooks/ansy_systemd_timers.yml` → `playbooks/systemd_timers.yml`
- role: `roles/ansy_systemd_timers/` → `roles/systemd_timers/`

## 実施内容

### 1. playbook の hosts を変数化

```yaml
hosts: "{{ target_hosts | default('control_nodes') }}"
```

- デフォルトは `control_nodes`（quory）
- ansy で実行する場合は `-e "target_hosts=dev_nodes"` で指定

### 2. role 名変更

- `ansy_systemd_timers` → `systemd_timers`

### 3. WorkingDirectory と実行ユーザーを変数化

`defaults/main.yml` で定義:

| 変数名 | デフォルト値 |
|--------|-------------|
| `systemd_timers_working_dir` | `/home/yoshi/homelab-ansible` |
| `systemd_timers_run_user` | `yoshi` |
| `systemd_timers_playbook_bin` | `/usr/bin/ansible-playbook` |

group_vars または `-e` で上書き可能。

## 制約

- timer の内容（スケジュール・playbook 名）は変更しない
- 既存の動作を壊さない
- 秘密情報を扱わない

## スコープ

- 対象: リネームおよび変数化のみ
- 除外: timer の追加・削除、unit ファイル内容の変更

## shell / Ansible 責務分離

- shell: なし
- Ansible tasks: unit ファイル配置・systemd reload・enable・start を担当
