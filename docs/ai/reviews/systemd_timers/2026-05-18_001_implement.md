# systemd_timers リネーム・変数化 実装レビュー

**日付**: 2026-05-18
**担当**: Claude Code (claude-sonnet-4-6)

---

## 変更概要

`ansy_systemd_timers` という ansy 固有の命名を汎用的な `systemd_timers` にリネームし、
実行対象ホストおよび WorkingDirectory・実行ユーザーを変数化した。

---

## 変更ファイル一覧

| 変更種別 | 旧パス | 新パス |
|----------|--------|--------|
| リネーム | `playbooks/ansy_systemd_timers.yml` | `playbooks/systemd_timers.yml` |
| リネーム | `roles/ansy_systemd_timers/` | `roles/systemd_timers/` |

---

## 変更詳細

### 1. playbooks/systemd_timers.yml

- `hosts: dev_nodes` → `hosts: "{{ target_hosts | default('control_nodes') }}"`
- デフォルトは `control_nodes`（quory で実行）
- ansy で実行する場合: `ansible-playbook playbooks/systemd_timers.yml -e "target_hosts=dev_nodes"`
- role 名を `ansy_systemd_timers` → `systemd_timers` に変更

### 2. roles/systemd_timers/defaults/main.yml

変数プレフィックスを `ansy_systemd_timers_` → `systemd_timers_` に変更:

| 旧変数名 | 新変数名 | デフォルト値 |
|----------|----------|-------------|
| `ansy_systemd_timers_working_dir` | `systemd_timers_working_dir` | `/home/yoshi/homelab-ansible` |
| `ansy_systemd_timers_run_user` | `systemd_timers_run_user` | `yoshi` |
| `ansy_systemd_timers_playbook_bin` | `systemd_timers_playbook_bin` | `/usr/bin/ansible-playbook` |

`systemd_ansible_timers` リストの内容（スケジュール・playbook名）は変更なし。

### 3. roles/systemd_timers/templates/ansible-timer.service.j2

テンプレート内の変数参照を新プレフィックスに更新:

- `{{ ansy_systemd_timers_run_user }}` → `{{ systemd_timers_run_user }}`
- `{{ ansy_systemd_timers_working_dir }}` → `{{ systemd_timers_working_dir }}`
- `{{ ansy_systemd_timers_playbook_bin }}` → `{{ systemd_timers_playbook_bin }}`

### 4. roles/systemd_timers/tasks/main.yml / templates/ansible-timer.timer.j2

変数参照なし（`systemd_ansible_timers` と `timer.*` のみ）のため内容変更なし。

---

## 実行方法

```bash
# quory (control_nodes) で実行（デフォルト）
ansible-playbook playbooks/systemd_timers.yml

# ansy (dev_nodes) で実行
ansible-playbook playbooks/systemd_timers.yml -e "target_hosts=dev_nodes"

# WorkingDirectory / 実行ユーザーを上書きする場合
ansible-playbook playbooks/systemd_timers.yml \
  -e "systemd_timers_working_dir=/opt/ansible" \
  -e "systemd_timers_run_user=ansible"
```

---

## 動作への影響

- タイマーの内容（スケジュール・playbook名）は変更なし
- systemd unit ファイルのパスや名前は変更なし
- 既存の稼働中タイマーに対して再適用した場合、unit ファイルの内容が同一であれば変更なし
- group_vars で `systemd_timers_working_dir` / `systemd_timers_run_user` を定義することで環境ごとに上書き可能
