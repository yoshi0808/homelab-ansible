## 変更内容の要約

### [Low] reboot 後の facts 再収集

**問題:** `gather_facts: true` で収集した `ansible_date_time.iso8601` は play 開始時点の時刻。reboot 後の結果通知 `Time` にも同じ値が使われていた。

**対応:** `Print post-reboot result` の直前に `Gather facts after reboot` task を追加した。

```yaml
- name: Gather facts after reboot
  ansible.builtin.setup:
    gather_subset:
      - date_time
```

`gather_subset: [date_time]` で日時のみを再収集し、`ansible_date_time` を上書きする。
以降の `Send result notification` で参照する `ansible_date_time.iso8601` がサービス確認完了後の時刻になる。

---

### [Low] needrestart 実行パスをフルパスに統一

**変更前:**
```yaml
cmd: needrestart -b
```

**変更後:**
```yaml
cmd: /usr/sbin/needrestart -b
```

存在確認（`ansible.builtin.stat: path: /usr/sbin/needrestart`）と実行コマンドのパスを一致させた。
PATH 解決への依存をなくし、確認したバイナリを確実に実行する。
