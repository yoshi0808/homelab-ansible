# 要求仕様: ubuntu_nightly.yml レポート保存追加

## 目的
実行結果を毎回 JSON として保存し、後から実行履歴を確認できるようにする。

## 対象
- グループ: radius_servers
- ホスト: authy

## 作成・更新ファイル
- playbooks/ubuntu_nightly.yml

## 実施内容
- reboot の要否にかかわらず、毎回実行結果を JSON として保存する
- 保存先: reports/ubuntu-nightly/{{ inventory_hostname }}_{{ timestamp }}.json
- reboot 不要の場合は End play の直前に保存する
- reboot 実行の場合は Send result notification の直前に保存する

reboot 不要時の JSON:
{
  "host": "authy",
  "executed_at": "2026-05-10T02:00:00+0900",
  "reboot_required": false,
  "rebooted": false,
  "result": "SKIPPED"
}

reboot 実行時の JSON:
{
  "host": "authy",
  "executed_at": "2026-05-10T02:00:00+0900",
  "reboot_required": true,
  "rebooted": true,
  "freeradius": "active",
  "port_1812": "listening" or "not listening",
  "port_1813": "listening" or "not listening",
  "criticals": [],
  "result": "OK" or "CRITICAL"
}

## 参照ファイル
- roles/radius_healthcheck/tasks/main.yml（Save report タスクのパターンを踏襲）

## 制約
- ubuntu_nightly.yml のみ修正する
- 保存は delegate_to: localhost / become: false で行う
- 秘密情報を扱わない
- 変更系（変更あり: ファイル保存）

## 初回実装スコープ
- レポートディレクトリ作成タスクの追加
- reboot 不要時のレポート保存タスクの追加
- reboot 実行時のレポート保存タスクの追加

## 初回除外スコープ
- レポートのローテーション・削除
- MD 形式でのレポート出力

## shell / Ansible 責務分離
- shell: なし
- Ansible tasks: レポート生成・保存をすべて担当
