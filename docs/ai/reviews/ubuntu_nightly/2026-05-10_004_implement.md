## 変更内容の要約

### [Medium] Load mail variables の実行順序移動

**変更前の tasks 順序:**
1. Load mail variables
2. Check reboot-required flag
3. End play if reboot is not required

**変更後の tasks 順序:**
1. Check reboot-required flag
2. Check if needrestart is available
3. Run needrestart in batch mode
4. Evaluate reboot requirement
5. **End play if reboot is not required**
6. **Load mail variables** ← ここに移動

`mail.yml` が存在しない・読めない環境でも、reboot が不要であれば通知なしで正常終了するようになった。
reboot が必要な場合のみ mail vars を読み込む。

---

### [Medium] needrestart による reboot 判定追加

`ubuntu_vm_patch_policy.md` Section 4.3 では、以下のいずれかを満たす場合に reboot 要と定義されている。

- `/var/run/reboot-required` が存在する
- `needrestart` が reboot 要と判定する

追加した tasks:

**Check if needrestart is available**
- `ansible.builtin.stat` で `/usr/sbin/needrestart` の存在を確認
- `register: needrestart_bin`

**Run needrestart in batch mode**
- `ansible.builtin.command: needrestart -b`
- `changed_when: false`, `failed_when: false`
- `when: needrestart_bin.stat.exists`（needrestart 未インストール時はスキップ）

**Evaluate reboot requirement**
- `ansible.builtin.set_fact` で `reboot_required` 変数を設定
- `reboot_flag.stat.exists or (needrestart_bin.stat.exists and 'NEEDRESTART-KSTA: 3' in (needrestart_output.stdout | default('')))`
- `NEEDRESTART-KSTA: 3` は needrestart がカーネル ABI 不一致による reboot 必要と判定した状態を示す
- needrestart がスキップされた場合、`needrestart_output.stdout` は未定義だが `| default('')` で空文字列に fallback する

**End play if reboot is not required**
- `when: not reboot_required` に変更（旧: `when: not reboot_flag.stat.exists`）

### [Low] UDP ポート判定を exact match に変更

**変更前:**
```yaml
- name: Check listening ports
  ansible.builtin.command:
    cmd: ss -ulnp
  register: ss_output

# 判定: ':1812' in ss_output.stdout
```
`:18120` などが存在すると `:1812` に誤マッチする可能性があった。

**変更後:**
```yaml
- name: Check 1812/udp listening
  ansible.builtin.shell:
    cmd: ss -ulnH 'sport = :1812'
  register: port_1812_output

- name: Check 1813/udp listening
  ansible.builtin.shell:
    cmd: ss -ulnH 'sport = :1813'
  register: port_1813_output
```
- `ss 'sport = :PORT'` フィルタでポート番号の exact match
- `-H` でヘッダ行を除外
- `ansible.builtin.shell` を使用（`sport = :1812` のクォートが必要なため）
- stdout が空でなければ Listen 中と判定（`port_1812_output.stdout | length > 0`）

## 注意点

- `needrestart -b` は `become: true`（play レベル）で実行されるため sudo 権限で動作する。authy の `ann` ユーザーが NOPASSWD sudo を持っていることが前提（`core.md` Section 5 に従う）。
- `needrestart` が `/usr/sbin/needrestart` 以外のパスにインストールされている場合は `needrestart_bin.stat.exists` が false になり、スキップされる。authy は Ubuntu の標準パスを使っているため問題ない。
- `ansible.builtin.shell` の使用は `ss 'sport = :1812'` のクォートが必要なことによる。shell スクリプトを書いているのではなく、単一コマンドの引数クォートのためであり `core.md` Section 7 の制約（収集と JSON 整形のみ）は適用されない（本 playbook はそもそも shell スクリプトを使わない方針）。
