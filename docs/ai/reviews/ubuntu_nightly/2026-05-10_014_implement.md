## 変更内容の要約

### Build skipped report / Build result report の executed_at

**変更前:**
```yaml
executed_at: "{{ ansible_facts['date_time']['iso8601'] }}"
```

`ansible_facts['date_time']['iso8601']` は UTC で末尾が `Z`（例: `2026-05-10T17:00:00Z`）になる。

**変更後:**
```yaml
executed_at: "{{ lookup('pipe', 'date +%Y-%m-%dT%H:%M:%S+09:00') }}"
```

`lookup('pipe', ...)` はコントローラー（localhost）上でコマンドを実行する。
コントローラーの TZ が JST（Asia/Tokyo）であれば `2026-05-10T02:00:00+09:00` の形式で出力される。
末尾を `+09:00` に固定しているため、コントローラーの TZ 設定に依存せず常に JST 表記になる。
