## 変更内容の要約

### playbooks/ubuntu_nightly.yml

tasks の構成:

1. **Load mail variables**
   - `ansible.builtin.include_vars` で `inventories/vars/mail.yml` を読み込む
   - `delegate_to: localhost`, `become: false`（radius_healthcheck パターン踏襲）

2. **Check reboot-required flag**
   - `ansible.builtin.stat` で `/var/run/reboot-required` の存在を確認
   - `register: reboot_flag`

3. **End play if reboot is not required**
   - `ansible.builtin.meta: end_host`
   - `when: not reboot_flag.stat.exists`
   - reboot 不要のとき以降の tasks はすべてスキップ、通知なし

4. **Send reboot start notification**
   - `community.general.mail` で reboot 開始を通知
   - `ignore_errors: true`（通知失敗で reboot を止めない）
   - subject: `[ubuntu_nightly] Reboot started - <hostname>`

5. **Reboot host**
   - `ansible.builtin.reboot` で reboot 実行
   - `reboot_timeout: 300`（起動完了待ちを最大 300 秒）

6. **Check freeradius service status**
   - `ansible.builtin.command: systemctl is-active freeradius`
   - `changed_when: false`, `failed_when: false`
   - stdout が `active` かどうかで判定

7. **Check listening ports**
   - `ansible.builtin.command: ss -ulnp`
   - stdout に `:1812` / `:1813` が含まれるかで判定

8. **Evaluate service and port results**
   - `ansible.builtin.set_fact` で `nightly_criticals` リストを構築
   - freeradius が active でない場合 / 各ポートが Listen していない場合に追記

9. **Print post-reboot result**
   - `ansible.builtin.debug` でコンソールにサマリーを表示

10. **Send result notification**
    - `community.general.mail` で結果通知（OK / CRITICAL）
    - subject: `[ubuntu_nightly] OK - <hostname>` or `[ubuntu_nightly] CRITICAL - <hostname>`
    - body にサービス状態・ポート状態・Criticals 一覧を含める
    - `ignore_errors: true`

11. **Fail if critical**
    - `ansible.builtin.fail` で playbook を FAILED 終了
    - `when: nightly_criticals | length > 0`

## 注意点

- `ansible.builtin.meta: end_host` は、reboot 不要のとき以降の tasks をすべてスキップする。`end_play` ではなくホスト単位で終了する。hosts が複数になったとき（将来的な拡張）も想定内の動作。
- `reboot_timeout: 300` は起動完了待ちの上限。authy が 300 秒以内に起動しない場合は FAILED になる。
- 通知失敗（SMTP 障害等）で reboot が止まらないよう `ignore_errors: true` を設定している。ただし `Fail if critical` は `ignore_errors` なし（意図的な FAILED 終了）。
- `ss -ulnp` の出力で `:1812` / `:1813` を文字列一致で確認している。ポート番号が異なる形式で出力される環境では一致しない可能性があるが、authy 上の Ubuntu では通常このフォーマットで出力される。
