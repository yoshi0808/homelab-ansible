# 要求仕様: ubuntu_nightly.yml 新規作成

## 目的
深夜に reboot_required を確認し、必要な場合のみ authy を reboot する。
reboot 後にサービス疎通を確認し、結果を通知する。
手動対応忘れを防ぎ、reboot タイミングを深夜に限定するために作る。

## 対象
- グループ: radius_servers
- ホスト: authy

## 作成・更新ファイル
- playbooks/ubuntu_nightly.yml

## 実施内容
- /var/run/reboot-required の有無を確認する
- reboot_required = false の場合、何もしない（通知なし・正常終了）
- reboot_required = true の場合、以下を順に行う
  - reboot 開始通知を送る
  - reboot する
  - 起動完了を待つ
  - freeradius サービスの状態を確認する
  - 1812/udp および 1813/udp の Listen を確認する
  - 結果通知を送る（OK または CRITICAL）

## 参照ファイル
- roles/radius_healthcheck/tasks/main.yml（mail vars 読み込みと community.general.mail 呼び出しパターンを踏襲）

## 制約
- role は作らず playbook 単体で完結させる
- radius_healthcheck.yml には一切手を入れない
- 変更系（変更あり: reboot）
- 秘密情報を扱わない
- reboot_required = false のときは通知しない

## 初回実装スコープ
- playbooks/ubuntu_nightly.yml の新規作成
- reboot_required 確認 → 条件付き reboot → サービス確認 → 通知の一連フロー

## 初回除外スコープ
- systemd timer の設定
- role 化
- authy 以外の VM への対応

## shell / Ansible 責務分離
- shell: なし（本 playbook は Ansible tasks のみで完結）
- Ansible tasks: reboot_required 確認・reboot 実行・サービス確認・通知をすべて担当
