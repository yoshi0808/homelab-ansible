## 変更内容の要約

### 1. レポート保存の追加

#### play vars に報告先ディレクトリを定義

```yaml
vars:
  nightly_report_dir: "{{ playbook_dir }}/../reports/ubuntu-nightly"
```

`playbook_dir` は `playbooks/` に解決されるため、`reports/ubuntu-nightly/` がリポジトリルート直下に作成される。
`.gitignore` の `reports/**/*.json` ルールにより、生成された JSON は git 管理対象外になる。

#### 追加 tasks の構成

**Ensure report directory exists**（常に実行・条件なし）
```yaml
- name: Ensure report directory exists
  ansible.builtin.file:
    path: "{{ nightly_report_dir }}"
    state: directory
    mode: "0755"
  delegate_to: localhost
  become: false
```
両パス共通で1回だけ実行する。

**reboot 不要パス（when: not reboot_required）**
```yaml
- name: Build skipped report
  ansible.builtin.set_fact:
    nightly_report:
      host: "{{ inventory_hostname }}"
      executed_at: "{{ ansible_facts['date_time']['iso8601'] }}"
      reboot_required: false
      rebooted: false
      result: "SKIPPED"
  when: not reboot_required

- name: Save skipped report
  ansible.builtin.copy:
    content: "{{ nightly_report | to_nice_json }}"
    dest: "{{ nightly_report_dir }}/{{ inventory_hostname }}_{{ ansible_facts['date_time']['iso8601_basic_short'] }}.json"
  delegate_to: localhost
  become: false
  when: not reboot_required
```
`End play if reboot is not required` の直前に配置。
`executed_at` は play 開始時点の facts（`gather_facts: true` で収集済み）。

**reboot 実行パス（Save result notification の直前）**
```yaml
- name: Build result report
  ansible.builtin.set_fact:
    nightly_report:
      host: "{{ inventory_hostname }}"
      executed_at: "{{ ansible_facts['date_time']['iso8601'] }}"
      reboot_required: true
      rebooted: true
      freeradius: "{{ freeradius_active.stdout.strip() }}"
      port_1812: "{{ 'listening' if port_1812_output.stdout | length > 0 else 'not listening' }}"
      port_1813: "{{ 'listening' if port_1813_output.stdout | length > 0 else 'not listening' }}"
      criticals: "{{ nightly_criticals }}"
      result: "{{ 'CRITICAL' if nightly_criticals | length > 0 else 'OK' }}"

- name: Save result report
  ansible.builtin.copy:
    content: "{{ nightly_report | to_nice_json }}"
    dest: "{{ nightly_report_dir }}/{{ inventory_hostname }}_{{ ansible_facts['date_time']['iso8601_basic_short'] }}.json"
  delegate_to: localhost
  become: false
```
`Gather facts after reboot` の後に配置。`executed_at` は reboot 完了・サービス確認後の時刻になる。

#### tasks 全体の順序

```
Check reboot-required flag
Check if needrestart is available
Run needrestart in batch mode
Evaluate reboot requirement
Ensure report directory exists      ← 常に実行
Build skipped report                ← when: not reboot_required
Save skipped report                 ← when: not reboot_required
End play if reboot is not required  ← when: not reboot_required（SKIPPED パスはここで終了）
Load mail variables
Send reboot start notification
Reboot host
Check freeradius service status
Check 1812/udp listening
Check 1813/udp listening
Evaluate service and port results
Gather facts after reboot
Print post-reboot result
Build result report                 ← reboot パスのレポート構築
Save result report                  ← reboot パスのレポート保存
Send result notification
Fail if critical
```

### 2. DEPRECATION WARNING の修正

`ansible_date_time.iso8601` を `ansible_facts["date_time"]["iso8601"]` に変更した。

対象箇所:

| task | 変更前 | 変更後 |
|---|---|---|
| Send reboot start notification body | `ansible_date_time.iso8601` | `ansible_facts["date_time"]["iso8601"]` |
| Send result notification body | `ansible_date_time.iso8601` | `ansible_facts["date_time"]["iso8601"]` |

レポート JSON の `executed_at` および保存ファイル名は、当初から `ansible_facts['date_time']` 形式で実装したため変更なし。
