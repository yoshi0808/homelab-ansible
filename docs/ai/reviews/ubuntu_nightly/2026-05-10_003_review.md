# Review: ubuntu_nightly.yml

Review date: 2026-05-10

## Summary

`playbooks/ubuntu_nightly.yml` は新規 playbook として構文チェックを通過しており、要求仕様の主要フローである `/var/run/reboot-required` 確認、条件付き reboot、freeradius と UDP 1812/1813 の確認、結果通知は実装されている。

ただし、運用上の期待とポリシーに照らすと修正した方がよい点がある。

## Findings

### Medium: reboot 不要時にも mail vars 読み込み失敗で playbook が失敗し得る

- File: `playbooks/ubuntu_nightly.yml`
- Lines: 8-21

`Load mail variables` が `/var/run/reboot-required` の確認より前に実行されている。

要求仕様では、`reboot_required = false` の場合は「何もしない（通知なし・正常終了）」とされている。しかし現在の順序では、reboot が不要でも `inventories/vars/mail.yml` が存在しない、読めない、または実行環境に未配置の場合、`meta: end_host` に到達する前に失敗する。

通知を送らない経路では mail vars は不要なので、`Check reboot-required flag` と `End play if reboot is not required` を先に実行し、その後に `Load mail variables` を移動するのが安全。

### Medium: Ubuntu VM Patch Policy の `needrestart` 判定が未実装

- File: `playbooks/ubuntu_nightly.yml`
- Lines: 14-21
- Policy: `docs/ai/prompts/ubuntu_vm_patch_policy.md` lines 98-103

ポリシーでは、方針1の VM の reboot 判定は以下のいずれかと定義されている。

- `/var/run/reboot-required` が存在する
- `needrestart` が reboot 要と判定する

現在の実装は `/var/run/reboot-required` のみを見ているため、`needrestart` 側だけが reboot 要と判定するケースでは深夜 reboot が実行されない。結果として「手動対応忘れを防ぐ」という目的から外れる可能性がある。

要求仕様ファイルでは `/var/run/reboot-required` の確認だけが明記されているため、これは要求仕様とポリシーの不一致でもある。実装側で `needrestart` 判定を追加するか、今回の初回スコープでは除外する旨を要求仕様・実装メモに明記した方がよい。

### Low: UDP port 判定が部分文字列一致で false positive になり得る

- File: `playbooks/ubuntu_nightly.yml`
- Lines: 51-65, 73-75, 91-94

`ss -ulnp` の stdout に `:1812` / `:1813` が含まれるかで Listen 判定しているため、例えば別プロセスが `:18120` を Listen しているような場合に `1812/udp` を OK と誤判定する余地がある。

`ss -H -lun 'sport = :1812'` と `ss -H -lun 'sport = :1813'` を別々に実行して stdout の有無を見る、または JSON/構造化しやすい形に寄せると判定がより堅くなる。
