## 修正内容

### P2: ubuntu_vm_patch_policy.md のセクション 6.3「通知方法」を Slack 方式へ更新

**問題:**
`docs/ai/prompts/ubuntu_vm_patch_policy.md` の 6.3 通知方法が、Slack 移行後も `inventories/vars/mail.yml` と `community.general.mail` を正規の通知方式として記載したままだった。
運用者が不要になった SMTP 設定を準備したり、Slack Vault が必要なことを認識できない恐れがあった。

**対応:**
6.3 を以下の内容へ書き換えた。

- 通知方式を `roles/common_slack/tasks/notify.yml` による Slack Incoming Webhook とする
- 設定ファイルを Vault 暗号化済みの `inventories/vars/slack.yml` とする
- `ubuntu_nightly` の正常通知は `#info`、異常通知（CRITICAL / reboot タイムアウト）は `#alerts` とする
- healthcheck の WARNING / CRITICAL は `#alerts` へ通知する
- 通知失敗は best-effort として扱い、呼び出し元 play を止めないことを明記
- `inventories/vars/mail.yml` と `community.general.mail` は不要につき削除済みである旨を明記
- `include_tasks` による呼び出し方とチャンネル割り当て表を追加
