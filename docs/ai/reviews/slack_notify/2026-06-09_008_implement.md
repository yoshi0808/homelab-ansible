## 修正内容

### P2: mail.yml が「削除済み」という事実と異なる記述を修正

**問題:**
006_implement で追加した「`inventories/vars/mail.yml` は不要になったため削除済み」という記述が、実際のリポジトリ状態と一致していなかった。
`git ls-files` で確認すると `inventories/vars/mail.yml` は現在も Git 管理対象として存在し、削除差分も git diff にない。
運用者が秘密情報の棚卸しや将来の整理時に誤認する恐れがあった。

**対応:**
「削除済み」という記述を、実態に即した「現在も Git 管理対象として存在するが、Slack 移行対象のタスクからは参照しない」へ修正した。
（`mail.yml` の実ファイル削除は要求仕様の明示スコープ外であるため、今回は文書を実態に合わせる対応とした。）

**変更箇所:** `docs/ai/prompts/ubuntu_vm_patch_policy.md:212`

変更前:
```
`inventories/vars/mail.yml` は不要になったため削除済み。`community.general.mail` モジュールも使用しない。
```

変更後:
```
`inventories/vars/mail.yml` は現在も Git 管理対象として存在するが、Slack 移行対象のタスクからは参照しない。`community.general.mail` モジュールも使用しない。
```
