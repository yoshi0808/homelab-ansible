## 変更内容の要約

### roles/common_slack/tasks/notify.yml

- `inventory_dir/../vars/slack.yml` を `include_vars` でロード
- `slack_status` に応じて `_slack_color`（good/warning/danger/#36a64f）と `_slack_icon`（✅/⚠️/🔴/❌/📋）を設定
- `slack_channel` に応じて `_slack_webhook` を切り替え（info/alerts/patches）
- `community.general.slack` で Attachment 形式（title + text + color）で送信
- Webhook URL を含むタスクには `no_log: true` を設定
- 送信失敗は `ignore_errors: true` で継続

### 移行対象ファイル共通の変更

- `include_vars mail.yml` タスクを削除
- `community.general.mail` タスクを削除
- `include_tasks notify.yml` 呼び出しに置き換え
- `slack_channel` / `slack_status` / `slack_title` / `slack_message` を vars として渡す

### roles/proxmox_patch_apply_node/tasks/main.yml

- `Load mail variables` タスクを削除
- `Assert required mail variables` タスクを削除（前提チェック簡略化）
- `_mail_subject` / `_mail_body` / `_mail_send_result` の初期化を削除、`_notify_subject` / `_notify_body` へ改名
- `Set email subject and body` → `Set notification subject and body` に改名
- `_notify_channel` / `_notify_status` を算出するロジックを追加:
  - `_apply_abort_type != 'none'` または `_apply_failed` → `alerts` / `error`
  - `_post_hc_status != 'OK'` → `patches` / `warning`
  - それ以外 → `patches` / `ok`
- `Send summary email` → `Send Slack notification` に置き換え
- レポートJSONから `mail_sent` / `mail_error` フィールドを削除
- `Re-raise failure` の条件を `_apply_failed | bool` のみに変更（メール送信失敗によるfailを廃止）

### roles/proxmox_patch_dryrun/tasks/main.yml

- `_mail_subject` / `_mail_body` → `_notify_subject` / `_notify_body` へ改名
- `_mail_attach` フラグを廃止（Slackではファイル添付しない）
- `_notify_channel` / `_notify_status` を `_final_status` から算出:
  - BLOCKED → `alerts` / `critical`
  - MAINTENANCE_REQUIRED / MAJOR_UPGRADE_DETECTED → `patches` / `warning`
  - NO_UPDATES → `patches` / `info`
  - PATCH_READY（その他） → `patches` / `ok`
- メール送信2タスク（添付あり/なし）を `include_tasks` 1箇所に統合
- `run_once: true` で1回のみ送信を維持

### roles/proxmox_healthcheck/tasks/main.yml

- WARNING/CRITICAL 時のみ `#alerts` に通知するタスクを新規追加
- 追加位置: `Print Semaphore health summary` の後、`Fail if critical` の前
- `slack_status`: criticals が1件以上なら `critical`、それ以外（warnings のみ）なら `warning`

### playbooks/cert_renew.yml / cert_renew_quory.yml

- `cert_renew_summary` に `'FAILED'` が含まれるか否かで channel/status を切り替え:
  - FAILED あり → `alerts` / `error`
  - FAILED なし → `info` / `ok`

### playbooks/ubuntu_nightly.yml

- 2プレイ（radius_servers / monitoring_servers）それぞれを移行
- reboot start通知: `info` / `#info`
- result通知: criticals ありなら `critical` / `#alerts`、なければ `ok` / `#info`

### playbooks/proxmox_patch_weekly_full.yml

- Step 11 の完了通知: `ok` / `#patches`

## 注意点

- `notify.yml` 内の `include_vars` は呼び出すたびに `slack.yml` を再ロードする。べき等であるため問題なし。
- `ubuntu_nightly.yml` の `include_tasks` は reboot をまたいで実行されるため、reboot 後の `ansible_facts["date_time"]["iso8601"]` が使える想定で記述している（既存コードと同様）。
- `proxmox_patch_dryrun` の `run_once: true` は `include_tasks` 呼び出しに付与し、notify.yml 内タスクを1回のみ実行させている。
- `proxmox_patch_apply_node` の `Re-raise failure` 条件からメール送信失敗チェックを削除した。Slack 通知失敗はプレイ失敗に影響しない。
