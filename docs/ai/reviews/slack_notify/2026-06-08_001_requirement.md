# 要求仕様: Ansible Slack通知共通化

## 目的
既存のメール通知をSlack通知に完全移行する。
共通のSlack通知タスクを作成し、各role/playbookから呼び出す形にする。

## Slack通知の設計

### チャンネルとWebhook
inventories/vars/slack.yml（Ansible Vault暗号化済み）に以下が定義されている。

```yaml
slack_webhook_info: "..."
slack_webhook_alerts: "..."
slack_webhook_patches: "..."
```

### 共通通知タスク
roles/common_slack/tasks/notify.ymlを新規作成する。

呼び出し方:
```yaml
- name: Send Slack notification
  ansible.builtin.include_tasks: "{{ playbook_dir }}/../roles/common_slack/tasks/notify.yml"
  vars:
    slack_channel: "alerts"   # info / alerts / patches
    slack_status: "critical"  # ok / warning / critical / error / info
    slack_title: "タイトル"
    slack_message: "本文"
```

### メッセージ色とアイコン
| status | color | icon |
|---|---|---|
| ok | good（緑） | ✅ |
| warning | warning（黄） | ⚠️ |
| critical | danger（赤） | 🔴 |
| error | danger（赤） | ❌ |
| info | #36a64f（青） | 📋 |

### Webhook切り替え
slack_channelに応じてWebhook URLを切り替える。
- info → slack_webhook_info
- alerts → slack_webhook_alerts
- patches → slack_webhook_patches

## 制約
- community.general.slackモジュールを使用する
- slack.ymlはVault暗号化済み。ロード方法はmail.ymlと同じ方式を踏襲する
- メール通知タスクは削除する（mail.ymlのロードも不要になれば削除）
- no_logはWebhook URLを含むタスクに設定する
