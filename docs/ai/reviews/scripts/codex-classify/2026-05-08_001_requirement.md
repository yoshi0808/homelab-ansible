# Requirement: scripts/codex-classify.sh

作成日: 2026-05-08

---

## 目的

Ansible から呼ばれ、Codex CLI に changelog 分類・日本語レポート生成を依頼する。

---

## 呼び出し方

```bash
scripts/codex-classify.sh <input_json> <output_json>
```

---

## 処理フロー

1. 引数チェック
2. `docs/ai/prompts/proxmox_patch_policy.md` から Section 3 と Section 16 のみ抽出
3. `input_json` と抽出したポリシーでプロンプトを生成（tempファイル）
4. `codex exec` で Codex CLI を呼び出す
5. 出力から JSON を抽出・検証して `output_json` に保存

---

## 出力JSON形式

```json
{
  "status_inputs": {
    "important_component_updates": [],
    "important_component_removals": [],
    "replacement_suspected": false,
    "major_upgrade_suspected": false,
    "urgency_candidate": "NORMAL"
  },
  "package_classifications": [
    {
      "package": "",
      "important_component": false,
      "remove_action": false,
      "replacement_suspected": false,
      "security_sensitive": false,
      "cve_types_detected": [],
      "urgency_candidate": "LOW",
      "urgency_reasoning": "",
      "confidence": "medium"
    }
  ],
  "mail_subject": "",
  "mail_body": "",
  "report_md": ""
}
```

---

## Codex への指示（プロンプトに含める）

- `changelog_diff` を読み CVE タイプを識別する（LPE / RCE / DoS / XSS / 認証バイパス / VM escape 等）
- patch policy の URGENT / HIGH 判断基準テーブルと照合し `urgency_candidate` を生成する
- 日本語で `mail_subject` / `mail_body` を生成する（`mail_body` は 1000 文字程度）
- 日本語で `report_md` を生成する（changelog 差分全文・分析結果を含む）
- JSON 形式のみ出力する（前後の説明文不要）

---

## 制約

- `scripts/` に配置する
- homelab-ansible リポジトリのルートから実行する前提
- `codex exec` はリポジトリディレクトリ内でのみ動作する
- tempファイルは終了時に必ず削除する
- JSON 抽出は Python3 で行う
