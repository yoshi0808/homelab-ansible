## 変更内容の要約

### scripts/codex-classify.sh

#### 1. 引数チェック

- 引数が 2 つでない場合は Usage を出して `exit 1`
- `input_json` ファイルが存在しない場合は `exit 1`
- `docs/ai/prompts/proxmox_patch_policy.md` が存在しない場合は `exit 1`
- `output_json` の親ディレクトリが存在しない場合は `exit 1`

#### 2. tempファイルと cleanup

- `mktemp` でプロンプトファイルと Codex 出力ファイルを作成
- `trap cleanup EXIT` により、正常終了・異常終了どちらでも両ファイルを削除する

#### 3. Section 3 / Section 16 の抽出

- `awk` で `## 3.` または `## 16.` から次の `## N.` の手前まで抽出する関数 `extract_section` を定義
- エスケープ警告を避けるため `\.` ではなく `[.]` を使用

#### 4. プロンプト生成

- heredoc（シングルクォート `'PROMPT_EOF'`）でシェル変数展開を防ぎながら以下を組み立てる
  - 日本語での指示（CVE識別・urgency判断・メール生成・report_md生成・JSON出力のみ）
  - Section 3 と Section 16 の抽出テキスト
  - input_json の全内容
  - 出力スキーマ例

**プロンプト修正（2026-05-08）: Codex がテンプレートをそのまま返す問題の対処**

Codex が input_json を実際に分析せず、スキーマのダミーエントリをそのまま返す問題が発生したため、指示とスキーマ説明を以下のように強化した。

- 指示ブロックの冒頭に「updates リストの各パッケージを実際に分析すること」「ダミーを削除して実パッケージ名に置き換えること」「空の JSON やテンプレートをそのまま返すことは禁止」を明示
- 出力スキーマの `package` フィールドを `""` から `"<updates の各パッケージ名に置き換えること>"` に変更
- 出力スキーマの `urgency_reasoning` を `""` から `"<分析根拠を記述すること>"` に変更
- スキーマ直前に「このエントリはダミー。このまま返さないこと」の注記を追加

#### 5. Codex CLI 呼び出し

- `cd "$REPO_ROOT"` でリポジトリルートに移動してから実行（codex exec の動作前提を満たすため）
- プロンプトは stdin 経由で渡す（changelog が長大でも引数長の制限に当たらない）

```bash
codex exec < "$PROMPT_FILE" > "$CODEX_OUTPUT" 2>&1
```

#### 6. JSON 抽出・検証（Python3）

- 環境変数でファイルパスを渡し、`<< 'PYEOF'` heredoc で Python スクリプトを実行する（シェル変数展開を防ぐため）
- Codex 出力の最初の `{` を起点に、文字列リテラル内のブレースを除外しながら深度カウントで対応する `}` を探す
- `json.loads()` でパースし、必須キー 5 件（`status_inputs` / `package_classifications` / `mail_subject` / `mail_body` / `report_md`）の存在を検証
- `json.dump(..., ensure_ascii=False, indent=2)` で `output_json` に保存

## 注意点

- `codex exec` はリポジトリルートで実行しないと動作しない前提のため、スクリプト内で `cd "$REPO_ROOT"` を行っている
- Python の JSON 抽出はコードブロックマーカー（` ```json ` 等）が Codex 出力に混入しても、最初の `{` を起点に探すため対応できる
- `set -euo pipefail` を設定しているため、途中でエラーが起きると trap が実行されて tempファイルは確実に削除される
