## Findings

### 1. `codex exec` 失敗時に詳細ログが失われる

Severity: Medium

該当箇所:

- `scripts/codex-classify.sh:132`
- `scripts/codex-classify.sh:39`

現在は以下の形になっている。

```bash
codex exec "$(cat "$PROMPT_FILE")" > "$CODEX_OUTPUT" 2>&1
```

`set -e` が有効なので、`codex exec` が非 0 で終了すると、その場でスクリプトが終了する。

その後 `trap cleanup EXIT` により `$CODEX_OUTPUT` が削除されるため、Ansible 側には Codex CLI の stderr / stdout が残らない。

通常成功パスは確認済みとのことなので、実運用上ただちに壊れるものではない。

ただし、以下のような失敗時に原因調査が難しくなる。

- 認証切れ
- モデルエラー
- Codex CLI 側の仕様変更
- ネットワーク失敗

推奨修正:

```bash
if ! codex exec "$(cat "$PROMPT_FILE")" > "$CODEX_OUTPUT" 2>&1; then
    echo "Error: codex exec failed" >&2
    sed -n '1,200p' "$CODEX_OUTPUT" >&2
    exit 1
fi
```

これは `codex exec` の呼び出し形式を変えず、エラー時の観測性だけを改善する修正である。
