## 反映した修正

### 1. 出力スキーマを policy Section 16 に合わせた

`scripts/codex-classify.sh` のプロンプト内 JSON スキーマ例に以下を追加した。

- `status_inputs.security_sensitive_updates`
- `package_classifications[].evidence`

これにより、`docs/ai/prompts/proxmox_patch_policy.md` Section 16 の最低限 JSON 出力と整合する。

### 2. JSON 検証を強化した

Codex CLI の出力 JSON に対して、以下の検証を追加した。

- `status_inputs` が object であること
- `status_inputs.security_sensitive_updates` が存在すること
- `package_classifications` が list であること
- `package_classifications` が空でないこと
- 入力 JSON の `updates` が list であること
- 入力 JSON の `updates` 数と `package_classifications` 数が一致すること
- `package_classifications[].package` が空文字またはダミー文字列のままでないこと
- `package_classifications[].evidence` が存在すること

これにより、Codex がスキーマのダミーをそのまま返すケースや、パッケージ分類漏れを検出できるようにした。
