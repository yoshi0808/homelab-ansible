## 変更内容の要約

### shell スクリプト（両ファイル共通）

`free -b` を1回呼び出してバイト単位の total / used / available を取得し、環境変数
`MEM_TOTAL_BYTES` / `MEM_USED_BYTES` / `MEM_AVAILABLE_BYTES` として Python ブロックへ渡す。

Python 側で MB 換算（1048576 除算 + round）および `used_percent` を計算し、JSON に `memory` キーを追加。

```json
"memory": {
    "total_mb": 3840,
    "used_mb": 768,
    "available_mb": 3000,
    "used_percent": 20
}
```

`free` コマンド失敗時は `mem_line="0 0 0"` にフォールバックし、Python 側で
`mem_total > 0` チェックにより `used_percent = 0` を返す。

shell は収集と JSON 整形のみを行い、判定・通知は行わない（`core.md` §7 準拠）。

### Ansible tasks/main.yml（両ファイル共通）

**Classify criticals and warnings**

| 条件 | 分類 |
|---|---|
| `memory.used_percent >= 90` | criticals |
| `memory.used_percent >= 80` | warnings |

メッセージ例: `memory used_percent is 92%`

**Build report**

`memory` セクション（total_mb / used_mb / available_mb / used_percent）をレポートに追加。

**Print health summary**

`Memory` 行を追加:

```
Memory    : total=3840MB used=768MB available=3000MB used_percent=20%
```

## 注意点

- `used_percent` は Python の `round()` で整数に丸める（0.5 は偶数丸め）。
- `free -b` の列順は Ubuntu 標準 procps に依存する（$2=total, $3=used, $7=available）。
- `monitoring_warnings` は既存実装で `"{{ [] }}"` だったが、メモリ warning 追加に伴い条件式に変更した。
