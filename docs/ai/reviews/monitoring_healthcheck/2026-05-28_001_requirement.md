# 要求仕様: monnie monitoring healthcheck 追加

## 目的
監視基盤 VM（monnie）の Prometheus / Grafana / Loki の稼働確認を行う。
authy の radius_healthcheck と同じパターンで実装する。

## 対象
- グループ: monitoring_servers（新規作成）
- ホスト: monnie（monnie.internal）

## 作成・更新ファイル
- inventories/homelab/hosts.yml（monitoring_servers グループ追加）
- inventories/homelab/group_vars/monitoring_servers.yml（新規作成）
- roles/monitoring_healthcheck/tasks/main.yml（新規作成）
- roles/monitoring_healthcheck/files/monitoring-healthcheck.sh（新規作成）
- roles/monitoring_healthcheck/defaults/main.yml（新規作成）
- playbooks/monitoring_healthcheck.yml（新規作成）
- playbooks/ubuntu_nightly.yml（monitoring_servers を対象に追加）
- roles/ansy_systemd_timers → roles/systemd_timers の defaults/main.yml
  （ansible-monitoring-healthcheck timer を追加）

## 実施内容
- monitoring-healthcheck.sh で以下を収集する
  - prometheus サービス状態（systemctl is-active prometheus）
  - grafana サービス状態（systemctl is-active grafana-server）
  - loki サービス状態（systemctl is-active loki）
  - 9090/tcp Listen 確認
  - 3000/tcp Listen 確認
  - 3100/tcp Listen 確認
  - collected_at（JST）
- Ansible tasks で CRITICAL / WARNING を判定・通知する
- レポートを reports/monitoring-health/ に保存する

## 参照ファイル
- roles/radius_healthcheck/（全体のパターンを踏襲）
- roles/systemd_timers/defaults/main.yml（timer追加）

## timer 追加内容
```yaml
  - name: monitoring-healthcheck
    description: Run monitoring healthcheck
    schedule: "*-*-* 05:35:00"
    playbook: playbooks/monitoring_healthcheck.yml
    persistent: true
```

## 制約
- shell 側は収集と JSON 整形のみ
- 判定・通知は Ansible tasks 側で行う
- 秘密情報を扱わない
- read-only（変更操作なし）

## 初回実装スコープ
- 上記ファイルの新規作成・更新すべて

## 初回除外スコープ
- promtail の healthcheck

## shell / Ansible 責務分離
- shell: 収集と JSON 整形のみ
- Ansible tasks: 判定・通知・レポート保存を担当
