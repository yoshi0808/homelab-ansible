## 実装概要

### Medium 対応: `collection_ok` フィールドの追加

**Shell 側（両スクリプト共通）**

`free -b` の実行可否と出力の有無を `if` 文で判定し、`mem_collection_ok` 変数に
`"true"` / `"false"` をセットする。

```bash
if mem_line=$(free -b 2>/dev/null | awk '/^Mem:/{print $2, $3, $7}') && [ -n "$mem_line" ]; then
  mem_collection_ok="true"
  # ...バイト値をセット
else
  mem_collection_ok="false"
  mem_total_bytes=0
  mem_used_bytes=0
  mem_available_bytes=0
fi
```

`|| mem_line="0 0 0"` フォールバックを廃止し、コマンド失敗・空出力の両方を
`collection_ok=false` として区別できるようにした。

Python 側では `MEM_COLLECTION_OK` 環境変数を受け取り、JSON の `memory.collection_ok`
に Python bool（`true` / `false`）として出力する。

**Ansible 側（両 tasks/main.yml 共通）**

Classify criticals に `collection_ok` チェックを追加:

```yaml
+ ([] if monitoring_health.memory.collection_ok
       else ['memory collection failed (free -b returned no data)'])
```

Build report の `memory` セクションに `collection_ok` を追加。

Print health summary の Memory 行に `collection_ok=...` を追加。

---

### Low 対応: warning を 80 <= used_percent < 90 に限定

warning 条件を `>= 80` から `>= 80 かつ < 90` に変更することで、
criticals と warnings の重複を排除した。

```yaml
# 変更前
+ ([] if monitoring_health.memory.used_percent | int < 80
       else ['memory used_percent is ...'])

# 変更後
+ ([] if monitoring_health.memory.used_percent | int < 80 or monitoring_health.memory.used_percent | int >= 90
       else ['memory used_percent is ...'])
```

## 注意点

- `collection_ok` が `false` のとき、`used_percent=0` は `< 80` かつ `< 90` なので閾値チェックは発火しない。critical は `collection_ok=false` の 1 件のみ追加される。
- Jinja2 で `monitoring_health.memory.collection_ok` は Python bool として評価されるため、`if ... else` 条件式でそのまま truthy/falsy として使える。
