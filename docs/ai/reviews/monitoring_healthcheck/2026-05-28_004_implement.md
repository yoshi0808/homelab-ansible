# 追加実装内容: monnie monitoring healthcheck（レビュー指摘対応）

## 対応した指摘

### [Medium] reboot 後のサービス起動待ちがなく、誤 CRITICAL になる可能性がある

`playbooks/ubuntu_nightly.yml` の monitoring_servers 向け play において、
`ansible.builtin.reboot` 完了直後に即時でサービス状態とポート Listen を確認していたため、
Prometheus / Grafana / Loki の起動が間に合わず誤 CRITICAL になる可能性があった。

### [Low] read-only 要件と script 配置タスクの扱いについて

コード変更なし。「read-only」はサービス・設定・reboot などの変更操作をしないという意味であり、
healthcheck script の配置（`/usr/local/sbin/` への copy）は許容する運用解釈とする。
既存の `radius_healthcheck` と同じ扱い。

## 変更ファイル

- `playbooks/ubuntu_nightly.yml`（monitoring_servers post-reboot サービス/ポート確認に retry を追加）

## 変更内容の詳細

`playbooks/ubuntu_nightly.yml` の monitoring_servers 向け play にある 6 つの post-reboot 確認タスクに、
以下を追加した。

```
retries: 12
delay: 5
until: <起動確認条件>
```

`retries: 12` / `delay: 5` で最大 60 秒間待機する。
`failed_when: false` は維持しているため、60 秒経過しても起動しない場合でもタスク自体は失敗せず、
最終値が register に残り、`nightly_criticals` の評価で CRITICAL として検出される。

変更対象タスク:

| タスク名 | until 条件 |
|---|---|
| Check prometheus service status | `prometheus_active.stdout.strip() == 'active'` |
| Check grafana-server service status | `grafana_active.stdout.strip() == 'active'` |
| Check loki service status | `loki_active.stdout.strip() == 'active'` |
| Check 9090/tcp listening | `port_9090_output.stdout \| length > 0` |
| Check 3000/tcp listening | `port_3000_output.stdout \| length > 0` |
| Check 3100/tcp listening | `port_3100_output.stdout \| length > 0` |
