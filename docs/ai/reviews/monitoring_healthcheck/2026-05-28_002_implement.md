monitoring_servers グループ（monnie）を新規追加し、Prometheus / Grafana / Loki の稼働確認を行う。

## 変更内容の要約

### inventories/homelab/hosts.yml
- `monitoring_servers` グループを追加
- `monnie: ansible_host: monnie.internal` を登録（core.md の名前解決方針に従い IP 直書きではなく内部 DNS 名を使用）

### inventories/homelab/group_vars/monitoring_servers.yml
- `ansible_user: ann` / `ansible_ssh_private_key_file: ~/.ssh/id_ann` を設定（radius_servers.yml と同じ）

### roles/monitoring_healthcheck/files/monitoring-healthcheck.sh
- `systemctl is-active` で prometheus / grafana-server / loki の状態を収集
- `ss -H -ltn` で 9090/tcp・3000/tcp・3100/tcp の Listen 確認（radius は UDP だったが monitoring は TCP）
- `collected_at` を JST で付与
- shell 責務: 収集と JSON 整形のみ。判定・通知は行わない

### roles/monitoring_healthcheck/tasks/main.yml
- radius_healthcheck/tasks/main.yml と同じ構造
- CRITICAL 判定: prometheus / grafana-server / loki が非 active、または各ポートが Listen していない場合
- WARNING 判定: 現時点では条件なし（空リスト）
- レポートを `reports/monitoring-health/` に保存
- WARNING / CRITICAL 時に mail 通知
- CRITICAL 時に fail

### roles/monitoring_healthcheck/defaults/main.yml
- `monitoring_healthcheck_report_dir: "{{ playbook_dir }}/../reports/monitoring-health"`

### playbooks/monitoring_healthcheck.yml
- `hosts: monitoring_servers` / `become: true` / `roles: [monitoring_healthcheck]`
- radius_healthcheck.yml と同じ構造

### playbooks/ubuntu_nightly.yml
- 既存の radius_servers 向け play はそのまま維持（play 名に "(radius_servers)" を追記）
- monitoring_servers 向けの play を末尾に追加
  - reboot 判定ロジックは radius_servers と同一
  - post-reboot サービス確認は prometheus / grafana-server / loki + 各 TCP ポート
  - 通知・レポート保存パターンは radius_servers と統一

### roles/systemd_timers/defaults/main.yml
- `monitoring-healthcheck` timer を追加（schedule: `*-*-* 05:35:00`、persistent: true）
