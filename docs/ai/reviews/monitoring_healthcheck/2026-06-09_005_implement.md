## 実装概要

**Shell 側（両スクリプト共通）**

`mem_line` が空でないことを確認した後、3 値を切り出してから以下の追加チェックを行う。

1. `mem_total_bytes`、`mem_used_bytes`、`mem_available_bytes` がいずれも非負整数（`^[0-9]+$`）であること
2. `mem_total_bytes` が 0 より大きいこと

すべての条件を満たした場合にのみ `mem_collection_ok="true"` をセットする。
いずれかが失敗した場合は `mem_collection_ok="false"` とし、3 値を 0 にリセットする。

```bash
if mem_line=$(free -b 2>/dev/null | awk '/^Mem:/{print $2, $3, $7}') && [ -n "$mem_line" ]; then
  mem_total_bytes=$(echo "$mem_line" | cut -d' ' -f1)
  mem_used_bytes=$(echo "$mem_line" | cut -d' ' -f2)
  mem_available_bytes=$(echo "$mem_line" | cut -d' ' -f3)
  if [[ "$mem_total_bytes" =~ ^[0-9]+$ ]] && \
     [[ "$mem_used_bytes" =~ ^[0-9]+$ ]] && \
     [[ "$mem_available_bytes" =~ ^[0-9]+$ ]] && \
     [ "$mem_total_bytes" -gt 0 ]; then
    mem_collection_ok="true"
  else
    mem_collection_ok="false"
    mem_total_bytes=0
    mem_used_bytes=0
    mem_available_bytes=0
  fi
else
  mem_collection_ok="false"
  mem_total_bytes=0
  mem_used_bytes=0
  mem_available_bytes=0
fi
```

これにより `free` が `Mem: 0 0 ...` や非数値を返した場合も `collection_ok=false` として
CRITICAL に分類され、正常扱いされない。

**Ansible 側（両 tasks/main.yml）**

変更なし。前回の対応で `collection_ok` の CRITICAL 分類はすでに実装済み。

## 注意点

- `[[ =~ ]]` は bash 固有の構文。`#!/bin/bash` を使用しているため問題ない。
- `[ "$mem_total_bytes" -gt 0 ]` は regex チェック通過後に実行されるため、常に有効な整数文字列に対して評価される。
- Python 側の `int()` はすでに数値保証済みの値を受け取るため、例外は発生しない。
