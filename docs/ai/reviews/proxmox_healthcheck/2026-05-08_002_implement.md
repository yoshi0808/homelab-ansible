## 変更内容の要約

### proxmox-healthcheck.sh

bash で以下を収集し、Python3 ヒアドキュメントで JSON 化して stdout に出力する。

| 収集項目 | コマンド |
|---|---|
| PVEバージョン | `pveversion` |
| クラスタ・quorum | `pvecm status` + `Quorate:` 行を抽出 |
| ZFSプール状態 | `zpool status` + pool/state 行をパース |
| replication | `pvesr status` + 日時パターンで last_sync_epoch (Unix秒) を抽出 |
| systemd failed | `systemctl --failed --no-legend --no-pager` |
| 重要サービス | `systemctl is-active` × 4 |
| root FS 使用率 | `df -P /` |
| apt 整合性 | `apt-get check` (exit code を apt_check_ok に変換) |
| reboot-required | `/var/run/reboot-required` の有無 |
| VM / CT 一覧 | `qm list` / `pct list` |
| 時刻同期 | `chronyc tracking` |

shell は収集と JSON 整形のみ。判定・分類・保存は行わない。

### defaults/main.yml

```yaml
proxmox_healthcheck_report_dir: "{{ playbook_dir }}/../reports/proxmox-health"
proxmox_healthcheck_root_fs_critical_pct: 90
proxmox_healthcheck_root_fs_warning_pct: 80
proxmox_healthcheck_repl_critical_hours: 24
proxmox_healthcheck_repl_warning_hours: 2
```

host_vars で個別上書き可能。

### tasks/main.yml

1. スクリプトを `/usr/local/sbin/proxmox-healthcheck.sh` に配置
2. `command` で実行、stdout を JSON parse
3. replication 閾値 (epoch) を `set_fact` で計算
4. `selectattr` / `rejectattr` で stale な replication ジョブを抽出
5. DEGRADED/FAULTED ZFS プールを抽出
6. CRITICAL / WARNING リストを Jinja2 で構築
7. レポート dict を構築し `reports/proxmox-health/` に JSON 保存
8. サマリーを `debug` で表示
9. criticals あれば `fail` でplaybook 停止

### CRITICAL 判定条件

| 条件 | 内容 |
|---|---|
| `cluster.quorate != 'Yes'` | quorum なし |
| ZFS pool が DEGRADED または FAULTED | プール異常 |
| `apt.check_ok == false` | apt/dpkg 破綻 |
| 重要サービス (`pve_cluster`/`corosync`/`pvedaemon`/`pveproxy`) が active でない | サービス停止 |
| `systemd.failed_count > 0` | systemd failed unit あり |
| `root_used_pct >= 90` | root FS 90%超 |
| replication entries に `last_sync_epoch < (now - 24h)` | 24時間以上未実行（-1 = 未実行も含む） |

### WARNING 判定条件

| 条件 | 内容 |
|---|---|
| replication entries に `last_sync_epoch < (now - 2h)` かつ `>= (now - 24h)` | 2時間以上未実行 |
| `reboot_required == true` | reboot-required フラグあり |
| `80 <= root_used_pct < 90` | root FS 80%超 |

## 注意点

- `pvesr status` の出力フォーマットは Proxmox バージョンにより微妙に異なる場合がある。`last_sync_epoch = -1` (N/A) のエントリが存在する場合は CRITICAL 扱い（設定済みだが未実行とみなす）。
- replication ジョブが 1 件も設定されていない場合 (`entries: []`) は replication チェックをスキップする。
- `apt-get check` は Proxmox ノードで実行するため、apt の proxy 設定に依存しない（ネットワーク不要）。
- `ansible_date_time.epoch` は UTC epoch。`pvesr status` の Last Sync はノードのローカル時刻。Python の `datetime.timestamp()` はローカル TZ を使うため、ノードの TZ 設定が正しい前提。
