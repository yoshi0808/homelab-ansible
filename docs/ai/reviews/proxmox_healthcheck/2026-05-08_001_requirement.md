# proxmox_healthcheck 要求仕様

## 目的
pve1/pve2 がパッチ適用可能な健康状態かを確認する。
daily監視 および patch前後の GO/NO-GO 判断に使う。

## 対象
- グループ: proxmox
- --limit による単一ノード実行も可

## 確認項目（shell が収集）
- pveversion
- pvecm status（クラスタ・quorum状態）
- zpool status（ZFSプール健全性）
- pvesr status（replication状態・最終実行時刻）
- systemctl --failed
- 重要サービス状態（pve-cluster/corosync/pvedaemon/pveproxy）
- root filesystem使用率
- apt-get check（apt/dpkg破綻確認）
- /var/run/reboot-required の有無
- VM/CT の所在と稼働状態（qm list / pct list）
- chrony 時刻同期状態

## 判定（Ansible側）
CRITICAL:
  - quorumなし
  - ZFS DEGRADED/FAULTED
  - apt/dpkg異常
  - 重要service停止
  - systemd failed unitsあり
  - root filesystem 90%以上
  - replication最終実行から24時間以上経過

WARNING:
  - replication最終実行から2時間以上経過
  - reboot-required あり
  - root filesystem 80%以上（かつ90%未満）

## CRITICAL時の動作
- ansible.builtin.fail でplaybookを停止する
- 後続のpatch playbookに進まないことを保証する

## 判定閾値
- defaults/main.yml に定義する
- host_vars で上書き可能にする

## 制約
- 変更操作なし（read only）
- レポートは reports/proxmox-health/ に保存

## 初回除外
- メール通知（未準備）
- QDevice状態（quory未着）
- VM内部状態（Guest Agent経由）
