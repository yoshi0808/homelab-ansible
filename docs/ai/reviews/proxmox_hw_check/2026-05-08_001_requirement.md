# proxmox_hw_check 要求仕様

## 目的
pve1/pve2 の物理ハードウェア構成を確認・記録する。
構成変更後・パッチ前の基準値記録に使う。

## 対象
- グループ: proxmox

## 確認項目

### 記録のみ（判定なし）
- CPU情報（モデル・コア数）
- メモリ総量
- ZFSプール構成
- ZFS scrub最終実行日時
- ディスク使用量

### 判定あり
- NICリンク状態（downならWARNING）
- NICエラーカウンタ（閾値超えでWARNING）

## 制約
- 変更操作なし（read only）
- レポートは reports/proxmox-hardware/ に保存

## 初回除外
- CPU温度
- SMART情報
- NICファームウェアバージョン
- メール通知

## 前提
- core.md のルールに従う
- 既存の roles/proxmox_healthcheck/ の構造を参考にする
- shell は収集とJSON整形のみ、判断はAnsible側
