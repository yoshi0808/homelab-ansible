## 変更内容の要約

### proxmox-hw-check.sh

bash で以下を収集し、Python3 ヒアドキュメントで JSON 化して stdout に出力する。

| 収集項目 | コマンド | 備考 |
|---|---|---|
| CPUモデル・構成 | `lscpu` | model, sockets, cores/socket, threads/core, total CPUs |
| メモリ総量 | `/proc/meminfo` | MemTotal を MB換算 |
| ZFSプール構成 | `zpool list -H -o name,size,alloc,free,cap,health` | プールごとのサイズ・使用率・健全性 |
| ZFS scrub情報 | `zpool status` | poolごとの `scan:` 行を抽出 |
| ディスク使用量 | `df -Ph` | 全マウントポイント（POSIX形式） |
| NICリンク状態・エラーカウンタ | `ip -s link show` | `lo`・`veth*`・`tap*`・`fwbr*`・`fwln*`・`dummy*` を除外 |

NIC ごとに `total_errors = rx_errors + tx_errors`、`total_dropped = rx_dropped + tx_dropped` を Python 側で計算してフィールドに追加。Ansible が `selectattr` で直接比較できるようにする。

### defaults/main.yml

```yaml
proxmox_hw_check_report_dir: "{{ playbook_dir }}/../reports/proxmox-hardware"
proxmox_hw_check_nic_errors_warning_threshold: 100
proxmox_hw_check_nic_dropped_warning_threshold: 1000
```

host_vars で個別上書き可能。

### tasks/main.yml

1. スクリプトを `/usr/local/sbin/proxmox-hw-check.sh` に配置
2. `command` で実行、stdout を JSON parse
3. `state == 'DOWN'` の NIC を抽出 → WARNING
4. `total_errors > threshold` の NIC を抽出 → WARNING
5. `total_dropped > threshold` の NIC を抽出 → WARNING
6. warnings リストを構築（CRITICAL なし）
7. レポート dict を構築し `reports/proxmox-hardware/` に JSON 保存
8. サマリーを `debug` で表示
9. fail タスクなし（WARNING のみの role のため）

### 判定条件

| 条件 | レベル |
|---|---|
| NIC の state が `DOWN` | WARNING |
| NIC の `total_errors > 100`（デフォルト） | WARNING |
| NIC の `total_dropped > 1000`（デフォルト） | WARNING |
| CPU/メモリ/ZFS/ディスク | 記録のみ（判定なし） |

ZFS pool の health は記録するが、異常判定は proxmox_healthcheck role の責務。
本 role はハードウェア棚卸し・基準値記録が目的のため fail タスクを持たない。

## 注意点

- `ip -s link show` の RX/TX 統計フォーマットは iproute2 バージョンにより微妙に異なる場合がある。パーサーはヘッダ行（`RX:`/`TX:`）の次行を値行として読むステートマシン方式で、フォーマット変化に対して比較的ロバスト。
- `df -Ph` で filesystem 名が長い場合、POSIX 形式では名前が1行、サイズ等が次行に分かれる。パーサーはその両パターンを処理する。
- NIC の `UNKNOWN` 状態（一部のブリッジ等で発生）は WARNING 対象外。`DOWN` のみを警告する。
- `vmbr*` (Proxmox bridge), `bond*` は収集対象に含まれる。
