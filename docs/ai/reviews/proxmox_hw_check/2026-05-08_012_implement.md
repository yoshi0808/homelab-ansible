## 変更内容の詳細

### tasks/main.yml（Medium 対応）

`total_packets == 0` の NIC は rate 判定を行わないよう、`item.total_packets | int > 0` を条件に追加した。

```yaml
# 変更前
{{ _nics_errors + ([item.name] if
   (item.total_errors | float / (item.total_packets | int + 1) * 100)
   > (proxmox_hw_check_nic_errors_warning_pct | float)
   else []) }}

# 変更後
{{ _nics_errors + ([item.name] if
   (item.total_packets | int > 0) and
   ((item.total_errors | float / (item.total_packets | int + 1) * 100)
   > (proxmox_hw_check_nic_errors_warning_pct | float))
   else []) }}
```

`_nics_dropped` も同様に修正。

Jinja2 の演算子優先順位上、`and` の両辺の比較式をそれぞれ括弧でくくっている。

### 007_requirement.md（制約の記述修正）

変更前:
```
パケット数ゼロの NIC（未使用 NIC）は分母 `+1` によりゼロ除算を回避し、WARNING を出さない
```

変更後:
```
`total_packets == 0` の NIC（未使用 NIC）は rate 判定の対象外とし、WARNING を出さない
（`total_packets > 0` の NIC のみ判定する）
分母の `+1` はゼロ除算防止のための保護コードとして維持する
```
