# proxmox_hw_check 仕様変更要求仕様（NIC 判定を割合ベースに変更）

## 目的

NIC の error / drop 判定を絶対値閾値から、総パケット数に対する割合（%）ベースの判定に変更する。
絶対値比較では traffic 量の差（高速物理 NIC と低速 VM NIC）を考慮できないため、割合ベースに切り替える。

## 対象

- グループ: proxmox（pve1 / pve2）
- 変更対象ファイル:
  - `roles/proxmox_hw_check/files/proxmox-hw-check.sh`
  - `roles/proxmox_hw_check/defaults/main.yml`
  - `roles/proxmox_hw_check/tasks/main.yml`

## 変更する判定仕様

### NIC エラー判定

| 項目 | 仕様 |
|---|---|
| 判定方式 | `total_errors / (total_packets + 1) * 100` が閾値（%）を超えた場合に WARNING |
| デフォルト閾値 | `proxmox_hw_check_nic_errors_warning_pct: 0.1`（0.1%超で WARNING） |
| ゼロ除算対策 | 分母に `+1` する |
| host_vars 上書き | 可 |

### NIC ドロップ判定

| 項目 | 仕様 |
|---|---|
| 判定方式 | `total_dropped / (total_packets + 1) * 100` が閾値（%）を超えた場合に WARNING |
| デフォルト閾値 | `proxmox_hw_check_nic_dropped_warning_pct: 0.5`（0.5%超で WARNING） |
| ゼロ除算対策 | 分母に `+1` する |
| host_vars 上書き | 可 |

### 判定対象 NIC

- 変更なし。`lo` / `veth*` / `tap*` / `fwbr*` / `fwln*` / `fwpr*` / `dummy*` を除外し、`vmbr*` / `bond*` / 物理 NIC を対象とする。

## 責務分離

- shell: `rx_packets` / `tx_packets` / `total_packets` を収集し JSON に出力する（データ収集のみ）
- Ansible tasks: `total_packets` を分母として割合を計算し、WARNING 判定を行う

## 制約

- 変更操作なし（read only）
- shell に判定ロジックを持たせない
- `total_packets == 0` の NIC（未使用 NIC）は rate 判定の対象外とし、WARNING を出さない（`total_packets > 0` の NIC のみ判定する）
- 分母の `+1` はゼロ除算防止のための保護コードとして維持する

## 初回除外（引き続き対象外）

- CPU温度
- SMART情報
- NICファームウェアバージョン
- メール通知
