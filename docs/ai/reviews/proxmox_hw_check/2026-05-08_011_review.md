## Findings

1. Medium: `total_packets == 0` でも error/drop があれば WARNING になる

   - 該当箇所: `docs/ai/reviews/proxmox_hw_check/2026-05-08_007_requirement.md:45-49`, `roles/proxmox_hw_check/tasks/main.yml:43-58`
   - requirement では「パケット数ゼロの NIC（未使用 NIC）は分母 `+1` によりゼロ除算を回避し、WARNING を出さない」と明記されている。
   - しかし実装は `(item.total_errors / (item.total_packets + 1) * 100) > threshold` と `(item.total_dropped / (item.total_packets + 1) * 100) > threshold` の比較のみで、`total_packets > 0` の条件がない。
   - そのため、例えば `total_packets: 0`, `total_dropped: 1` の NIC は `1 / (0 + 1) * 100 = 100%` となり WARNING になる。要件どおり「パケット数ゼロなら WARNING を出さない」にするなら、error/drop rate 判定に `item.total_packets | int > 0` を追加する必要がある。
