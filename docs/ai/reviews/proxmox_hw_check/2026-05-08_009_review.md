1. Medium: requirement が割合ベース判定への仕様変更を表現できていない

   - 該当箇所: `docs/ai/reviews/proxmox_hw_check/2026-05-08_007_requirement.md:19-21`, `docs/ai/reviews/proxmox_hw_check/2026-05-08_008_implement.md:5-7`, `roles/proxmox_hw_check/defaults/main.yml:3-4`

   - このままだと、後から読むと「カウンタの絶対値を閾値比較する仕様」にも読める。今回の変更を正にするなら、requirement 側に `error/drop は total_packets に対する割合で判定する`, `errors warning pct = 0.1`, `dropped warning pct = 0.5` のように明記した方がよい。

2. Low: implement の Next step files が実際の連番とずれている

   - 実装動作には影響しないが、core.md のレビュー運用記録としては修正した方が追いやすい。

## 良い点

- shell は `rx_packets` / `tx_packets` / `total_packets` を観測値として追加するだけで、判定は Ansible tasks 側に残っている。
- `ip -s link show` の `bytes packets errors dropped` 列に対して `vals[1]` / `vals[2]` / `vals[3]` を読む実装は妥当。
- NIC 収集失敗時は `proxmox_hw.nics | default([])` により rate 判定ループは空で終わり、収集失敗自体は `_collection_failures` の WARNING として扱われる。
- 構文確認と Ansible の式評価は通っている。


結果:

- shell 構文チェックは成功。
- playbook syntax-check は成功。

- Ansible/Jinja の list 加算式は期待通り `["eth0"]` を返すことを確認した。
