# proxmox_hw_check 追加実装内容（レビュー指摘対応）

## 対応したレビュー指摘

### High: NIC 収集失敗が OK 扱いになる（レビュー Findings 1）

- `ip -s link show` が失敗すると `nics: []` が返り、NIC チェックがすべて素通りしていた。
- 修正: shell が `collection_status.nics` を JSON に出力するようにし、Ansible が false の場合に WARNING を生成する。

### Medium: ZFS / disk 収集失敗が空配列として保存される（レビュー Findings 2）

- `zpool list` / `zpool status` / `df` の失敗と「プールなし・ファイルシステムなし」が区別できなかった。
- 修正: `collection_status.zfs_pools` / `collection_status.zfs_scrub` / `collection_status.disk` を追加。収集失敗時に Ansible が WARNING を生成する。

### Medium: `fwpr*` インターフェースが除外されていない（レビュー Findings 3）

- Proxmox firewall が有効な VM/CT で `fwpr*` インターフェースが作られるが、除外リストに入っていなかった。
- 修正: `excluded_prefixes` に `'fwpr'` を追加。

### tasks/main.yml

#### Find collection failures タスクを追加（Parse hw check JSON の直後）

```yaml
- name: Find collection failures
  ansible.builtin.set_fact:
    _collection_failures: "{{ []
      + ([] if proxmox_hw.collection_status.nics else ['NIC collection failed: ip -s link show returned unavailable'])
      + ([] if proxmox_hw.collection_status.zfs_pools else ['ZFS pool collection failed: zpool list returned unavailable'])
      + ([] if proxmox_hw.collection_status.zfs_scrub else ['ZFS scrub collection failed: zpool status returned unavailable'])
      + ([] if proxmox_hw.collection_status.disk else ['Disk collection failed: df returned unavailable'])
      }}"
```

#### Classify warnings に _collection_failures を追加

```yaml
proxmox_hw_warnings: "{{ []

  }}"
```

## 未対応事項

- `datetime.utcnow()` の DeprecationWarning は Python 3.14 環境のみで発生し stdout JSON に影響しない。今回は対応範囲外。
