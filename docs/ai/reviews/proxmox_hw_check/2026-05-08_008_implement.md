# proxmox_hw_check 追加実装内容（NIC 判定を割合ベースに変更）

## 実装概要

NIC の error / drop 判定を絶対値比較から割合（%）ベースの比較に変更した。
割合の計算は Ansible tasks 側で実施し、shell は収集のみを担う方針を維持している。
パーセンテージ計算に必要な `total_packets` を shell が新たに収集し、判定は Ansible 側で行う。

## 変更内容の詳細

### proxmox-hw-check.sh

`ip -s link show` の出力列は `bytes packets errors dropped ...` の順。
パーセンテージ計算に `packets`（vals[1]）が必要なため、収集対象に追加した。

NIC 初期化フィールドに追加:

```python
"rx_packets": 0,
"tx_packets": 0,
```

RX / TX 値読み取り時に `vals[1]` を収集:

```python
current_nic['rx_packets'] = safe_int(vals[1])
current_nic['tx_packets'] = safe_int(vals[1])
```

集計フィールドに `total_packets` を追加:

```python
nic['total_packets'] = nic['rx_packets'] + nic['tx_packets']
```

### defaults/main.yml

```yaml
# 変更前
proxmox_hw_check_nic_errors_warning_threshold: 100
proxmox_hw_check_nic_dropped_warning_threshold: 1000

# 変更後
proxmox_hw_check_nic_errors_warning_pct: 0.1
proxmox_hw_check_nic_dropped_warning_pct: 0.5
```

host_vars で個別上書き可能。

### tasks/main.yml

「Find NICs with high error counts」1タスクを、以下の3タスクに置き換えた。

```yaml
- name: Initialize NIC rate check results
  ansible.builtin.set_fact:
    _nics_errors: []
    _nics_dropped: []

- name: Check NIC error rates
  ansible.builtin.set_fact:
    _nics_errors: >-
      {{ _nics_errors + ([item.name] if
         (item.total_errors | float / (item.total_packets | int + 1) * 100)
         > (proxmox_hw_check_nic_errors_warning_pct | float)
         else []) }}
  loop: "{{ proxmox_hw.nics | default([]) }}"

- name: Check NIC drop rates
  ansible.builtin.set_fact:
    _nics_dropped: >-
      {{ _nics_dropped + ([item.name] if
         (item.total_dropped | float / (item.total_packets | int + 1) * 100)
         > (proxmox_hw_check_nic_dropped_warning_pct | float)
         else []) }}
  loop: "{{ proxmox_hw.nics | default([]) }}"
```

計算式: `drop_pct = total_dropped / (total_packets + 1) * 100`  
分母に `+1` してゼロ除算を防ぐ。

WARNING メッセージも更新:
- `NIC \1 error count exceeds threshold` → `NIC \1 error rate exceeds threshold`
- `NIC \1 dropped count exceeds threshold` → `NIC \1 drop rate exceeds threshold`

## 実行した確認コマンドと結果

```bash
bash -n roles/proxmox_hw_check/files/proxmox-hw-check.sh
# → syntax OK

ANSIBLE_LOCAL_TEMP=/tmp/ansible-local ANSIBLE_REMOTE_TEMP=/tmp/ansible-remote \
  ansible-playbook --syntax-check -i inventories/homelab/hosts.yml playbooks/proxmox_hw_check.yml
# → playbook: playbooks/proxmox_hw_check.yml  (syntax OK)

bash roles/proxmox_hw_check/files/proxmox-hw-check.sh 2>/dev/null | python3 -c "
import json, sys
d = json.load(sys.stdin)
for nic in d['nics']:
    tp = nic['total_packets']
    te = nic['total_errors']
    td = nic['total_dropped']
    print(f\"{nic['name']}: packets={tp} errors={te} dropped={td} error_pct={te/(tp+1)*100:.4f}% drop_pct={td/(tp+1)*100:.4f}%\")
"
# → eth0: packets=691492 errors=0 dropped=41541 error_pct=0.0000% drop_pct=6.0074%
```

ansy の eth0 で `total_packets` / `total_dropped` が正しく収集され、パーセンテージが計算できることを確認。
pve1 / pve2 の物理 NIC では通常 drop は 0% 付近のため、デフォルト閾値 0.5% は適切な水準。

## 注意点

- `set_fact` をループで使う場合、ループ前に変数を `[]` で初期化しないと前回 playbook 実行の値が残る可能性がある。`Initialize NIC rate check results` タスクがこれを担っている。
- `proxmox_hw.nics | default([])` により、NIC 収集失敗（空リスト）時はループがスキップされ `_nics_errors` / `_nics_dropped` は `[]` のまま維持される。収集失敗自体は `_collection_failures` で別途 WARNING として扱われる。
