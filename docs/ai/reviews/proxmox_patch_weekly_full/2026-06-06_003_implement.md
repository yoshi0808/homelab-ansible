# implement: preflight pve1/pve2直接実行チェック追加 + Policy前提追記

## 実装概要

`2026-06-06_002_review.md` の High 指摘に対応した。

### High 指摘の内容

preflight が `_cluster_vm_names`（QEMU VM 名リスト）に対してのみチェックするため、
pve1 / pve2 自身から実行した場合に通過してしまうという指摘。

pve1 / pve2 は Proxmox 物理ノードであり QEMU VM ではないため、
`_cluster_vm_names` に含まれず、preflight を誤通過する。

「pve からの ansible 実行は現実としてしないので、
Policy に ansible 端末は ansy または quory という前提を記載しても良い」

→ `proxmox_patch_policy.md` セクション 16.8 に前提節を追加した。

コード側でも pve1/pve2 直接実行を検出して abort するチェックを追加し、
Policy と実装を一致させた。

## 変更内容の要約

### proxmox_patch_weekly_full.yml

Step 0 に `Abort if executing host is a Proxmox node` タスクを追加した。

```yaml
- name: Abort if executing host is a Proxmox node
  ansible.builtin.fail:
    msg: >-
      Executing host '{{ _controller_hostname }}' is a Proxmox node.
      proxmox_patch_weekly_full must be run from an external control node
      (e.g. quory), not from pve1 or pve2 themselves.
  when: _controller_hostname in groups['proxmox']
```

これにより、preflight は以下の2段階で実行ホストを検証する：

| チェック | 拒否条件 | 対象ケース |
| --- | --- | --- |
| cluster VM チェック | `_controller_hostname in _cluster_vm_names` | ansy 等のクラスタ上の QEMU VM |
| Proxmox node チェック | `_controller_hostname in groups['proxmox']` | pve1 / pve2 自身から実行 |

### proxmox_patch_policy.md

セクション 16.8 の冒頭（16.8.1 の前）に **16.8.0 Ansible 実行端末の前提** を追加した。

内容：

- ansy（Proxmox 上の VM）と quory（クラスタ外物理ノード）のみが Ansible 実行端末
- 管理対象ホスト（pve1 / pve2 / authy / monnie 等）自身では実行しない
- weekly full playbook は preflight でも pve1 / pve2 からの実行を拒否することを明記

## 注意点
