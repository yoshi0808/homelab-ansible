# 実装記録 2026-06-06_019

日付: 2026-06-06  
対応: NO_UPDATES 経路で `proxmox_restore_vm_placement` の loop タスクが `'item' is undefined` エラーを出す問題

---

## 問題の概要

`proxmox_patch_weekly_full.yml` から `import_playbook: proxmox_restore_vm_placement.yml` を `when: not _weekly_full_no_updates` でインポートしているにもかかわらず、NO_UPDATES 経路で以下のエラーが発生していた。

```
fatal: [pve1 -> {{ item.node }}]: FAILED! => {"msg": "Task failed: 'item' is undefined"}
```

エラーは `roles/proxmox_restore_vm_placement/tasks/main.yml` の以下のタスクで発生：

```yaml
- name: Migrate non-HA QEMU VMs back to target_node
  loop: "{{ _vms_to_restore | selectattr('type', 'equalto', 'qemu') | list }}"
  delegate_to: "{{ item.node }}"   ← ループ外コンテキストで item が未定義

- name: Migrate non-HA LXC containers back to target_node
  loop: "{{ _vms_to_restore | selectattr('type', 'equalto', 'lxc') | list }}"
  delegate_to: "{{ item.node }}"   ← 同上
```

### 原因

`import_playbook` の `when:` は Ansible 2.14+ でサポートされているが、`delegate_to: "{{ item.node }}"` を持つ loop タスクのコンパイル時に `item` が未定義のスコープで評価されることがあり、`import_playbook when:` だけでは確実に防げない。

---

## 修正内容

`playbooks/proxmox_restore_vm_placement.yml` の第2 play を修正した。

### 変更前

```yaml
- name: Restore VM placement to target_node
  hosts: "{{ target_node }}"
  gather_facts: true
  become: true
  roles:
    - proxmox_restore_vm_placement
```

### 変更後

```yaml
- name: Restore VM placement to target_node
  hosts: "{{ target_node }}"
  gather_facts: false
  become: true
  tasks:
    - name: Skip if called from weekly-full and no updates available
      ansible.builtin.meta: end_play
      when: hostvars['localhost']['_weekly_full_no_updates'] | default(false)

    - name: Gather facts
      ansible.builtin.setup:

    - name: Run restore VM placement role
      ansible.builtin.include_role:
        name: proxmox_restore_vm_placement
```

### 変更点の説明

| 変更 | 理由 |
|---|---|
| `gather_facts: true` → `gather_facts: false` | skip guard より先に SSH 接続・fact 収集が走るのを防ぐ |
| `roles:` → `tasks:` + `include_role:` | skip guard タスクを role 実行より前に置けるようにする |
| skip guard (meta: end_play) を先頭に追加 | NO_UPDATES 経路で role の loop タスクに到達する前に play を終了させる |
| `ansible.builtin.setup:` を明示追加 | `proxmox_healthcheck` role (role 内 Phase 4 で呼ばれる) が必要とする ansible_facts を収集する |

### standalone 実行への影響

`proxmox_restore_vm_placement.yml` を単体で実行する場合、`hostvars['localhost']['_weekly_full_no_updates']` は存在しないため `default(false)` が適用され、skip guard は発動しない。動作は従来と同じ。

## NO_UPDATES 経路での動作（修正後）

| ステップ | 動作 |
|---|---|
| Step 2b | `_weekly_full_no_updates: true` をセット |
| Step 6 (`import_playbook: proxmox_restore_vm_placement.yml`) | `when:` で skip（かつ play 内 skip guard も保険として動作） |
| Step 6 内 play 1 (localhost validate) | 実行されても assert のみ、副作用なし |
