## 変更内容の要約

### 1. `hostname -s` への変更

`lookup('pipe', 'hostname')` を `lookup('pipe', 'hostname -s')` に変更。

FQDN（例: `quory.internal`）を返す環境でも short hostname（`quory`）として扱うようにした。
既存の `_cluster_vm_names` チェックおよび `groups['proxmox']` チェックも short hostname を前提にしているため、一貫した動作になる。

### 2. `proxmox_patch_weekly_full_allowed_controllers` 変数の追加

preflight play の `vars` セクションに以下を追加した。

```yaml
vars:
  proxmox_patch_weekly_full_allowed_controllers:
    - quory
```

デフォルトは `['quory']`。必要に応じて実行時変数で上書き可能。

### 3. allowlist チェックタスクの追加

既存の2つの abort タスク（クラスタ VM チェック、Proxmox ノードチェック）に続けて、3つ目のタスクを追加した。

```yaml
- name: Abort if executing host is not an allowed controller
  ansible.builtin.fail:
    msg: >-
      Executing host '{{ _controller_hostname }}' is not in the allowed
      controllers list ({{ proxmox_patch_weekly_full_allowed_controllers | join(', ') }}).
      proxmox_patch_weekly_full must be run from an allowed external control
      node. To override, set proxmox_patch_weekly_full_allowed_controllers.
  when: _controller_hostname not in proxmox_patch_weekly_full_allowed_controllers
```

---

## preflight の確認順序（変更後）

| チェック | 拒否条件 |
|---|---|
| 1. クラスタ VM チェック | `_controller_hostname` がクラスタ上の QEMU VM 名に含まれる |
| 2. Proxmox ノードチェック | `_controller_hostname` が `groups['proxmox']`（pve1/pve2）に含まれる |
| 3. allowlist チェック（新規） | `_controller_hostname` が `proxmox_patch_weekly_full_allowed_controllers` に含まれない |

チェック3により、クラスタ外だが quory でもない別の管理端末からの実行を拒否できる。

## 注意点

- `proxmox_patch_weekly_full_allowed_controllers` は play vars で定義している。将来的に quory 以外からも実行する必要が生じた場合は、この変数を `group_vars` や実行時変数で上書きすることで対応できる。
