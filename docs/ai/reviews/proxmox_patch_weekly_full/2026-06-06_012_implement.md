## 実装概要

`proxmox_patch_weekly_full.yml` の Step 2b で `dry-run Status = NO_UPDATES` の場合に `fail` になっていた問題を修正した。

`NO_UPDATES` はパッチが不要な正常状態であり、メール通知して正常終了するように変更した。`MAINTENANCE_REQUIRED` / `BLOCKED` / `MAJOR_UPGRADE_DETECTED` / `UNKNOWN` の場合のみ fail にする。

## 変更内容の要約

### 1. Step 2b の再設計

変更前:

```yaml
- name: Abort if dry-run status is not PATCH_READY
  hosts: localhost
  tasks:
    - name: Fail if not PATCH_READY
      ansible.builtin.fail:
        msg: "Dry-run status is {{ _status }}. ..."
      when: _status != 'PATCH_READY'
```

この実装では `NO_UPDATES` も `fail` になっていた。

変更後: 2つのタスクに分割した。

```yaml
- name: Handle dry-run status - notify NO_UPDATES or abort on error
  hosts: localhost
  vars:
    _status: "{{ hostvars[groups['proxmox'][0]]['_final_status'] | default('UNKNOWN') }}"
  tasks:
    - name: Notify NO_UPDATES and end playbook cleanly
      when: _status == 'NO_UPDATES'
      block:
        - name: Load mail variables / Send NO_UPDATES notification / Set skip flag / End play
          ...

    - name: Fail if dry-run status requires manual action or is unexpected
      ansible.builtin.fail:
      when: _status != 'PATCH_READY'
```

`NO_UPDATES` の場合:
1. `vars/mail.yml` を読み込む
2. メール通知（件名: `[Proxmox Patch] NO_UPDATES - 適用すべき更新なし`）を送信する
3. `_weekly_full_no_updates: true` を localhost の fact にセットする
4. `meta: end_play` でこの play を終了する

その後のタスクは `_status != 'PATCH_READY'` の場合のみ fail するが、`NO_UPDATES` は `meta: end_play` で到達しないため実行されない。

### 2. `import_playbook` 6 箇所への `when:` 追加

Ansible 2.14 以降では `import_playbook` に `when:` が使える（実環境: Ansible core 2.20.1）。

対象ステップ（Steps 3, 4, 6, 7, 8, 10）に以下を追加:

```yaml
when: not (hostvars['localhost']['_weekly_full_no_updates'] | default(false))
```

Step 2b で `_weekly_full_no_updates: true` がセットされた場合、これらの play がすべてスキップされる。

### 3. インライン play のスキップガード追加（Steps 5, 9）

`import_playbook` ではなく直接 play として記述されているステップ（pve2/pve1 の post-patch healthcheck）の冒頭に追加:

```yaml
- name: Skip if NO_UPDATES
  ansible.builtin.meta: end_play
  when: hostvars['localhost']['_weekly_full_no_updates'] | default(false)
```

### 4. Step 11 Summary mail のスキップガード追加

NO_UPDATES の通知は Step 2b で送信済みのため、Step 11 の summary mail は重複送信しない。

```yaml
- name: Skip if NO_UPDATES (notification already sent in Step 2b)
  ansible.builtin.meta: end_play
  when: _weekly_full_no_updates | default(false)
```

Step 11 は localhost を対象にしているため、`hostvars['localhost']['...']` ではなく変数名を直接参照している。

## Status ごとの動作まとめ

| Status | 動作 |
|---|---|
| `PATCH_READY` | そのまま Step 3（evacuate pve2）以降に進む |
| `NO_UPDATES` | メール通知して正常終了。Steps 3-11 はすべてスキップ |
| `MAINTENANCE_REQUIRED` | `fail`（手動対応が必要） |
| `BLOCKED` | `fail`（apt/dpkg 破綻。パッチ操作停止） |
| `MAJOR_UPGRADE_DETECTED` | `fail`（メジャーアップグレード疑い。手動対応） |
| `UNKNOWN` | `fail`（予期しない状態） |

## skip 機構の仕組み

```
Step 2b (localhost play)
  NO_UPDATES の場合:
    → メール送信
    → set_fact: _weekly_full_no_updates = true  ← localhost の hostvars に格納
    → meta: end_play                             ← この play を終了

Step 3-10 (import_playbook)
  when: not (hostvars['localhost']['_weekly_full_no_updates'] | default(false))
    → NO_UPDATES フラグが立っていればスキップ

Steps 5, 9 (inline play on pve2/pve1)
  タスク冒頭: meta: end_play when: hostvars['localhost']['_weekly_full_no_updates']...

Step 11 (localhost play)
  タスク冒頭: meta: end_play when: _weekly_full_no_updates | default(false)
```

`set_fact` で設定した fact は同一 playbook 実行内の後続 play からも `hostvars['localhost']['varname']` で参照できる。
