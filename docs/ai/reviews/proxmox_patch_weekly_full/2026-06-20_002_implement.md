# Implement: proxmox_patch_weekly_full — Step 2b スキップ拡張

## 実装概要

Step 2b（dry-run status 分岐）で、warning レベルのスキップステータス
`MAINTENANCE_REQUIRED` / `MAJOR_UPGRADE_DETECTED` を `NO_UPDATES` と同様に
「スキップフラグ → `meta: end_play`（正常終了 rc=0）」へ変更。
`BLOCKED` / `UNKNOWN` / 想定外は `fail` のまま。

## 作成・更新ファイル

- `playbooks/proxmox_patch_weekly_full.yml`（Step 2b のみ）
- `docs/ai/reviews/proxmox_patch_weekly_full/2026-06-20_001_requirement.md`（要求仕様）

## 変更内容

1. clean-skip 対象を `_weekly_full_skip_statuses`
   （`NO_UPDATES` / `MAINTENANCE_REQUIRED` / `MAJOR_UPGRADE_DETECTED`）として定義。
   - 該当時: `_weekly_full_skip: true` を set_fact → `meta: end_play`（正常終了）。
   - `fail` の `when` は `_status != 'PATCH_READY'` のまま。skip 該当は end_play で
     先に抜けるため、結果として `BLOCKED` / `UNKNOWN` / 想定外のみ fail になる。
2. スキップフラグを `_weekly_full_no_updates` → `_weekly_full_skip` にリネーム。
   `NO_UPDATES` 以外もスキップ対象になったため意味を正す。downstream 9 箇所も追従。
3. Step 2b のコメントを新しい status 対応へ更新。
