## 変更内容の要約

### override 例を JSON 形式に変更

変更前:

```
-e 'proxmox_patch_weekly_full_allowed_controllers=["<hostname>"]'
```

変更後:

```
-e '{"proxmox_patch_weekly_full_allowed_controllers":["<hostname>"]}'
```

#### 背景

Ansible の `-e key=value` 形式では、`value` が `["quory"]` のような JSON 配列リテラルに見えても、文字列として解釈される。

```bash
# 確認コマンド（レビューによる検証結果）
ansible localhost -m debug -a 'msg={{ x | type_debug }}:{{ x }}' -e 'x=["quory"]'
# 結果: str:["quory"]
```

`when: _controller_hostname not in proxmox_patch_weekly_full_allowed_controllers` は、右辺が文字列の場合 Jinja2 の substring 判定として動作する。このため単一ホスト名では偶然通過する可能性があり、型の不一致が潜在的な誤動作の原因になり得る。

JSON 形式の extra vars（`-e '{...}'`）であれば Ansible はリストとして正しく解釈する。

```bash
# JSON 形式では list 型になる
ansible localhost -m debug -a 'msg={{ x | type_debug }}:{{ x }}' \
  -e '{"x":["quory"]}'
# 結果: list:['quory']
```
