# Requirement: proxmox_patch_weekly_full — dry-run スキップ時のジョブ終了コード調整

## 目的

`proxmox_patch_weekly_full` で、**パッチ適用を意図的にスキップした「高リスク」ステータス**
（dry-run ロールが warning として Slack 通知済み）のとき、ジョブを `fail` ではなく
**正常終了（rc=0）** させる。

Semaphore はジョブ結果を success/fail でしか扱えず（warning なし）、fail だと実際には
「壊れていない・正しくスキップした」のに Slack に ❌Error 通知が出てしまうため。

## 背景

- Slack 通知は Semaphore がジョブ終了結果（success/fail）を見て出している。
- dry-run ロール `proxmox_patch_dryrun` は `_final_status` ごとに自前で Slack 通知の
  重大度を決めている（`common_slack/tasks/notify.yml`）:
  - `MAINTENANCE_REQUIRED` / `MAJOR_UPGRADE_DETECTED` → warning ⚠️
  - `BLOCKED` → critical 🔴 / `NO_UPDATES` → info
- 一方 weekly_full の Step 2b は `PATCH_READY` と `NO_UPDATES` 以外を一律 `fail` にしていた
  ため、warning レベルのスキップでもジョブ失敗 → Semaphore が ❌Error を出していた。

## 対象

- 対象 playbook: `playbooks/proxmox_patch_weekly_full.yml`（**Step 2b のみ**）
- 対象グループ: 既存どおり（proxmox / localhost）。変更なし。

## 実施内容（status → ジョブ終了の対応）

| status | 変更後 |
|---|---|
| `PATCH_READY` | 続行（変更なし） |
| `NO_UPDATES` | スキップ＋正常終了（変更なし） |
| `MAINTENANCE_REQUIRED` | **fail → スキップ＋正常終了** |
| `MAJOR_UPGRADE_DETECTED` | **fail → スキップ＋正常終了** |
| `BLOCKED` | **fail のまま**（apt/dpkg 破損は本物の異常） |
| `UNKNOWN` / その他 | **fail のまま** |

- 「通知を行った上で正常終了」は、dry-run ロールが該当ステータスで既に warning 通知を
  送っているため、weekly_full 側で通知を追加する必要はない。**Step 2b の終了挙動のみ変更**する。

## 制約・スコープ

- **VM 構成チェック**（evacuate / restore_vm_placement 等、別 play）は**変更しない** → Error のまま。
- read-only 化ではない。変更系 playbook の終了コード制御のみ。
- 秘密情報は扱わない。shell / script は変更しない（判定は Ansible tasks 側）。

## 初回で含める / 除外する

- 含める: Step 2b の clean-skip 対象ステータス拡張、スキップフラグ名の明確化。
- 除外: dry-run ロールの通知ロジック、VM 構成チェック、その他 play。
