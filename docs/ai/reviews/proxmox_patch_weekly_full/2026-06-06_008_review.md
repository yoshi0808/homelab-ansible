## Findings

### Low: エラーメッセージの `-e` 例はリストではなく文字列として解釈される

- 該当:
  - `playbooks/proxmox_patch_weekly_full.yml:77-85`
  - `docs/ai/reviews/proxmox_patch_weekly_full/2026-06-06_007_implement.md`
- 状況:
  - 今回の修正で、allowlist の上書き方法が `extra vars` であることは明確になった。
  - ただしメッセージに示されている `-e 'proxmox_patch_weekly_full_allowed_controllers=["<hostname>"]'` は、Ansible の `key=value` 形式として渡すとリストではなく文字列になる。
  - ローカル確認でも `-e 'x=["quory"]'` は `x | type_debug == str` になった。
- 影響:
  - 実際の `when: _controller_hostname not in proxmox_patch_weekly_full_allowed_controllers` は、右辺が文字列でも substring 判定として動くため、単一ホスト名では偶然通過する可能性がある。
  - しかし変数型は意図した list ではなく、複数 controller や将来の厳密な型チェックを追加した場合に誤解・不具合の元になる。
  - `join(', ')` も文字列に対しては文字単位 join になり、エラーメッセージが読みづらくなる可能性がある。
- 対応案:
  - メッセージの例を JSON 形式の extra vars に変更する。
  - 例:
    ```bash
    -e '{"proxmox_patch_weekly_full_allowed_controllers":["<hostname>"]}'
    ```
