## 変更内容の要約

### エラーメッセージの明確化

変更前:

```
To override, set proxmox_patch_weekly_full_allowed_controllers.
```

変更後:

```
To override, use extra vars:
-e 'proxmox_patch_weekly_full_allowed_controllers=["<hostname>"]'
```

#### 背景

`proxmox_patch_weekly_full_allowed_controllers` は preflight play の `vars` ブロックに定義されている。

Ansible の変数優先順位では play vars は `group_vars` や `host_vars` より高いため、inventory 側から上書きすることができない。

上書きできるのは play vars より優先順位が高い extra vars (`-e`) のみ。

変更前のメッセージは「set」とだけ書いており、`group_vars` でも設定できると誤解させる余地があった。変更後は `-e` での上書き方法を具体的に示している。

---

## 変数優先順位の整理（参考）

| 定義場所 | 優先順位 | 上書き可否 |
|---|---|---|
| play `vars` | 高い | ─（定義側） |
| `group_vars` | play vars より低い | 不可 |
| `host_vars` | play vars より低い | 不可 |
| extra vars (`-e`) | 最高 | 可 |

quory 専用固定を強く維持するために、変数は引き続き play `vars` で定義する。
将来 quory 以外の controller を許可する必要が生じた場合は `-e` で上書きする。
