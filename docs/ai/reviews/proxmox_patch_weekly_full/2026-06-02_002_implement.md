# 実装内容: proxmox_patch_weekly_full

要求仕様: `docs/ai/reviews/proxmox_patch_weekly_full/2026-06-02_001_requirement.md`

## 実装概要

pve2 → pve1 の順でローリングアップデートを行う全体制御 playbook を実装した。
既存のサブ playbook を `import_playbook` で連結し、各フェーズ間に停止条件チェックを挟む構成にした。

---

## 設計の考え方

### import_playbook + vars

各サブ playbook を `import_playbook` で呼び出し、`vars: target_node: pve2/pve1` を渡す。

```yaml
- import_playbook: proxmox_evacuate_node.yml
  vars:
    target_node: pve2
```

Ansible の `import_playbook` は静的インポートのため、`vars` で渡した変数はインポート先の全 play で有効になる。同じ playbook を `target_node=pve2` と `target_node=pve1` で 2 回呼び出しても、各インポートで独立したスコープが維持される。

### _final_status の参照

`proxmox_patch_dryrun` role は `_final_status` を以下のように設定する。

```yaml
run_once: true
delegate_to: localhost
```

これにより `_final_status` は `localhost` host の fact として記録される。後続の `hosts: localhost` play では直接 `_final_status` として参照できる。

### post-healthcheck の実装

steps 5 / 9 のポスト healthcheck は、`hosts: pve2` / `hosts: pve1` のインライン play で実装した。`include_role: proxmox_healthcheck` で healthcheck を実行し、`proxmox_report.result.status != 'OK'` の場合に `fail` する。

### 停止の連鎖

Ansible はデフォルトで play が失敗すると以降の plays は実行されない。そのため「pve2 が NG なら pve1 へ進まない」は、pve2 関連 plays のいずれかが fail すると自動的に達成される。初期 healthcheck play に `any_errors_fatal: true` を設定することで、片方のノードの失敗で即座に停止させる。

## 実装内容

### playbooks/proxmox_patch_weekly_full.yml（新規）

| ステップ | 実装方法 |
| --- | --- |
| 1a: healthcheck 両ノード | `import_playbook: proxmox_healthcheck.yml` |
| 1b: 両ノード OK チェック | インライン play `hosts: proxmox`、`any_errors_fatal: true` |
| 2a: dry-run | `import_playbook: proxmox_patch_dryrun.yml` |
| 2b: PATCH_READY チェック | インライン play `hosts: localhost`、`_final_status != 'PATCH_READY'` で fail |
| 3: evacuate pve2 | `import_playbook: proxmox_evacuate_node.yml vars: target_node: pve2` |
| 4: patch apply pve2 | `import_playbook: proxmox_patch_apply_node.yml vars: target_node: pve2` |
| 5: post-healthcheck pve2 | インライン play `hosts: pve2`、`include_role: proxmox_healthcheck` + fail |
| 6: restore pve2 | `import_playbook: proxmox_restore_vm_placement.yml vars: target_node: pve2` |
| 7: evacuate pve1 | `import_playbook: proxmox_evacuate_node.yml vars: target_node: pve1` |
| 8: patch apply pve1 | `import_playbook: proxmox_patch_apply_node.yml vars: target_node: pve1` |
| 9: post-healthcheck pve1 | インライン play `hosts: pve1`、`include_role: proxmox_healthcheck` + fail |
| 10: restore pve1 | `import_playbook: proxmox_restore_vm_placement.yml vars: target_node: pve1` |
| 11: summary mail | インライン play `hosts: localhost`、`community.general.mail` で完了通知 |
