# 要求仕様: proxmox_patch_weekly_full

## 目的

pve2 → pve1 の順でローリングアップデートを行う全体制御 playbook。
quory（外部 control node）からのみ実行する。

## 対象

- pve1, pve2

## 作成ファイル

- playbooks/proxmox_patch_weekly_full.yml（新規）

## 全体フロー

### pve2 フロー

1. pve1 / pve2 healthcheck（両方 OK でなければ停止）
2. pve2 dry-run（PATCH_READY でなければ停止）
3. proxmox_evacuate_node.yml を target_node=pve2 で実行
4. proxmox_patch_apply_node.yml を target_node=pve2 で実行
5. pve2 post-healthcheck OK 確認
6. proxmox_restore_vm_placement.yml を target_node=pve2 で実行

### pve1 フロー（pve2 完了後のみ）

7. proxmox_evacuate_node.yml を target_node=pve1 で実行
8. proxmox_patch_apply_node.yml を target_node=pve1 で実行
9. pve1 post-healthcheck OK 確認
10. proxmox_restore_vm_placement.yml を target_node=pve1 で実行

### 完了

11. summary mail 送信

## 停止条件

- healthcheck が WARNING / CRITICAL
- dry-run Status が PATCH_READY 以外
- evacuate 失敗
- patch apply 失敗
- post-healthcheck NG
- pve2 が NG の状態で pve1 へ進まない

## 制約

- control node が Proxmox クラスタ外にあることを確認する
- proxmox_patch_apply_node.yml は PATCH_READY の自動 apply モードで呼び出す
- 各 playbook は import_playbook で呼び出す
- unsafe（OS パッチ適用あり）
- 秘密情報を扱わない

## 参照ファイル

- playbooks/proxmox_evacuate_node.yml
- playbooks/proxmox_restore_vm_placement.yml
- playbooks/proxmox_patch_apply_node.yml
- playbooks/proxmox_patch_dryrun.yml
- playbooks/proxmox_healthcheck.yml

## 初回除外スコープ

- MAINTENANCE_REQUIRED 手動 apply モード（weekly_full は PATCH_READY のみ）

## shell / Ansible 責務分離

- shell: 対象外
- Ansible tasks: フロー制御・停止判定
