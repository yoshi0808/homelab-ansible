# Test Plan: proxmox_patch_weekly_full — Step 2b 終了分岐の検証

## 目的

commit 済みの Step 2b 変更が、dry-run status ごとに想定どおりの「ジョブ終了分岐」に
なることを、**実機に触れず・実パッチを走らせず**安全に検証する。

## 安全制約（厳守）

- 実機 pve1 / pve2 に**接続しない**。evacuate / patch / restore を**一切実行しない**。
- `proxmox_patch_weekly_full.yml` 本体を end-to-end で**実行しない**（PATCH_READY が後続の
  evacuate/patch に進むため）。
- 生成物（アサーション用 playbook 等）は `test-output/`（gitignore 済み）に置く。
- read-only / localhost のみ。

## テスト項目と期待結果

### T1. 構文チェック（再確認）

```
ANSIBLE_LOCAL_TEMP=/tmp/ansible-local-tmp \
  ansible-playbook --syntax-check -i inventories/homelab/hosts.yml \
  playbooks/proxmox_patch_weekly_full.yml
```
期待: 成功（rc=0）。

### T2. Step 2b 構造の確認（実ファイル静的チェック）

`playbooks/proxmox_patch_weekly_full.yml` に対し以下を確認:

- `_weekly_full_skip_statuses` が `[NO_UPDATES, MAINTENANCE_REQUIRED, MAJOR_UPGRADE_DETECTED]`。
- skip ブロックが `_weekly_full_skip: true` を set_fact し `meta: end_play` する。
- fail タスクの `when` が `_status != 'PATCH_READY'`。
- 後続 play / import_playbook が `_weekly_full_skip` で gate されている（旧 `_weekly_full_no_updates` の残存なし）。

### T3. 終了分岐ロジックのアサーション（localhost のみ・実機非接続）

`test-output/` に最小の localhost playbook を作り、各 `_final_status` を変数で与えて
**Step 2b と同一の式**で分岐を評価し、期待分類を `assert` する:

| `_final_status` | 期待分類 |
|---|---|
| `NO_UPDATES` | skip（rc=0・後続に進まない） |
| `MAINTENANCE_REQUIRED` | skip（rc=0・後続に進まない） |
| `MAJOR_UPGRADE_DETECTED` | skip（rc=0・後続に進まない） |
| `BLOCKED` | fail（rc≠0） |
| `UNKNOWN` | fail（rc≠0） |
| `PATCH_READY` | continue（skip でも fail でもない） |

- `hosts: localhost` / `connection: local` のみ。proxmox グループには一切接続しない。
