## 修正内容

### Fix 1 (High): `_final_status` の参照先を正しい hostvars に変更

**対象ファイル**: `playbooks/proxmox_patch_weekly_full.yml`

`proxmox_patch_dryrun` role は `_final_status` を以下のように設定する。

```yaml
run_once: true
delegate_to: localhost
# delegate_facts: true は付いていない
```

Ansible の `set_fact` は、`delegate_to` だけでは delegate 先に fact を保存しない。
`delegate_facts: true` がない場合、fact は run_once の元ホスト（`groups['proxmox'][0]`）に保存される。
このため `hosts: localhost` play で `_final_status` を直接参照しても未定義になり、
`default('') != 'PATCH_READY'` が常に true となって playbook が停止していた。

**変更前**:

```yaml
- name: Abort if dry-run status is not PATCH_READY
  hosts: localhost
  gather_facts: false
  tasks:
    - name: Fail if not PATCH_READY
      ansible.builtin.fail:
        msg: >-
          Dry-run status is {{ _final_status | default('UNKNOWN') }}.
          Weekly full patch apply requires PATCH_READY. Stopping.
      when: (_final_status | default('')) != 'PATCH_READY'
```

**変更後**:

```yaml
- name: Abort if dry-run status is not PATCH_READY
  hosts: localhost
  gather_facts: false
  tasks:
    - name: Fail if not PATCH_READY
      ansible.builtin.fail:
        msg: >-
          Dry-run status is
          {{ hostvars[groups['proxmox'][0]]['_final_status'] | default('UNKNOWN') }}.
          Weekly full patch apply requires PATCH_READY. Stopping.
      when: hostvars[groups['proxmox'][0]]['_final_status'] | default('') != 'PATCH_READY'
```

また Step 2a のコメントに `delegate_facts: true` がない旨と、fact の実際の保存先についての説明を追記した。

### Fix 2 (High): evacuate 前に control node クラスタ外チェックを追加

**対象ファイル**: `playbooks/proxmox_patch_weekly_full.yml`

`proxmox_patch_apply_node` role 内の control node チェックは patch apply 時点に行われる。
しかし weekly_full では patch apply より先に `proxmox_evacuate_node` が走り、
制御 VM（ansy）をクラスタ内で migrate してしまう可能性があった。

「ansy 自身を同一 playbook 内で migrate して処理を継続する運用はしない」という policy の意図と矛盾するため、
weekly_full の冒頭（Step 0）に preflight チェックを追加した。

**追加した play**:

```yaml
- name: Preflight - Verify control node is not on Proxmox cluster
  hosts: "{{ groups['proxmox'][0] }}"
  gather_facts: false
  become: true
  tasks:
    - name: Get cluster resources
      ansible.builtin.command:
        cmd: pvesh get /cluster/resources --output-format json
      register: _preflight_pvesh_raw
      changed_when: false

    - name: Detect control node on cluster
      ansible.builtin.set_fact:
        _control_node_on_cluster: >-
          {{ _preflight_pvesh_raw.stdout | from_json |
             selectattr('type', 'defined') |
             selectattr('type', 'equalto', 'qemu') |
             selectattr('name', 'defined') |
             selectattr('name', 'search', proxmox_patch_apply_control_node_name) |
             list }}

    - name: Abort if control node is on cluster
      ansible.builtin.fail:
        msg: >-
          Control node '{{ proxmox_patch_apply_control_node_name }}' is running on
          the Proxmox cluster (...).
          proxmox_patch_weekly_full must be run from quory (external control node).
      when: _control_node_on_cluster | length > 0
```

`proxmox_patch_apply_control_node_name` はロール defaults（`ansy`）に定義されていたが、
ロール外のコンテキストでは参照できないため、`group_vars/proxmox.yml` に昇格させた。

### Fix 2 の付随変更: `proxmox_patch_apply_control_node_name` を group_vars に追加

**対象ファイル**: `inventories/homelab/group_vars/proxmox.yml`

**追加内容**:

```yaml
# Control node VM name on the Proxmox cluster.
# Used by proxmox_patch_apply_node role and weekly_full preflight check.
# weekly_full must NOT be run from this VM (run from quory instead).
proxmox_patch_apply_control_node_name: "ansy"
```

role defaults に同名の変数が定義されているため、group_vars がそれを上書きする形になるが、
値は同一（`ansy`）であり、動作は変わらない。これにより preflight play がロール外でも同変数を参照できる。
