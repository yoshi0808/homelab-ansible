# implement: proxmox_patch_weekly_full.yml preflight check 修正

## 実装概要

`proxmox_patch_weekly_full.yml` の Step 0（preflight）が、
quory から実行しているにもかかわらず「ansy がクラスタ上にいる」として abort していた。

preflight の判定対象を、ハードコードされた `proxmox_patch_apply_control_node_name`（"ansy"）から
実行しているホストの `ansible_hostname`（= `lookup('pipe', 'hostname')`）に変更した。
quory から実行している場合は quory がクラスタ上の VM でないため、ansy の存在に関わらず続行する。

## 作成・更新したファイル

| ファイル | 変更種別 |
|---|---|
| `playbooks/proxmox_patch_weekly_full.yml` | 更新（Step 0 preflight のみ） |

## 変更内容の要約

### Step 0 ヘッダーコメント

**変更前:**
```
# If ansy (Ansible dev VM) is running on pve1/pve2, evacuate would migrate it
# mid-run, breaking this playbook. Must be run from quory instead.
```

**変更後:**
```
# If the Ansible controller (ansible_hostname) is a VM running on pve1/pve2,
# evacuate would migrate it mid-run, breaking this playbook.
# Must be run from quory (external, non-cluster host) instead.
# ansy may be on the cluster without issue when running from quory.
```

### Detect タスクの変更

**変更前:**
- `selectattr('name', 'search', proxmox_patch_apply_control_node_name)` でフィルタ → "ansy" を検索
- `_control_node_on_cluster | length > 0` で abort 判定

**変更後:**
- `map(attribute='name')` でクラスタ全 VM 名リスト `_cluster_vm_names` を作成
- `lookup('pipe', 'hostname')` で実行ホスト名 `_controller_hostname` を取得
- `_controller_hostname in _cluster_vm_names` で abort 判定

### タスク名の変更

| 旧タスク名 | 新タスク名 |
|---|---|
| `Detect control node on cluster` | `Build cluster VM name list and controller hostname` |
| `Abort if control node is on cluster` | `Abort if executing host is on cluster` |

## スコープ外

`roles/proxmox_patch_apply_node/tasks/main.yml` の Phase 1 にも同様のチェックが存在するが、
こちらは「特定のターゲットノード（pve1 または pve2）上に制御 VM（ansy）がいる場合に abort する」
という別の目的を持つため、今回は変更しない。

## 注意点

- `lookup('pipe', 'hostname')` は Ansible コントローラー上で実行されるため、
  play が `hosts: pve1` を対象にしていても quory（またはそれを実行している ansy）の
  ホスト名が返る。
- ansy から実行した場合は `_controller_hostname = "ansy"` となり、
  ansy はクラスタ上の VM として検出されるため abort する（正しい動作）。
- quory から実行した場合は `_controller_hostname = "quory"` となり、
  quory はクラスタ上の VM ではないため continue する（今回修正の目的）。
