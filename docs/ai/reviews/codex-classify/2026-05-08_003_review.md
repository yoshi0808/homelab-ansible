## Findings

### 1. Section 16 が実際にはプロンプトに含まれていない

Severity: High

`scripts/codex-classify.sh` では Section 3 だけを抽出している。

該当箇所:

- `scripts/codex-classify.sh:55`

requirement では `docs/ai/prompts/proxmox_patch_policy.md` から Section 3 と Section 16 のみ抽出するとされている。

該当箇所:

- `docs/ai/reviews/scripts/codex-classify/2026-05-08_001_requirement.md:24`

implement でも Section 3 と Section 16 をプロンプトに含めると説明している。

該当箇所:

- `docs/ai/reviews/scripts/codex-classify/2026-05-08_002_implement.md:45-49`

しかし実装には `SECTION16` の抽出も、プロンプトへの追加もない。

このため、Codex CLI の役割・入力・出力・禁止事項・責務分離がプロンプトに渡らず、policy 準拠の出力になりにくい。

---

### 2. `codex exec` へのプロンプト渡しが引数になっている

Severity: High

実装では以下のように、プロンプト全文をコマンド引数として渡している。

該当箇所:

- `scripts/codex-classify.sh:130`

```bash
codex exec "$(cat "$PROMPT_FILE")" > "$CODEX_OUTPUT" 2>&1
```

一方、implement では stdin 経由で渡すと説明している。

該当箇所:

- `docs/ai/reviews/scripts/codex-classify/2026-05-08_002_implement.md:62-67`

changelog 全文を含む設計では、プロンプトが長くなりやすく、引数長制限に当たる可能性がある。

`codex exec --help` では stdin 入力がサポートされていることを確認した。ここは stdin 経由にするのが自然。

推奨:

```bash
codex exec < "$PROMPT_FILE" > "$CODEX_OUTPUT" 2>&1
```

---

### 3. 出力スキーマが policy Section 16 と一致していない

Severity: Medium

`proxmox_patch_policy.md` の Section 16 では、最低限の JSON 出力に以下が含まれている。

- `status_inputs.security_sensitive_updates`
- `package_classifications[].evidence`

該当箇所:

- `docs/ai/prompts/proxmox_patch_policy.md:1554-1583`

しかし `scripts/codex-classify.sh` のプロンプト例と検証では、それらを要求していない。

該当箇所:

- `scripts/codex-classify.sh:101-125`
- `scripts/codex-classify.sh:200-205`

後続の Ansible tasks が policy 側のスキーマ前提で読むと、分類結果が不足する可能性がある。

---

### 4. JSON 検証が最低限すぎてテンプレート返却を検出できない

Severity: Medium

Python の検証はトップレベル必須キーのみを確認している。

該当箇所:

- `scripts/codex-classify.sh:200-205`

implement には、Codex が input_json を実際に分析せず、スキーマのダミーエントリをそのまま返す問題が発生したと記録されている。

該当箇所:

- `docs/ai/reviews/scripts/codex-classify/2026-05-08_002_implement.md:51-58`

そのため、少なくとも以下は検証した方がよい。

- `package_classifications` が空ではない
- `package` が `<updates の各パッケージ名に置き換えること>` のままではない
- 入力 JSON の `updates` 数と `package_classifications` 数が一致する
- policy で必要な `security_sensitive_updates` / `evidence` が存在する
