# Requirement: codex exec 失敗時のログ出力

作成日: 2026-05-08

---

## 対象

- `scripts/codex-classify.sh`

---

## 背景

006_review の Findings に基づく修正。

`codex exec` が非 0 で終了した場合、`set -e` によりスクリプトが即終了し、
`trap cleanup EXIT` で `$CODEX_OUTPUT` が削除されるため、
Ansible 側には失敗原因が残らない。

---

## 修正内容

`codex exec` 失敗時に `$CODEX_OUTPUT` の先頭 200 行を stderr に出力してから終了する。

現在:

```bash
codex exec "$(cat "$PROMPT_FILE")" > "$CODEX_OUTPUT" 2>&1
```

修正後:

```bash
if ! codex exec "$(cat "$PROMPT_FILE")" > "$CODEX_OUTPUT" 2>&1; then
    echo "Error: codex exec failed" >&2
    sed -n '1,200p' "$CODEX_OUTPUT" >&2
    exit 1
fi
```

## 制約

- 修正箇所以外は変えない
