対象範囲を "Ubuntu VM 全般" から "Ubuntu ノード全般（VM・物理ノード）" へ広げ、
`quory`（物理ノード）の reboot 方針を明記する。

## 対象ファイル

- `docs/ai/prompts/ubuntu_vm_patch_policy.md`

## 実施内容

1. タイトル・版表記を v1.0 → v1.1 に更新する
2. 対象範囲を "Ubuntu VM" から "Ubuntu ノード全般（VM・物理ノード）" に変更する
3. ノード一覧に `quory`（物理ノード / Ansible 実行基盤 / Proxmox QDevice）を追加する
   - 種別: 物理ノード
   - reboot 方針: 自動 reboot（時刻固定、03:00）
   - healthcheck: なし
   - 理由: 自身が Ansible 実行基盤のため ubuntu_nightly.yml による管理不可
4. systemd timer 名を ansy systemd timers role の実際の名称に合わせる
   - `ubuntu-nightly.timer` → `ansible-authy-reboot-if-required.timer`
   - `radius-healthcheck.timer` → `ansible-authy-healthcheck.timer`
5. timer 実行時刻を確定値に変更する
   - `ubuntu_nightly.yml`: 02:00 頃 → 03:30
   - `radius_healthcheck.yml`: 06:00 頃 → 05:30
6. systemd timer 実行ノードを quory に明記する（quory 上で実行）
7. 深夜リブートスケジュール全体（UniFi Console → UniFi Device → quory → authy）を追加する
8. 現在の設定確認セクションを 2026-05-18 時点の内容に更新する
   - authy: 既存の Automatic-Reboot "false" 設定を確認
   - quory: Automatic-Reboot "true" / Automatic-Reboot-Time "03:00" の設定を追記

## 制約

- 秘密情報を扱わない
