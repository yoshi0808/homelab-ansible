## Findings

### 1. local-only storage VM を自動 migration しない方針を迂回する可能性がある

Severity: Medium

`roles/proxmox_evacuate_node/tasks/main.yml:129`

```yaml
qm migrate {{ item.vmid }} {{ destination_node }} --with-local-disks
```

今回の変更により、退避対象の QEMU VM すべてに `--with-local-disks` が常時付与される。

これは、実行時に発生した以下のエラーを解消する目的としては妥当。

```text
can't live migrate attached local disks without with-local-disks option
```

一方で、`docs/ai/prompts/proxmox_patch_policy.md` では local-only storage を使う VM / CT は自動 migration を行わず停止する条件として扱われている。

該当箇所:

- `5.6 自動 migration の扱い`
  - `対象 VM / CT が local-only storage を使っている`
- `proxmox_evacuate_node` の実行条件
  - `対象 VM / CT が local-only storage に依存していない`
- `proxmox_evacuate_node` の停止条件
  - `local-only storage 使用 VM / CT がある`

そのため、現状のままでは「local disk がある VM を検知したら止める」という policy 上の停止条件ではなく、「local disk も含めて自動で移行する」という挙動になる。

特に `--with-local-disks` は local disk の storage migration を許可するオプションなので、単なるエラー回避ではなく、VM ディスク配置を実際に動かす変更になる。patch policy を正とするなら、少なくとも以下のどちらかが必要。

- local-only storage VM を検出して fail し、`--with-local-disks` は明示的に許可された場合だけ使う
- policy 側を更新し、local disk migration を controlled apply の許容範囲に含める

現時点では requirement の「local-only storage 使用 VM の詳細チェックは初回除外スコープ」と、patch policy の「local-only storage 使用 VM / CT がある場合は停止」が緊張関係にある。今回の差分はその判断を実装側で暗黙に「移行する」へ倒しているため、レビュー上は未解決リスクとして扱う。

## 補足

`--with-local-disks` の追加自体は、QEMU VM の local disk live migration エラーに対する直接修正として筋が通っている。

ただし、この repository の patch policy では local-only storage を自動移行対象から外す安全側の方針が書かれているため、今回の変更を採用するなら policy 変更または明示的な opt-in 変数を併せて入れる方が安全。

例:

```yaml
proxmox_evacuate_allow_local_disk_migration: false
```

この値が `true` の場合のみ `--with-local-disks` を付与し、`false` の場合は local disk 検出時に fail する、という形なら policy と実行時回避の両方を両立しやすい。
