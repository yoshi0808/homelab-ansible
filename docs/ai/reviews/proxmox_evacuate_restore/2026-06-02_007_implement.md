# 追加実装内容: proxmox_evacuate_node（local disk migrate 修正）

対応: 実行時に発生した `qm migrate` エラーの修正

## 実装概要

`proxmox_evacuate_node` の実行時、非HA QEMU VM の移行で以下のエラーが発生した。

```text
can't live migrate attached local disks without with-local-disks option
```

ローカルディスクを持つ VM をライブマイグレーションする際は `--with-local-disks` オプションが必須のため、`qm migrate` コマンドに同オプションを追加した。

---

## 修正内容

### Fix: qm migrate に --with-local-disks を追加

**対象ファイル**: `roles/proxmox_evacuate_node/tasks/main.yml`

**変更前**:

```yaml
- name: Migrate non-HA QEMU VMs to destination_node
  ansible.builtin.command:
    cmd: >-
      qm migrate {{ item.vmid }} {{ destination_node }}
      {{ '--online 1' if item.status == 'running' else '' }}
```

**変更後**:

```yaml
- name: Migrate non-HA QEMU VMs to destination_node
  ansible.builtin.command:
    cmd: >-
      qm migrate {{ item.vmid }} {{ destination_node }} --with-local-disks
      {{ '--online 1' if item.status == 'running' else '' }}
```

**理由**:
- ローカルディスクを attach した VM はライブマイグレーション時に `--with-local-disks` が必須。
- `--with-local-disks` は QEMU の `qm migrate` 専用オプションのため、QEMU VM migrate タスクのみに追加した。LXC の `pct migrate` には不要かつ非対応のため変更しない。
- 共有ストレージのみの VM に付与しても無害であり、online / offline どちらの移行でも有効なため、`--online 1` の有無に関わらず常に付与する。
