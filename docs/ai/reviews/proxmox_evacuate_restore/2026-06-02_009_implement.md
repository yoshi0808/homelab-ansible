# 追加実装内容: proxmox_evacuate_node（local disk migration の opt-in 化）

対応レビュー: `docs/ai/reviews/proxmox_evacuate_restore/2026-06-02_008_review.md`

## 実装概要

レビュー 008 の Medium 指摘に対応した。

`2026-06-02_007_implement.md` で `qm migrate` に `--with-local-disks` を**常時付与**したが、
これは `docs/ai/prompts/proxmox_patch_policy.md` の安全側方針と矛盾していた。

policy 側（5.6 / evacuate 実行条件・停止条件）では、local-only storage を使う VM / CT は
**自動 migration せず停止する**と定めている。常時 `--with-local-disks` を付けると、
本来止めるべき local disk VM のディスク配置まで自動で動かしてしまう。

そこで opt-in 変数を導入し、既定では policy 通り「local disk VM は止める」挙動に戻した。

---

## 修正内容

### Fix 1: opt-in 変数 `proxmox_evacuate_allow_local_disk_migration` を追加

**対象ファイル**: `roles/proxmox_evacuate_node/defaults/main.yml`

**追加内容**:

```yaml
# Allow live-migrating QEMU VMs that have local (non-shared) disks.
# Per proxmox_patch_policy.md (5.6), local-only storage VMs are NOT auto-migrated
# by default; the evacuation must stop so a human can decide. Set to true only
# when local disk storage migration is explicitly approved for this run; this
# appends --with-local-disks to qm migrate.
proxmox_evacuate_allow_local_disk_migration: false
```

**理由**: policy を正とし、既定は安全側（local disk VM は自動移行しない）に固定する。
明示的に承認された run でのみ `true` を渡して local disk migration を許可する。

---

### Fix 2: `--with-local-disks` を opt-in 変数で条件付き付与に変更

**対象ファイル**: `roles/proxmox_evacuate_node/tasks/main.yml`

**変更前**（007 の状態）:

```yaml
- name: Migrate non-HA QEMU VMs to destination_node
  ansible.builtin.command:
    cmd: >-
      qm migrate {{ item.vmid }} {{ destination_node }} --with-local-disks
      {{ '--online 1' if item.status == 'running' else '' }}
```

**変更後**:

```yaml
- name: Migrate non-HA QEMU VMs to destination_node
  ansible.builtin.command:
    cmd: >-
      qm migrate {{ item.vmid }} {{ destination_node }}
      {{ '--online 1' if item.status == 'running' else '' }}
      {{ '--with-local-disks' if (proxmox_evacuate_allow_local_disk_migration | bool) else '' }}
```

**挙動**:

- `proxmox_evacuate_allow_local_disk_migration: false`（既定）
  - `--with-local-disks` を付けない。
  - local disk を持つ VM は `qm migrate` が
    `can't live migrate attached local disks without with-local-disks option`
    で失敗し、block の rescue 経由で playbook が停止する。
  - これは policy「local-only storage 使用 VM / CT がある → 停止」と一致する。
- `proxmox_evacuate_allow_local_disk_migration: true`（明示承認時）
  - `--with-local-disks` を付与し、local disk ごと移行する。

**理由**: 「単なる実行時エラー回避」と「policy の安全側停止条件」を両立させるため、
レビュー 008 が提案した opt-in 変数方式を採用した。

## スコープに関する補足

requirement（001）では「local-only storage 使用 VM の詳細チェック」は初回除外スコープ。
そのため、VM config を事前パースして local storage を能動的に検出する処理は**実装しない**。

既定（`false`）では `qm migrate` の自然なエラーで停止させることで、
詳細検出を実装せずに policy の停止挙動を満たす。能動的な事前検出は将来スコープとする。
