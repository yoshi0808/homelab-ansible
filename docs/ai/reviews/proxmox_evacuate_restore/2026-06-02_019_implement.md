# 追加実装内容: maintenance mode 待ち正規表現の修正

対応レビュー: `docs/ai/reviews/proxmox_evacuate_restore/2026-06-02_018_review.md`

## 実装概要

レビュー 018 の 2 件の指摘に対応した。

`ha-manager status` の LRM 行フォーマットを誤って解釈していたため、
`until` 条件が実際の出力に一致せず、timeout になる問題を修正した。

---

## 修正内容

### Fix 1 (High): enable 待ちの正規表現を実際の出力形式に修正

**対象ファイル**: `roles/proxmox_evacuate_node/tasks/main.yml`

`ha-manager status` の実際の LRM 行フォーマットは以下のとおり。

```text
lrm pve1 (maintenance mode, Thu Feb 1 17:56:54 2024)
```

`node名` の直後は `:` ではなく ` (` になるため、従来のパターンでは一致しなかった。

**変更前**:

```yaml
select('match', 'lrm ' ~ inventory_hostname ~ ':.*maintenance mode') |
```

**変更後**:

```yaml
select('match', '^lrm ' ~ inventory_hostname ~ ' \\(.*maintenance mode') |
```

- 行頭 `^` を追加して前方一致を明確化
- `:` を ` \\(` に変更（`ha-manager status` 実出力に合わせる）

---

### Fix 2 (Medium): disable 待ちの条件を `active|idle` に拡張

**対象ファイル**: `roles/proxmox_restore_vm_placement/tasks/main.yml`

disable 後に LRM が戻る状態は、HA サービスが存在する場合は `active`、
HA サービスがない場合は `idle` になる。

`proxmox_restore_vm_placement` は `hacritical` VM の HA 自動復帰を期待する role のため、
HA サービスが存在して LRM が `active` に戻るケースが通常形。`idle` のみを条件にすると
正常ケースが timeout になっていた。

実際の出力例:

```text
lrm pve1 (active, Thu Feb 27 18:25:52 2025)
lrm pve2 (idle, Thu Feb 27 18:25:52 2025)
```

**変更前**:

```yaml
select('match', 'lrm ' ~ inventory_hostname ~ ':.*idle') |
```

**変更後**:

```yaml
select('match', '^lrm ' ~ inventory_hostname ~ ' \\((active|idle),') |
```

- Fix 1 と同様に `:` → ` \\(` に修正
- `idle` のみから `active|idle` に拡張（maintenance mode 以外の通常状態を許容）
- カンマ `,` を末尾に加えて `(active,` / `(idle,` の形式に絞り込み、誤マッチを防止
