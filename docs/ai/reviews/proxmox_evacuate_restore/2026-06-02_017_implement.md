# 追加実装内容: メンテナンスモードコマンド修正 + 完了待ち追加

対応: 実環境でのメンテナンスモードコマンド誤りの修正

## 実装概要

`pvecm node-maintenance` コマンドは実環境では存在せず、正しくは
`ha-manager crm-command node-maintenance` を使う必要があった。

また、メンテナンスモードの有効化・無効化は非同期で処理されるため、
`ha-manager status` で LRM の状態が切り替わるまでポーリング待ちする処理を追加した。

対象は以下の 2 役割:
- `proxmox_evacuate_node` — enable とその完了待ち
- `proxmox_restore_vm_placement` — disable とその完了待ち

---

## 修正内容

### Fix 1: enable コマンドを修正し、完了待ちを追加

**対象ファイル**: `roles/proxmox_evacuate_node/tasks/main.yml`

**変更前**:

```yaml
- name: Enable maintenance mode on target node
  ansible.builtin.command:
    cmd: pvecm node-maintenance enable {{ inventory_hostname }}
  changed_when: true

- name: Record maintenance mode enabled
  ansible.builtin.set_fact:
    _evac_maintenance_enabled: true
```

**変更後**:

```yaml
- name: Enable maintenance mode on target node
  ansible.builtin.command:
    cmd: ha-manager crm-command node-maintenance enable {{ inventory_hostname }}
  changed_when: true

- name: Wait for maintenance mode to become active
  ansible.builtin.command:
    cmd: ha-manager status
  register: _evac_maint_poll
  changed_when: false
  retries: "{{ (proxmox_evacuate_maintenance_timeout | int) // (proxmox_evacuate_maintenance_poll_interval | int) }}"
  delay: "{{ proxmox_evacuate_maintenance_poll_interval }}"
  until: >-
    _evac_maint_poll.stdout_lines |
    select('match', 'lrm ' ~ inventory_hostname ~ ':.*maintenance mode') |
    list | length > 0

- name: Record maintenance mode enabled
  ansible.builtin.set_fact:
    _evac_maintenance_enabled: true
```

**完了条件**: `ha-manager status` の出力に `lrm <node>: ... maintenance mode` の行が現れる。

### Fix 2: evacuate role の defaults にタイムアウト変数を追加

**対象ファイル**: `roles/proxmox_evacuate_node/defaults/main.yml`

**追加**:

```yaml
# Seconds to wait for maintenance mode to become active (lrm <node>: maintenance mode)
proxmox_evacuate_maintenance_timeout: 120

# Poll interval in seconds while waiting for maintenance mode activation
proxmox_evacuate_maintenance_poll_interval: 5
```

### Fix 3: disable コマンドを修正し、完了待ちを追加

**対象ファイル**: `roles/proxmox_restore_vm_placement/tasks/main.yml`

**変更前**:

```yaml
- name: Disable maintenance mode on target node
  ansible.builtin.command:
    cmd: pvecm node-maintenance disable {{ inventory_hostname }}
  changed_when: true

- name: Record maintenance mode disabled
  ansible.builtin.set_fact:
    _restore_maintenance_disabled: true
```

**変更後**:

```yaml
- name: Disable maintenance mode on target node
  ansible.builtin.command:
    cmd: ha-manager crm-command node-maintenance disable {{ inventory_hostname }}
  changed_when: true

- name: Wait for maintenance mode to become inactive
  ansible.builtin.command:
    cmd: ha-manager status
  register: _restore_maint_poll
  changed_when: false
  retries: "{{ (proxmox_restore_maintenance_timeout | int) // (proxmox_restore_maintenance_poll_interval | int) }}"
  delay: "{{ proxmox_restore_maintenance_poll_interval }}"
  until: >-
    _restore_maint_poll.stdout_lines |
    select('match', 'lrm ' ~ inventory_hostname ~ ':.*idle') |
    list | length > 0

- name: Record maintenance mode disabled
  ansible.builtin.set_fact:
    _restore_maintenance_disabled: true
```

**完了条件**: `ha-manager status` の出力に `lrm <node>: ... idle` の行が現れる。
