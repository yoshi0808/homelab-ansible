# 追加実装内容: proxmox_evacuate_node（local disk migration の policy 整合化）

対応レビュー: `docs/ai/reviews/proxmox_evacuate_restore/2026-06-02_013_review.md`

## 実装概要

レビュー 013 の Medium 指摘に対応した。

`2026-06-02_012_implement.md` で `proxmox_evacuate_allow_local_disk_migration` の既定値を
`true` にしたが、これは `proxmox_patch_policy.md` (5.6) の「local-only storage VM は停止」
方針と role default が逆向きになるという不整合を招いていた。

レビュー 013 が示した 2 つの選択肢のうち、以下を採用した。

> role default は `false` のまま維持し、homelab 固有の承認済み設定として
> inventory / group_vars 側で `true` を設定する

**理由**:
- Ansible のベストプラクティスに沿っている
  （`defaults/main.yml` = 汎用安全基準、`group_vars` = 環境固有設定）
- `proxmox_patch_policy.md` の文言を変更せずに済む
- role を別環境で使う場合も安全側（`false`）が既定として機能する
- homelab 固有の承認（全 VM が `local-zfs`）は `group_vars/proxmox.yml` に明示的に記録される

---

## 修正内容

### Fix 1: role default を false に戻す（コメント更新）

**対象ファイル**: `roles/proxmox_evacuate_node/defaults/main.yml`

**変更前**（012 の状態）:

```yaml
# Allow live-migrating QEMU VMs that have local (non-shared) disks.
# When true, --with-local-disks is appended to qm migrate so VMs on local
# storage can be evacuated. This homelab uses local-zfs for all VMs, so the
# default is true; without it evacuation fails with
# "can't live migrate attached local disks without with-local-disks option".
# Set to false to enforce the proxmox_patch_policy.md (5.6) safe-side behavior
# of stopping evacuation when a VM uses local-only storage.
proxmox_evacuate_allow_local_disk_migration: true
```

**変更後**:

```yaml
# Allow live-migrating QEMU VMs that have local (non-shared) disks.
# When true, --with-local-disks is appended to qm migrate.
# Default is false per proxmox_patch_policy.md (5.6): evacuations must stop
# when a VM uses local-only storage so a human can decide.
# Override to true in group_vars or playbook vars for environments where
# local-disk live migration is explicitly approved (e.g. local-zfs homelabs).
proxmox_evacuate_allow_local_disk_migration: false
```

---

### Fix 2: homelab 固有の承認設定を group_vars に追加

**対象ファイル**: `inventories/homelab/group_vars/proxmox.yml`

**変更前**:

```yaml
---
ansible_user: ann
ansible_ssh_private_key_file: ~/.ssh/id_ann
```

**変更後**:

```yaml
---
ansible_user: ann
ansible_ssh_private_key_file: ~/.ssh/id_ann

# All VMs in this homelab use local-zfs (local-only storage).
# --with-local-disks is required for live migration; approved for this environment.
proxmox_evacuate_allow_local_disk_migration: true
```

**理由**:
- この homelab では全 VM が `local-zfs` を使用しており、`--with-local-disks` なしの
  live migration は必ず失敗する。
- 環境固有の承認済み設定を `group_vars/proxmox.yml` に置くことで、
  role default（policy 準拠）と実環境の要件を分離できる。

## policy との整合

本変更後の状態:

| 設定箇所 | 値 | 意味 |
|---|---|---|
| `defaults/main.yml` | `false` | policy 準拠の安全側基準（変更なし） |
| `group_vars/proxmox.yml` | `true` | この homelab での明示的承認（local-zfs 環境） |

Ansible の変数優先順位により `group_vars` が `defaults` を上書きするため、
この homelab の実行時は `--with-local-disks` が付与される。
policy 文書の変更は不要。
