# 実装内容: proxmox_evacuate_node / proxmox_restore_vm_placement

## 実装概要

ローリングアップデート用のVM退避・復帰playbookを実装した。
Proxmox VMタグ（preferpve1/preferpve2/hacritical）を参照してmigration対象を判定し、
HA維持・メンテナンスモード制御・残存確認まで一貫して行う。

## proxmox_evacuate_node

### playbook構成

3つのplayで構成される。

1. **localhost play**: target_node の入力検証（pve1 or pve2 のみ許可）
2. **destination_node play**: destination_node の healthcheck を実行し、OK でない場合は中止
3. **target_node play**: `proxmox_evacuate_node` roleを実行

`destination_node` は target_node の反対ノードとして自動導出（override 可）。

### roleの処理フロー

| Phase | 処理 |
|---|---|
| 1 | target_node の healthcheck（proxmox_healthcheck role呼び出し）。OKでなければ中止 |
| 2 | `pvesh get /cluster/resources` でVM/CT一覧を取得し、タグで分類 |
| 3 | 非HA VM/CT（preferpveタグあり、hacriticalなし）を destination_node へ migrate |
| 4 | `pvecm node-maintenance enable <target_node>` でメンテナンスモード有効化 |
| 5 | hacriticalタグのVMがHAによって退避されるまで待機（retries/untilでポーリング） |
| 6 | 残存running VMを確認し、残っていれば force-stop |

### タグ判定ロジック

`pvesh get /cluster/resources` の `tags` フィールド（セミコロン区切り）をJinja2 `selectattr('tags', 'search', ...)`でフィルタ。

| 対象 | 条件 |
|---|---|
| migrate対象 | `preferpve*`タグあり かつ `hacritical`タグなし |
| HA管理（待機対象） | `hacritical`タグあり |
| skip（テンプレート等） | `tags`フィールドなし または preferpve/hacritical なし |

### 変数（defaults）

| 変数 | デフォルト | 説明 |
|---|---|---|
| `proxmox_evacuate_report_dir` | `{{ reports_base_dir }}/proxmox-evacuate` | レポート保存先 |
| `proxmox_evacuate_ha_timeout` | 300 | HA退避待ちタイムアウト（秒） |
| `proxmox_evacuate_ha_poll_interval` | 10 | HA退避ポーリング間隔（秒） |

### 実行例

```bash
ansible-playbook -i inventories/homelab/hosts.yml playbooks/proxmox_evacuate_node.yml \
  -e target_node=pve2
```

## proxmox_restore_vm_placement

### playbook構成

2つのplayで構成される。

1. **localhost play**: target_node の入力検証
2. **target_node play**: `proxmox_restore_vm_placement` role を実行

### roleの処理フロー

| Phase | 処理 |
|---|---|
| 1 | `pvesh get /cluster/resources` で VM/CT 一覧を取得し、migrate back 対象を特定 |
| 2 | `prefer<target_node>`タグあり・hacriticalなし・現在 target_node にいない VM を target_node へ migrate |
| 3 | migrate 後に stopped な VM を起動（qm start / pct start） |
| 4 | `pvecm node-maintenance disable <target_node>` でメンテナンスモード解除 |
| 5 | `hacritical` かつ `prefer<target_node>` タグのVMがHAで復帰するまで待機 |
| 6 | `proxmox_healthcheck` role を呼び出してpost-restore healthcheckを実行 |

### restore対象の判定

`prefer<target_node>`タグ（preferpve1またはpreferpve2）を持ち、hacriticalタグを持たず、
現在 target_node 以外にいる VM/CT を migrate back 対象とする。

```jinja2
# inventory_hostname = pve2 の場合
selectattr('tags', 'search', '(^|;)preferpve2(;|$)')
rejectattr('tags', 'search', '(^|;)hacritical(;|$)')
rejectattr('node', 'equalto', 'pve2')
```

### 変数（defaults）

| 変数 | デフォルト | 説明 |
|---|---|---|
| `proxmox_restore_report_dir` | `{{ reports_base_dir }}/proxmox-restore` | レポート保存先 |
| `proxmox_restore_ha_timeout` | 300 | HA復帰待ちタイムアウト（秒） |
| `proxmox_restore_ha_poll_interval` | 10 | HA復帰ポーリング間隔（秒） |
| `proxmox_restore_vm_start_delay` | 10 | migrate後の起動確認前待機（秒） |

### 実行例

```bash
ansible-playbook -i inventories/homelab/hosts.yml playbooks/proxmox_restore_vm_placement.yml \
  -e target_node=pve2
```

## 共通設計方針

- `pvesh` / `pvecm` / `qm` / `pct` コマンド実行はすべてAnsible `command` taskで行い、shell は使用しない
- 判定・制御・待機はAnsible tasks側で行う（core.mdの責務分離方針に準拠）
- rescue ブロックで失敗を捕捉し、always ブロックで必ずレポートを保存する
- レポートは `reports_base_dir` 配下にタイムスタンプ付きJSONで保存
- タグなしVM（テンプレート等）はmigration対象から自動的に除外される
- `proxmox_vm_home_nodes.yml` は使用せず、Proxmoxタグで home node を判定する

## 注意点

- `pvecm node-maintenance enable/disable` はPVE 7.x以降で使用可能
- HA VMが現状ゼロの場合、メンテナンスモード待機ステップはスキップされる（`when: _vms_ha | length > 0`）
- qm migrate のオンラインマイグレーションには共有ストレージ（Ceph等）が必要
  - local-only ストレージのVMはmigration失敗でフェイルする（要事前確認）
- pct migrate --restart は running CT を停止してmigrateし、destination で再起動する
  - サービス停止が発生するため注意
