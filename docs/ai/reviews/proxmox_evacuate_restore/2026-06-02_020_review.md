## 確認内容

### Fix 1: enable 待ちの LRM 正規表現

`roles/proxmox_evacuate_node/tasks/main.yml:169`

```yaml
select('match', '^lrm ' ~ inventory_hostname ~ ' \\(.*maintenance mode') |
```

`ha-manager status` の LRM 行は `lrm pve1 (maintenance mode, ...)` の形式なので、node 名の直後に `\\(` を期待する今回の条件は前回指摘と整合している。

### Fix 2: disable 待ちで active / idle を許容

`roles/proxmox_restore_vm_placement/tasks/main.yml:172`

```yaml
select('match', '^lrm ' ~ inventory_hostname ~ ' \\((active|idle),') |
```

HA サービスが存在する場合の `active` と、HA サービスがない場合などの `idle` をどちらも成功条件に含めている。

`maintenance mode` は許容しないため、「maintenance mode 解除後に通常状態へ戻った」ことを確認する条件として妥当。

### コマンド修正と timeout 変数

`pvecm node-maintenance` から `ha-manager crm-command node-maintenance` への変更、および maintenance mode enable / disable の timeout 変数追加は、前回 `017_implement` からの差分として維持されている。

```yaml
proxmox_evacuate_maintenance_timeout: 120
proxmox_evacuate_maintenance_poll_interval: 5
proxmox_restore_maintenance_timeout: 120
proxmox_restore_maintenance_poll_interval: 5
```
