## Findings

重大な追加指摘はありません。

今回の実装は、evacuate 側で採用済みの local disk migration opt-in 設計と整合している。

- role default は `false` で、`proxmox_patch_policy.md` の安全側方針に合わせている
- homelab 固有の承認として `inventories/homelab/group_vars/proxmox.yml` で `true` を設定している
- QEMU VM の restore migration のみに `--with-local-disks` を条件付き付与している
- LXC の `pct migrate` には非対応オプションを追加していない

## 確認内容

### Fix 1: restore role default は安全側の false

`roles/proxmox_restore_vm_placement/defaults/main.yml:24`

```yaml
proxmox_restore_allow_local_disk_migration: false
```

role 単体の既定値は `false` なので、別環境や inventory override なしの実行では local-only storage VM の migration を自動許可しない。

これは `proxmox_patch_policy.md` の local-only storage に関する安全側方針と整合している。

### Fix 2: homelab group_vars で明示承認

`inventories/homelab/group_vars/proxmox.yml:8`

```yaml
proxmox_restore_allow_local_disk_migration: true
```

`pve1` / `pve2` は `inventories/homelab/hosts.yml` で `proxmox` グループ配下にあるため、この homelab の restore 実行時は role default を上書きして `true` になる。

実際に以下で `pve2` の hostvars に `proxmox_restore_allow_local_disk_migration: true` が含まれることを確認した。

### Fix 3: QEMU migrate back のみに条件付き付与

`roles/proxmox_restore_vm_placement/tasks/main.yml:84`

```yaml
{{ '--with-local-disks' if (proxmox_restore_allow_local_disk_migration | bool) else '' }}
```

`qm migrate` のみに追加されており、`pct migrate` は変更されていない。この点は `2026-06-02_022_implement.md` の説明どおり。
