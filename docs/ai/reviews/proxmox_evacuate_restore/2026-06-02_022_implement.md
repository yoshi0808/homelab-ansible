# 追加実装内容: proxmox_restore_vm_placement（local disk migration opt-in 追加）

対応: 実行時に発生した `qm migrate` エラーの修正

## 実装概要

`proxmox_restore_vm_placement` の実行時、非HA QEMU VM の migrate back で
evacuate 側と同じエラーが発生する問題を修正した。

```text
can't live migrate attached local disks without with-local-disks option
```

evacuate 側（`2026-06-02_014_implement.md`）と同じ設計を採用した。

- role default は `false`（`proxmox_patch_policy.md` 5.6 準拠の安全側基準）
- `inventories/homelab/group_vars/proxmox.yml` に `true` を設定（homelab 固有承認）
- `qm migrate` に `--with-local-disks` を opt-in 変数で条件付き付与

---

## 修正内容

### Fix 1: opt-in 変数 `proxmox_restore_allow_local_disk_migration` を追加

**対象ファイル**: `roles/proxmox_restore_vm_placement/defaults/main.yml`

**追加内容**:

```yaml
# Allow live-migrating QEMU VMs that have local (non-shared) disks.
# When true, --with-local-disks is appended to qm migrate.
# Default is false per proxmox_patch_policy.md (5.6); override to true in
# group_vars or playbook vars for environments where local-disk migration is
# explicitly approved (e.g. local-zfs homelabs).
proxmox_restore_allow_local_disk_migration: false
```

---

### Fix 2: homelab 固有の承認設定を group_vars に追加

**対象ファイル**: `inventories/homelab/group_vars/proxmox.yml`

**追加内容**:

```yaml
proxmox_restore_allow_local_disk_migration: true
```

evacuate 側の `proxmox_evacuate_allow_local_disk_migration: true` と同じコメントブロック内に追記した。

---

### Fix 3: `qm migrate` に `--with-local-disks` を条件付き付与

**対象ファイル**: `roles/proxmox_restore_vm_placement/tasks/main.yml`

**変更前**:

```yaml
- name: Migrate non-HA QEMU VMs back to target_node
  ansible.builtin.command:
    cmd: >-
      qm migrate {{ item.vmid }} {{ inventory_hostname }}
      {{ '--online 1' if item.status == 'running' else '' }}
```

**変更後**:

```yaml
- name: Migrate non-HA QEMU VMs back to target_node
  ansible.builtin.command:
    cmd: >-
      qm migrate {{ item.vmid }} {{ inventory_hostname }}
      {{ '--online 1' if item.status == 'running' else '' }}
      {{ '--with-local-disks' if (proxmox_restore_allow_local_disk_migration | bool) else '' }}
```

**理由**:
- この homelab では全 VM が `local-zfs` を使用しており、`--with-local-disks` なしの live migration は必ず失敗する。
- LXC の `pct migrate` には `--with-local-disks` が非対応のため変更しない。
- evacuate 側と同じ opt-in 設計により、policy 準拠の role default と homelab 固有の実態を分離している。
