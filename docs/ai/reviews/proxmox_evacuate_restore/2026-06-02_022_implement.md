# 追加実装内容: proxmox_restore_vm_placement（migrate を source node で実行）

対応: 実行時に発生した `qm migrate` / `pct migrate` エラーの修正

## 実装概要

`proxmox_restore_vm_placement` の実行時、非HA VM の migrate back で以下のエラーが発生した。

```text
target: target is local node.
```

**原因**: playbook は `target_node`（例: pve2）上で動作する。そのため
`qm migrate <vmid> pve2` を pve2 自身で実行してしまい、自ノードへの migration として
拒否される。

`qm migrate` / `pct migrate` は VM が**現在いるノード（source node）上で実行**しなければならない。

**修正**: migrate タスクに `delegate_to: "{{ item.node }}"` を追加した。
`item.node` は `pvesh get /cluster/resources` の応答に含まれる現在の配置ノード名で、
これにより各 VM の source node 上でコマンドが実行される。

---

## 修正内容

### Fix: migrate タスクに `delegate_to: "{{ item.node }}"` を追加

**対象ファイル**: `roles/proxmox_restore_vm_placement/tasks/main.yml`

#### QEMU VM

**変更前**:

```yaml
- name: Migrate non-HA QEMU VMs back to target_node
  ansible.builtin.command:
    cmd: >-
      qm migrate {{ item.vmid }} {{ inventory_hostname }}
      {{ '--online 1' if item.status == 'running' else '' }}
      {{ '--with-local-disks' if (proxmox_restore_allow_local_disk_migration | bool) else '' }}
  loop: "{{ _vms_to_restore | selectattr('type', 'equalto', 'qemu') | list }}"
  loop_control:
    label: "QEMU {{ item.vmid }} {{ item.name | default('') }} from {{ item.node }} [{{ item.status }}]"
  changed_when: true
```

**変更後**:

```yaml
- name: Migrate non-HA QEMU VMs back to target_node
  ansible.builtin.command:
    cmd: >-
      qm migrate {{ item.vmid }} {{ inventory_hostname }}
      {{ '--online 1' if item.status == 'running' else '' }}
      {{ '--with-local-disks' if (proxmox_restore_allow_local_disk_migration | bool) else '' }}
  loop: "{{ _vms_to_restore | selectattr('type', 'equalto', 'qemu') | list }}"
  loop_control:
    label: "QEMU {{ item.vmid }} {{ item.name | default('') }} from {{ item.node }} [{{ item.status }}]"
  delegate_to: "{{ item.node }}"
  changed_when: true
```

#### LXC コンテナ

**変更前**:

```yaml
- name: Migrate non-HA LXC containers back to target_node
  ansible.builtin.command:
    cmd: >-
      pct migrate {{ item.vmid }} {{ inventory_hostname }}
      {{ '--restart 1' if item.status == 'running' else '' }}
  loop: "{{ _vms_to_restore | selectattr('type', 'equalto', 'lxc') | list }}"
  loop_control:
    label: "LXC {{ item.vmid }} {{ item.name | default('') }} from {{ item.node }} [{{ item.status }}]"
  changed_when: true
```

**変更後**:

```yaml
- name: Migrate non-HA LXC containers back to target_node
  ansible.builtin.command:
    cmd: >-
      pct migrate {{ item.vmid }} {{ inventory_hostname }}
      {{ '--restart 1' if item.status == 'running' else '' }}
  loop: "{{ _vms_to_restore | selectattr('type', 'equalto', 'lxc') | list }}"
  loop_control:
    label: "LXC {{ item.vmid }} {{ item.name | default('') }} from {{ item.node }} [{{ item.status }}]"
  delegate_to: "{{ item.node }}"
  changed_when: true
```

**理由**:

- `qm migrate` / `pct migrate` は source node（VM が現在いるノード）で実行する必要がある。
- `item.node` は `pvesh get /cluster/resources` の応答フィールドで、VM の現在の配置ノード名が入っている。
- `inventory_hostname`（target node）での実行を delegate_to で source node に切り替えることで、
  "target is local node" エラーを解消する。
- `qm start` / `pct start`（migration 後の起動確認）は target node（`inventory_hostname`）上で
  実行するため変更不要。
