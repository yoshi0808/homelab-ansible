## 修正内容

### Fix 1 (High): post-restore healthcheck 非OK で playbook を失敗させる

**対象ファイル**: `roles/proxmox_restore_vm_placement/tasks/main.yml`

**変更前**:

```yaml
- name: Warn if post-restore healthcheck is not OK
  ansible.builtin.debug:
    msg: "WARNING: Post-restore healthcheck on {{ inventory_hostname }}: {{ proxmox_report.result.status }}"
  when: proxmox_report.result.status != 'OK'
```

**変更後**:

```yaml
- name: Abort if post-restore healthcheck is not OK
  ansible.builtin.fail:
    msg: >-
      Post-restore healthcheck on {{ inventory_hostname }} is
      {{ proxmox_report.result.status }}:
      {{ (proxmox_report.result.criticals + proxmox_report.result.warnings) | join(' | ') }}
  when: proxmox_report.result.status != 'OK'
```

**理由**: WARNING/CRITICAL のまま SUCCESS と記録されると、上位フロー（weekly_full 等）が誤って pve1 へ進む可能性があるため。

---

### Fix 2 (Medium): HA 復帰待ちで `status == running` も確認する

**対象ファイル**: `roles/proxmox_restore_vm_placement/tasks/main.yml`

**変更前** (until条件):

```yaml
until: >-
  (_restore_ha_poll.stdout | from_json |
   ...
   rejectattr('node', 'equalto', inventory_hostname) |
   list | length) == 0
```

target_node 以外にいる HA VM が 0 件になるまで待つだけだった。
target_node に戻ったが stopped の場合は条件を満たしてしまっていた。

**変更後** (until条件):

```yaml
until: >-
  ((_restore_ha_poll.stdout | from_json |
    ...
    rejectattr('node', 'equalto', inventory_hostname) |
    list | length) == 0) and
  ((_restore_ha_poll.stdout | from_json |
    ...
    selectattr('node', 'equalto', inventory_hostname) |
    rejectattr('status', 'equalto', 'running') |
    list | length) == 0)
```

条件を 2 本立てに変更:
1. target_node 以外にいる HA VM（prefer<target_node>+hacritical）が 0 件
2. target_node 上にいるが running でない HA VM が 0 件

両方が満たされたときに待機を終了する。

---

### Fix 3 (Low): destination_node の override コメントを削除

**対象ファイル**: `playbooks/proxmox_evacuate_node.yml`

**変更前**:
```
# destination_node: auto-derived as the opposite node (optional override)
```

**変更後**:
```
# destination_node: auto-derived as the opposite node (pve1 <-> pve2)
```

現在の実装では `hosts:` と `vars:` の両方が target_node の反対側を直接算出しており、external override が実際には効かない。コメントが誤解を招くため「optional override」の記述を削除し、自動導出であることのみを示す表記に変更した。
