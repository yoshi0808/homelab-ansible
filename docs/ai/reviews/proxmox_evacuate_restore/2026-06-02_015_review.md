## 確認内容

### Fix 1: role default を policy 準拠の false に戻している

`roles/proxmox_evacuate_node/defaults/main.yml:16`

```yaml
proxmox_evacuate_allow_local_disk_migration: false
```

role default が `false` なので、別環境や inventory override なしの実行では、local-only storage VM の migration を自動許可しない。

これは `docs/ai/prompts/proxmox_patch_policy.md` の以下の方針と整合している。

- local-only storage を使う VM / CT は自動 migration せず停止する
- local-only storage 使用 VM / CT がある場合は停止条件

### Fix 2: homelab 固有の承認設定を group_vars に分離している

`inventories/homelab/group_vars/proxmox.yml:7`

```yaml
proxmox_evacuate_allow_local_disk_migration: true
```

`pve1` / `pve2` は `inventories/homelab/hosts.yml` で `proxmox` グループ配下にあるため、この値は `proxmox_evacuate_node` role 実行時に role default を上書きする。

これにより、この homelab では `local-zfs` VM の live migration に必要な `--with-local-disks` が付与される一方、role 自体の既定値は安全側に維持される。

## 残リスク / 注意点

`proxmox_patch_policy.md` は一般方針として local-only storage VM / CT の自動 migration 停止を定めている。

今回の実装は、role default ではその方針を守りつつ、homelab inventory で明示的に例外承認する構成である。運用上は、この `group_vars/proxmox.yml` の設定が「local-zfs VM の local disk migration を許可する homelab 固有判断」であることを維持する必要がある。
