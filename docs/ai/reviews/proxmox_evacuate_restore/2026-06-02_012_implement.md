# 追加実装内容: proxmox_evacuate_node（local disk migration デフォルト true 化）

対応: 実環境（全VMが local-zfs 使用）に合わせた既定値の変更

## 実装概要

`2026-06-02_009_implement.md` で導入した opt-in 変数
`proxmox_evacuate_allow_local_disk_migration` の既定値は `false` だった。

しかし実環境ではすべての VM が `local-zfs`（local-only storage）を使っているため、
既定 `false` のままだと退避時に必ず以下で失敗する。

```text
can't live migrate attached local disks without with-local-disks option
```

そこで、この homelab の実態に合わせて既定値を `true` に変更した。

---

## 修正内容

### Fix: proxmox_evacuate_allow_local_disk_migration の既定値を true に変更

**対象ファイル**: `roles/proxmox_evacuate_node/defaults/main.yml`

**変更前**:

```yaml
# Allow live-migrating QEMU VMs that have local (non-shared) disks.
# Per proxmox_patch_policy.md (5.6), local-only storage VMs are NOT auto-migrated
# by default; the evacuation must stop so a human can decide. Set to true only
# when local disk storage migration is explicitly approved for this run; this
# appends --with-local-disks to qm migrate.
proxmox_evacuate_allow_local_disk_migration: false
```

**変更後**:

```yaml
# Allow live-migrating QEMU VMs that have local (non-shared) disks.
# When true, --with-local-disks is appended to qm migrate so VMs on local
# storage can be evacuated. This homelab uses local-zfs for all VMs, so the
# default is true; without it evacuation fails with
# "can't live migrate attached local disks without with-local-disks option".
# Set to false to enforce the proxmox_patch_policy.md (5.6) safe-side behavior
# of stopping evacuation when a VM uses local-only storage.
proxmox_evacuate_allow_local_disk_migration: true
```

**理由**:
- 実環境では全 VM が `local-zfs` を使用しており、`false` では退避が常に失敗する。
- 既定を `true` にすることで、`qm migrate` に `--with-local-disks` が付与され、
  local storage 上の VM も退避できる。
- opt-in 変数自体は維持しているため、policy 上の安全側挙動が必要な場合は
  `false` を明示的に渡すことで `5.6` の「local-only storage VM は停止」挙動に戻せる。

---

## policy との整合について

`docs/ai/prompts/proxmox_patch_policy.md` (5.6) は、local-only storage VM を
自動 migration せず停止する安全側方針を定めている。

本変更で既定が `true` になったことで、この homelab では「local disk を含めて自動移行する」
方が既定挙動になる。これは実環境の構成（全 VM が local-zfs）に対する controlled apply の
許容範囲を、policy より実態に寄せる判断である。

policy 文書側の更新が必要かどうかは、運用判断（Yoshinobu）に委ねる。
変数で挙動を切り替えられる構造は維持している。
