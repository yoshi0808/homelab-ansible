## 動作仕様

### tester_mode=true 時

- 変更タスク直前で `meta: end_play` または `meta: end_host` にて安全停止
- debug タスクで「操作内容・対象・skip_notifications=true」を表示
- common_slack notify.yml は送信せず debug 表示に切り替え
- `fail` は使用しない（plan-only = 成功終了）

### tester_mode=false（デフォルト）

- 既存動作に一切影響なし
- Semaphore / 手動実行とも `-e tester_mode=true` 未指定なら通常実行

## skip_notifications の伝播

各 play の `vars:` に以下を追加済み:

```yaml
skip_notifications: "{{ tester_mode | default(false) | bool }}"
```

common_slack/tasks/notify.yml が `skip_notifications | default(false) | bool` を参照。
