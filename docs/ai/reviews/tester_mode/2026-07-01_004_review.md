# Re-review - tester_mode safety gate

## Summary

前回 must-fix #1 の一部は修正されているが、まだ commit 不可。

特に `.claude/.settings.json` が削除状態になっており、前回指摘した「スコープ外の権限変更」が
解消されていない。また、証明書更新 playbook の cleanup play が `tester_mode=true` でも
実行され得る。

## Findings

### must-fix: `.claude/.settings.json` は削除ではなく元に戻す必要がある

対象:

- `.claude/.settings.json`

理由:

前回の指摘は「`.claude/.settings.json` の権限変更を今回の tester_mode 実装から除外する」
という意味だったが、現在はファイル全体が削除状態になっている。

このファイルには Claude Code 側の permission 設定が含まれており、削除は tester_mode 安全弁の
実装範囲外。さらに、削除によって `git commit` / `git push` / `ssh` / `rm` の確認ルールも
消えるため、安全性の観点では前回より悪化している。

修正案:

- `.claude/.settings.json` を HEAD の内容に戻す。
- tester_mode 実装の変更対象から完全に外す。

### must-fix: `cert_renew.yml` / `cert_renew_quory.yml` の cleanup play が tester_mode で実行され得る

対象:

- `playbooks/cert_renew.yml:122`
- `playbooks/cert_renew_quory.yml:75`

理由:

issue/deploy play には追加ゲートが入ったが、その後の cleanup play は `tester_mode=true` でも
`homelab_cert_renew` role の `cleanup` task を実行する。

`cleanup` は CA staging directory を `state: absent` にする変更操作であり、tester_mode の
「実変更なし」に反する。通常は prepare play が tester_mode で止まるため新規作成物はないが、
過去の失敗実行などで `/run/semaphore-ca` が残っている場合、tester 実行がそれを削除し得る。

修正案:

- cleanup play にも `tester_mode` gate を置いて `meta: end_play` する。
- もしくは cleanup role 側の先頭にも `tester_mode` gate を置く。
- send completion notification play は common_slack 側で送信抑止されるが、summary 生成に未定義
  fact が出ないかもあわせて確認する。

### suggestion: `proxmox_patch_weekly_full.yml` の tester_mode 表示は「停止」と言い切らない方がよい

対象:

- `playbooks/proxmox_patch_weekly_full.yml:176`

理由:

現在の `tester_mode` ブロックは `Set recovery mute for full patch window` play だけを
`end_play` する。後続の `import_playbook` 群は引き続き評価される。

今回、import 先の `proxmox_evacuate_node.yml` / `proxmox_patch_apply_node.yml` /
`proxmox_restore_vm_placement.yml` に追加ゲートが入ったため、危険操作への到達リスクは
かなり下がっている。ただし debug メッセージの「stop」は playbook 全体停止のように読める。

修正案:

- `tester_mode` 時でも後続の safe/dry-run phase は続く可能性があることをメッセージに出す。
- あるいは full playbook 全体で tester skip fact を使い、後続 import / post-healthcheck play も
  一貫して plan-only 終了に寄せる。
