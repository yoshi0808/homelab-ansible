# 要求仕様: ansy systemd timer setup

## 目的

quory 到着までの暫定運用として、ansy 上で Ansible playbook を定期実行する。
quory 到着後に同じ role を使って移行できるよう、Git 管理する。

## 対象

- ホスト: ansy
- グループ: dev_nodes

## 作成・更新ファイル

- roles/ansy_systemd_timers/tasks/main.yml
- roles/ansy_systemd_timers/defaults/main.yml
- roles/ansy_systemd_timers/templates/ansible-timer.service.j2
- roles/ansy_systemd_timers/templates/ansible-timer.timer.j2
- playbooks/ansy_systemd_timers.yml

## 実施内容

以下の timer を /etc/systemd/system へ配置・有効化する。

```yaml
systemd_ansible_timers:
  - name: authy-healthcheck
    description: Run authy healthcheck
    schedule: "*-*-* 05:30:00"
    playbook: playbooks/radius_healthcheck.yml
    persistent: true
  - name: proxmox-healthcheck
    description: Run Proxmox healthcheck
    schedule: "*-*-* 05:40:00"
    playbook: playbooks/proxmox_healthcheck.yml
    persistent: true
  - name: proxmox-patch-dryrun
    description: Run Proxmox patch dry-run
    schedule: "Fri *-*-* 17:30:00"
    playbook: playbooks/proxmox_patch_dryrun.yml
    persistent: true
  - name: authy-reboot-if-required
    description: Reboot authy if required
    schedule: "*-*-* 03:30:00"
    playbook: playbooks/ubuntu_nightly.yml
    persistent: false
```

## 制約

- Proxmox patch apply は timer 化しない
- timer 内で git pull しない
- 複数実行を防ぐため flock を使う
- WorkingDirectory=/home/yoshi/homelab-ansible
- User=yoshi
- 変更系（変更あり: systemd unit 配置・有効化）
- 秘密情報を扱わない

## 初回実装スコープ

- role と playbook の新規作成
- ansy での動作確認（syntax-check のみ）

## 初回除外スコープ

- quory への移行手順
- timer の削除 task

## shell / Ansible 責務分離

- shell: なし
- Ansible tasks: unit ファイル配置・systemd reload・enable・start を担当
