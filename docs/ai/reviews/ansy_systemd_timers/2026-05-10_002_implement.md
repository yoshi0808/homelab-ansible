## 変更内容の要約

### defaults/main.yml

以下の変数を定義した。

- `ansy_systemd_timers_working_dir`: `/home/yoshi/homelab-ansible`
- `ansy_systemd_timers_run_user`: `yoshi`
- `ansy_systemd_timers_playbook_bin`: `/usr/bin/ansible-playbook`
- `systemd_ansible_timers`: 4 本の timer 定義リスト

timer リストの内容:

| name | schedule | playbook | persistent |
|---|---|---|---|
| authy-healthcheck | `*-*-* 05:30:00` | playbooks/radius_healthcheck.yml | true |
| proxmox-healthcheck | `*-*-* 05:40:00` | playbooks/proxmox_healthcheck.yml | true |
| proxmox-patch-dryrun | `Fri *-*-* 17:30:00` | playbooks/proxmox_patch_dryrun.yml | true |
| authy-reboot-if-required | `*-*-* 03:30:00` | playbooks/ubuntu_nightly.yml | false |

### templates/ansible-timer.service.j2

以下の内容で systemd service unit を生成する。

- `Type=oneshot`
- `User=yoshi`（`ansy_systemd_timers_run_user` から展開）
- `WorkingDirectory=/home/yoshi/homelab-ansible`（`ansy_systemd_timers_working_dir` から展開）
- `ExecStart=/usr/bin/flock -n /run/lock/ansible-<name>.lock /usr/bin/ansible-playbook <playbook>`
  - `flock -n` により、既に実行中の場合は二重起動を防ぐ
  - ロックファイルは `/run/lock/ansible-<name>.lock`（world-writable の `/run/lock` に配置）
- `After=network-online.target` / `Wants=network-online.target`

### templates/ansible-timer.timer.j2

- `OnCalendar`: systemd calendar 形式のスケジュール
- `Persistent`: true/false（`timer.persistent` から `ternary` フィルタで展開）
- `WantedBy=timers.target`

### tasks/main.yml

1. service unit ファイルを `/etc/systemd/system/ansible-<name>.service` に配置
2. timer unit ファイルを `/etc/systemd/system/ansible-<name>.timer` に配置
3. `systemd daemon_reload: true` でデーモンをリロード
4. 各 timer を `enabled: true / state: started` で有効化・起動

loop には `loop_var: timer` を使用し、`item` との衝突を避けた。

### playbooks/ansy_systemd_timers.yml

- `hosts: dev_nodes`（ansy を対象）
- `become: true`（`/etc/systemd/system/` への書き込みと systemd 操作に必要）
- `gather_facts: false`（systemd unit 配置のみのため不要）

## 注意点

- `flock -n` でロック取得に失敗した場合（前回の実行がまだ動いている場合）、service は exit code 1 で終了する。systemd はこれを failure として記録するが、次の timer 発火は正常に行われる。
- `ansy_systemd_timers_playbook_bin` は apt インストールの `/usr/bin/ansible-playbook` を想定している。pip / venv インストールの場合はパスを変更すること。
- `authy-reboot-if-required` は `persistent: false` であるため、システム停止中に予定時刻が過ぎても再起動後に追いかけ実行しない。
