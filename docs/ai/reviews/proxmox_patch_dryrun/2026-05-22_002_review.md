## 確認したこと

### `scripts/codex-classify.sh`

差分:

```diff
-if ! codex exec "$(cat "$PROMPT_FILE")" > "$CODEX_OUTPUT" 2>&1; then
+if ! codex exec < "$PROMPT_FILE" > "$CODEX_OUTPUT" 2>&1; then
```

確認結果:

- `$(cat "$PROMPT_FILE")` による長大プロンプトのコマンドライン引数化を避けられている。
- `codex exec --help` で、PROMPT 未指定時は stdin から instructions を読む仕様であることを確認した。
- stdout / stderr の保存先は従来通り `"$CODEX_OUTPUT"` で維持されている。
- 既存の Python JSON 抽出・検証・保存処理には今回の差分による直接変更はない。
- kernel パッケージ検出による `status_inputs.reboot_expected` 強制補正は、現行ファイルの Python ブロック内、output JSON 保存直前に存在している。

### `docs/ai/prompts/ubuntu_vm_patch_policy.md`

このファイルは今回の Proxmox dry-run 実装メモの対象外。

内容面では、Ubuntu VM から Ubuntu ノード全般へ対象を広げ、`quory` の自動 reboot 時刻を 03:00 とする運用方針が追加されている。これは Proxmox QDevice / Ansible 実行基盤の運用に関係するため、別途レビュー前提を置いて確認するのがよい。
