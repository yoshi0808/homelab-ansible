## 対応内容

### 指摘 1: 片系 healthcheck NG 時にもう片系で dry-run が進む問題を修正

**変更ファイル:**

- `playbooks/proxmox_patch_dryrun.yml`

**内容:**

`any_errors_fatal: true` を追加した。

pve1 / pve2 のいずれかで healthcheck が CRITICAL / WARNING により fail した場合、全ホストの後続処理を即時停止する。
これにより、片系 NG 状態で dry-run collect が進む問題を解消した。

**変更ファイル:**

- `docs/ai/prompts/proxmox_patch_policy.md`

**内容:**

section 6.3 の対象を以下に改訂した。

変更前:
```
- pve1
- pve2
- 原則 pve2 を主対象にする
- `--limit` による単一ノード実行も可
```

変更後:
```
- pve1
- pve2
- pve1 / pve2 の固定ペアで実行する（単一ノード実行は非対応）
```
