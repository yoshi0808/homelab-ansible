## 対応内容

### 指摘 1 B: healthcheck を dry-run playbook 冒頭で呼び出し、OK でなければ停止

**変更ファイル:**

- `playbooks/proxmox_patch_dryrun.yml`
- `roles/proxmox_patch_dryrun/tasks/main.yml`

**内容:**

`playbooks/proxmox_patch_dryrun.yml` の roles リストに `proxmox_healthcheck` を `proxmox_patch_dryrun` の前に追加した。

`roles/proxmox_patch_dryrun/tasks/main.yml` の冒頭に healthcheck ゲートを追加した。
`proxmox_healthcheck` role が設定する `proxmox_report.result.status` を参照し、`OK` でなければ fail する。

- CRITICAL の場合: `proxmox_healthcheck` role 自体の fail タスクで停止
- WARNING の場合: `proxmox_patch_dryrun` tasks 冒頭のゲートで停止

これにより、pve1 / pve2 のいずれかで healthcheck が WARNING / CRITICAL の場合、dry-run は実行されない。

### 指摘 3 A: policy の出力定義を unified JSON / MD report 方針に合わせて修正

**変更ファイル:**

- `docs/ai/prompts/proxmox_patch_policy.md`

**内容:**

変更前:
```
- dry-run JSON report
- changelog text files

- mail body draft

```

```

- Codex classification JSON
- final report JSON

- final Status / Urgency
```
