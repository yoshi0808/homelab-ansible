# 要求仕様: reboot_expected Python側強制補正の追加

## 目的
Codex 出力に依存せず、kernel パッケージが含まれる場合は
Python 側で reboot_expected を確実に true に補正する。

## 対象
- scripts/codex-classify.sh

## 作成・更新ファイル
- scripts/codex-classify.sh

## 実施内容
Python 検証部分（PYEOF ブロック内）で、
Codex の出力 JSON を保存する直前に以下の補正を行う。

- input_data の updates リストを走査する
- パッケージ名（name フィールド）に "kernel" が含まれるものを抽出する
- 該当パッケージがある場合:
  - status_inputs.reboot_expected を true に強制上書きする
  - status_inputs.reboot_expected_reason が空または未設定の場合のみ
    "kernel packages included: <パッケージ名リスト>" を設定する
  - Codex が既に reboot_expected_reason を設定している場合はそのまま残す

## 参照ファイル
- scripts/codex-classify.sh（既存の修正）

## 制約
- scripts/codex-classify.sh の Python ブロック（PYEOF）内のみ修正する
- 既存の検証ロジックは壊さない
- 補正は output_json への書き込み直前に行う
- 秘密情報を扱わない

## 初回実装スコープ
- Python 側での kernel パッケージ検出と reboot_expected 強制補正

## 初回除外スコープ
- Ansible tasks 側での reboot_expected の利用
- apply 後の実際の reboot 判定改善（PENDING 1b）

## shell / Ansible 責務分離
- shell: codex-classify.sh の Python ブロックのみ修正
- Ansible tasks: 対象外
