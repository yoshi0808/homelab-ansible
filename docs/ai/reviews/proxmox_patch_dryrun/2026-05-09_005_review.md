## 結論

このまま commit はまだ非推奨。

`apt-get check` の追加と `NO_UPDATES` 通知方針の policy 反映は確認できた。一方で、`PATCH_READY` 条件に残っている healthcheck 要件が実装では確認されていないため、policy 準拠としてはまだ不一致が残る。

## 指摘事項

### 1. `PATCH_READY` 判定に healthcheck OK が入っていない

`docs/ai/prompts/proxmox_patch_policy.md:817` では、`PATCH_READY` 条件に `healthcheck が OK` が含まれている。

一方、`roles/proxmox_patch_dryrun/tasks/main.yml:159-169` の最終判定では、healthcheck 結果を参照していない。

現状では以下が満たされると `PATCH_READY` になり得る。

- `apt-get check` 成功
- `apt-get -s dist-upgrade` 成功
- remove なし
- 重要コンポーネント更新なし
- major upgrade 疑いなし

しかし、対象ノードの healthcheck が OK であることは確認されていない。

判断:

- policy の `PATCH_READY` を dry-run 結果として確定させるなら、healthcheck 結果を入力に含める必要がある。
- dry-run は「更新セットの分類のみ」で、apply 側で healthcheck を必ず再確認する方針なら、policy の `PATCH_READY` 条件から dry-run 側の healthcheck 要件を分離して明記した方がよい。

### 2. policy の安全度表示に `apt update あり` が残っている

`docs/ai/prompts/proxmox_patch_policy.md:425-428` では、`proxmox_patch_dryrun.yml` の安全度説明に以下が残っている。

```text
semi-safe
apt update あり
実パッチ適用なし
```

一方、同じ節の処理概要では `apt-get update` は変更操作にあたるため実行しない、と修正されている。

判断:

- 文書内の自己矛盾なので、commit 前に修正推奨。

### 3. changelog text files の出力定義はまだ実装とずれている

`docs/ai/prompts/proxmox_patch_policy.md:464-469` の出力には `changelog text files` が残っている。

実装では changelog diff を unified JSON に含め、Codex 生成の MD report に含める方針であり、パッケージ別の changelog text file は生成していない。

判断:

- unified JSON / MD report に含める方針でよいなら、policy の出力定義を合わせる。
- 個別 text file を正式成果物にするなら、実装追加が必要。

## 解消確認

### `apt-get check` 追加

以下を確認した。

- `roles/proxmox_patch_dryrun/files/proxmox-dryrun-collect.sh:11-12`
  - `apt-get check` を実行し、成否を `apt_check_ok` として保持している。
- `roles/proxmox_patch_dryrun/files/proxmox-dryrun-collect.sh:115-120`
  - JSON に `apt_check_ok` / `apt_check_output` を含めている。
- `roles/proxmox_patch_dryrun/files/proxmox-dryrun-merge.py:73-75`
  - `node_summaries` に `apt_check_ok` を含めている。
- `roles/proxmox_patch_dryrun/tasks/main.yml:80-87`
  - pve1 / pve2 のどちらかで `apt_check_ok` が false の場合、`BLOCKED` にしている。

この点はレビュー指摘に対応済み。

### `NO_UPDATES` 通知方針

以下を確認した。

- `docs/ai/prompts/proxmox_patch_policy.md:805-809`
  - `NO_UPDATES` でもメール通知する方針に修正済み。
- `docs/ai/prompts/proxmox_patch_policy.md:1443-1445`
  - メール通知テーブルも `NO_UPDATES` を送る方針に修正済み。

この点はレビュー指摘に対応済み。

## commit 可否

現時点では、そのまま commit は非推奨。

理由:

- policy の `PATCH_READY` 条件に healthcheck OK が残っているが、dry-run 実装では確認していない。
- policy の安全度表示に `apt update あり` が残っている。
- policy の出力定義に `changelog text files` が残っているが、実装は個別 text file を生成していない。

## 推奨対応

1. healthcheck を dry-run の `PATCH_READY` 判定に含めるか、policy 側で dry-run と apply の責務境界を明記する。
2. `proxmox_patch_dryrun.yml` の安全度説明から `apt update あり` を削除または修正する。
3. changelog の保存形式について、policy を unified JSON / MD report 方針に合わせるか、個別 text file 出力を実装する。
