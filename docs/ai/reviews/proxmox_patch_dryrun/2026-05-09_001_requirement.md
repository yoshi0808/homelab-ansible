# proxmox_patch_dryrun 要求仕様

作成日: 2026-05-09

---

## 目的

pve1/pve2 の更新候補を収集・分類し、クラスター全体の Status/Urgency を判定する。
結果をメールで通知する。

---

## 対象

- グループ: proxmox（pve1/pve2 同時実行）

---

## 処理フロー

### Phase 1: 並列収集（shell on pve1/pve2）

- `apt-get -s dist-upgrade` で更新候補取得
- インストール済みバージョンと差分 changelog 取得
- 新規パッケージは最新エントリ 1 件のみ
- `/var/run/reboot-required` の有無
- 各ノードの JSON 出力

### Phase 2: マージ（Ansible tasks on ansy）

- pve1/pve2 の更新候補を比較
- common / pve2_only / pve1_only に分類
- unified_dryrun.json を生成

### Phase 3: Codex CLI 呼び出し

- `scripts/codex-classify.sh` を呼び出す
- 入力: unified_dryrun.json
- 出力: 分類 JSON

### Phase 4: 最終判定（Ansible tasks）

- Codex 出力の urgency_candidate と cve_types_detected を patch policy テーブルと照合
- クラスター全体の Status/Urgency 確定
- どちらかのノードで URGENT なら全体 URGENT

### Phase 5: レポート保存

- `reports/proxmox-dryrun/` に保存
- JSON（機械処理用）
- MD（日本語・人間が読む用）

### Phase 6: メール送信

- 宛先: `inventories/vars/mail.yml`
- 件名: Codex 生成の mail_subject
- 本文: Codex 生成の mail_body（~1000 文字）
- 添付: 日本語 MD レポート

---

## 判定方針

- pve1/pve2 どちらかで URGENT → 全体 URGENT
- pve1/pve2 どちらかで MAINTENANCE_REQUIRED → 全体 MAINTENANCE_REQUIRED
- 両方 PATCH_READY → 全体 PATCH_READY
- NO_UPDATES → メール送信するが件名に明示

---

## レポート保存先

`reports/proxmox-dryrun/`

---

## 制約

- 変更操作なし（`apt-get -s` のみ）
- Codex CLI は ansy 上のみ実行
- メール認証情報は `inventories/vars/mail.yml`（git 管理しない）

---

## 初回除外

- systemd timer による自動実行
- URGENT 手動 override

---

## 前提

- `core.md` のルールに従う
- `proxmox_patch_policy.md` の方針に従う
- `scripts/codex-classify.sh` は実装済み
- shell は収集と JSON 整形のみ、判断は Ansible 側
