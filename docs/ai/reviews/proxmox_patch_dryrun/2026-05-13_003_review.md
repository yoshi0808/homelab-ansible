## 確認内容

### 1. 要求仕様との整合

要求仕様では、`scripts/codex-classify.sh` のみを対象に、以下を追加することになっていました。

- `updates` にパッケージ名として `kernel` を含むものがある場合は `reboot_expected: true`
- changelog から reboot が必要と判断される場合は `reboot_expected: true`
- それ以外は `false`
- `reboot_expected` が true の場合、メール本文に警告文を含める
- `reboot_expected` / `reboot_expected_reason` は Python 検証上は任意項目

差分では、プロンプト指示、スキーマ例、Python 検証の各箇所に上記が追加されており、要求範囲と一致しています。

### 2. スコープ

変更対象は `scripts/codex-classify.sh` のみです。

要求仕様で初回除外スコープとされていた以下には踏み込んでいません。

- Ansible tasks 側での `reboot_expected` 利用
- apply 後の実 reboot 判定改善

これは要求仕様どおりです。

## 残リスク・注意点

`reboot_expected` は Codex 出力に依存しており、スクリプト側で `updates[].name` を機械的に走査して補正する実装ではありません。

今回の要求は「Codex へのプロンプトに判定指示を追加」「出力スキーマ追加」「Python 検証対応」なので問題ではありませんが、将来的に reboot 必要性をより確実に扱うなら、Codex 出力後に Python 側で `kernel` を含む更新を検出して `reboot_expected` を強制補正する設計も検討できます。
