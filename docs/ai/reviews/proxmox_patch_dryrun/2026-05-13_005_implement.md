## 変更内容の要約

### 追加箇所

`output_json` への書き込み直前（行286-291）に以下を追加した。

```python
# kernel パッケージ検出による reboot_expected 強制補正
kernel_pkgs = [u.get('name', '') for u in updates if 'kernel' in u.get('name', '')]
if kernel_pkgs:
    status_inputs['reboot_expected'] = True
    if not status_inputs.get('reboot_expected_reason'):
        status_inputs['reboot_expected_reason'] = 'kernel packages included: ' + ', '.join(kernel_pkgs)
```

### 動作

- `input_data['updates']` の各要素の `name` フィールドを走査する
- `name` に `"kernel"` が含まれるものを抽出する
- 該当パッケージがある場合:
  - `status_inputs['reboot_expected']` を `True` に強制上書きする
  - `reboot_expected_reason` が空または未設定の場合のみ `"kernel packages included: ..."` を設定する
  - Codex が既に `reboot_expected_reason` を設定している場合はそのまま残す

### 設計上の注意点

- `updates` 変数は既に検証済み（必須チェック・型チェック済み）のリストを再利用している
- `status_inputs` は既に `data['status_inputs']` への参照であるため、補正後の値が `json.dump` の対象に反映される
- `u.get('name', '')` とすることで、`name` キーが欠損しているパッケージでも安全に処理できる
