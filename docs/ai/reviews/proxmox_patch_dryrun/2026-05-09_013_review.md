## 解消確認

### 1. `apt-get update` 出力の JSON stdout 汚染

`roles/proxmox_patch_dryrun/files/proxmox-dryrun-collect.sh:11-12` で、`apt-get update` の出力が変数に閉じ込められている。

```bash
apt_update_output=$(apt-get update -qq 2>&1) && apt_update_ok="true" || apt_update_ok="false"
```

これにより、shell の stdout は Python が最後に出力する JSON のみに保たれる。

### 2. `apt-get update` 失敗時の `BLOCKED` 判定

以下を確認した。

- `roles/proxmox_patch_dryrun/files/proxmox-dryrun-collect.sh:27-28`
  - `APT_UPDATE_OK` / `APT_UPDATE_OUTPUT` を Python 側へ渡している。
- `roles/proxmox_patch_dryrun/files/proxmox-dryrun-collect.sh:122-126`
  - JSON に `apt_update_ok` / `apt_update_output` を含めている。
- `roles/proxmox_patch_dryrun/files/proxmox-dryrun-merge.py:73-75`
  - `node_summaries` に `apt_update_ok` を含めている。
- `roles/proxmox_patch_dryrun/tasks/main.yml:89-98`
  - pve1 / pve2 のいずれかで `apt_update_ok` が false の場合、`BLOCKED` にしている。

これにより、古い package list のまま `PATCH_READY` 判定へ進む問題は解消されている。

### 3. BLOCKED 時の表示

`roles/proxmox_patch_dryrun/tasks/main.yml` の BLOCKED 用 MD report / mail 本文で、ノード状態に以下が表示されるようになっている。

- `apt_update_ok`
- `apt_check_ok`
- `sim_ok`

失敗箇所の切り分けに必要な最低限の情報は出る。
