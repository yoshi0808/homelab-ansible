# 実装レビュー: codex-classify.sh コマンドライン引数長制限の修正

- **日付**: 2026-05-22
- **対象ファイル**: `scripts/codex-classify.sh`

## 問題

`codex exec "$(cat "$PROMPT_FILE")"` がコマンドライン引数の長さ制限（ARG_MAX）を超えてエラーになる。
changelog が多い場合（36件以上）に発生。

## 修正内容

**変更前:**
```bash
if ! codex exec "$(cat "$PROMPT_FILE")" > "$CODEX_OUTPUT" 2>&1; then
```

**変更後:**
```bash
if ! codex exec < "$PROMPT_FILE" > "$CODEX_OUTPUT" 2>&1; then
```

## 理由

`$(cat FILE)` はファイル内容をシェル引数として展開するため、OS の ARG_MAX 制限に抵触する。
stdin リダイレクト (`< FILE`) に変更することで、ファイルサイズに依存しない安全な入力方式となる。
