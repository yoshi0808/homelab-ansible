## 対応内容

### 指摘 1: `apt-get update` の出力が JSON stdout を汚染する問題を修正

**変更ファイル:**

- `roles/proxmox_patch_dryrun/files/proxmox-dryrun-collect.sh`

**内容:**

変更前:
```bash
apt-get update -qq 2>&1
```

変更後:
```bash
apt_update_output=$(apt-get update -qq 2>&1) && apt_update_ok="true" || apt_update_ok="false"
```

出力を変数に閉じ込め、stdout には JSON のみが出力される状態にした。

### 指摘 2: `apt-get update` 失敗を Status 判定（BLOCKED）に含めた

**変更ファイル:**

- `roles/proxmox_patch_dryrun/files/proxmox-dryrun-collect.sh`
- `roles/proxmox_patch_dryrun/files/proxmox-dryrun-merge.py`
- `roles/proxmox_patch_dryrun/tasks/main.yml`

**collect.sh の変更内容:**

- `apt_update_ok` / `apt_update_output` を env var 経由で Python 側に渡す
- JSON 出力に `apt_update_ok` / `apt_update_output` を追加

**merge.py の変更内容:**

- `node_summaries` に `apt_update_ok` を追加

**tasks/main.yml の変更内容:**

- `_pre_status` の BLOCKED 判定に `apt_update_ok` チェックを追加（pve1/pve2 いずれかが false → BLOCKED）
- BLOCKED メール本文・MD レポートのノード状態表示に `apt_update_ok` / `apt_check_ok` を追加
