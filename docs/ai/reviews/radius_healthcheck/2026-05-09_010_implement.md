### 1. vars 読み込み task を先頭に追加


### 2. Send healthcheck notification task を追加

挿入位置: `Print health summary` の直後、`Fail if critical` の直前。

```yaml
- name: Send healthcheck notification
  community.general.mail:
    host: "{{ smtp_host }}"
    port: "{{ smtp_port }}"
    username: "{{ smtp_user }}"
    password: "{{ smtp_password }}"
    to: "{{ mail_to }}"
    subject: "[RADIUS Healthcheck] {{ radius_report.result.status }} - {{ inventory_hostname }}"
    body: |
      Host    : {{ inventory_hostname }}
      Status  : {{ radius_report.result.status }}

      Criticals:
      {{ radius_criticals | join('\n') if radius_criticals | length > 0 else '(none)' }}

      Warnings:
      {{ radius_warnings | join('\n') if radius_warnings | length > 0 else '(none)' }}
    secure: starttls
  delegate_to: localhost
  become: false
  when: radius_criticals | length > 0 or radius_warnings | length > 0
```

条件: `radius_criticals | length > 0 or radius_warnings | length > 0`（OK 時は送信しない）

添付なし、STARTTLS 送信。proxmox_patch_apply_node の Send summary email パターンと同一構造。
