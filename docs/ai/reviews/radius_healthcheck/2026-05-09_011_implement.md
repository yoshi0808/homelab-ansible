## 変更内容

### 指摘 2: Send healthcheck notification に ignore_errors: true を追加

**変更ファイル:** `roles/radius_healthcheck/tasks/main.yml`

変更前:
```yaml
    secure: starttls
  delegate_to: localhost
  become: false
  when: radius_criticals | length > 0 or radius_warnings | length > 0
```

変更後:
```yaml
    secure: starttls
  delegate_to: localhost
  become: false
  ignore_errors: true
  when: radius_criticals | length > 0 or radius_warnings | length > 0
```

**効果:**

SMTP 認証失敗・ネットワーク不通・STARTTLS エラーが発生しても play が止まらなくなった。

- CRITICAL 時: `Fail if critical` に必ず到達する
- WARNING 時: メール障害が Ansible 実行結果を FAILED にしなくなる

proxmox_patch_apply_node の Send summary email パターンとの整合が取れた。
