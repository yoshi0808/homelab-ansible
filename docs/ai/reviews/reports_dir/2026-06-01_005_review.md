# Review: reports ディレクトリの絶対パス化 追加実装

## 結論

追加実装は妥当。前回レビューで指摘した `playbooks/ubuntu_nightly.yml` の `playbook_dir` 依存は、2 箇所とも `reports_base_dir` 参照へ修正されている。

現時点の `playbooks/` と `roles/` には、reports 保存先としての `{{ playbook_dir }}/../reports/...` 参照は残っていない。今回の目的である「Semaphore 実行時に reports が `/opt/semaphore/...` 配下へ散らばることを防ぐ」は満たしている。

## Findings

なし。

## 確認結果

### 要求仕様との整合

- `inventories/homelab/group_vars/all.yml` に `reports_base_dir` が定義されている。

```yaml
reports_base_dir: /home/yoshi/homelab-ansible/reports
```

- 対象 role の `defaults/main.yml` は `reports_base_dir` 参照へ変更済み。
- `ubuntu_vm_patch_policy.md` で運用対象となっている `playbooks/ubuntu_nightly.yml` も、`nightly_report_dir` が `reports_base_dir` 参照へ変更済み。

### 差分確認

主な変更:

- `playbooks/ubuntu_nightly.yml`
  - `nightly_report_dir: "{{ reports_base_dir }}/ubuntu-nightly"` に変更
- `roles/*/defaults/main.yml`
  - 各 report dir を `{{ reports_base_dir }}/...` に変更
- `inventories/homelab/group_vars/all.yml`
  - `reports_base_dir` を新規定義

`git diff --stat` では untracked の `inventories/homelab/group_vars/all.yml` は表示されないため、commit 時に必ず含める必要がある。

### 残存検索

実行:

```bash
rg -n "playbook_dir.*reports|reports.*playbook_dir|reports_base_dir" playbooks roles inventories/homelab/group_vars
```

結果:

- `playbook_dir` と reports を組み合わせた実コード参照はなし。
- `reports_base_dir` 参照は以下に存在。
  - `playbooks/ubuntu_nightly.yml`
  - `roles/proxmox_healthcheck/defaults/main.yml`
  - `roles/proxmox_patch_dryrun/defaults/main.yml`
  - `roles/proxmox_patch_apply_node/defaults/main.yml`
  - `roles/proxmox_hw_check/defaults/main.yml`
  - `roles/radius_healthcheck/defaults/main.yml`
  - `roles/monitoring_healthcheck/defaults/main.yml`
  - `inventories/homelab/group_vars/all.yml`

## 補足

`reports_base_dir` を Git 管理の `group_vars/all.yml` に置く方針は、今回の要件と整合している。Semaphore 側で別パスにしたい場合も Extra Variables で上書きできるため、今回の実装で運用上の逃げ道も残っている。
