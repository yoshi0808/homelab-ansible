# Review: reports ディレクトリの絶対パス化

## 結論

修正対象 role の `defaults/main.yml` については、要求仕様どおり `{{ playbook_dir }}/../reports/...` から `{{ reports_base_dir }}/...` へ置き換えられている。

ただし、同じ reports 出力を持つ `playbooks/ubuntu_nightly.yml` に `{{ playbook_dir }}/../reports/ubuntu-nightly` が残っている。Semaphore 実行時に reports が `/opt/semaphore/...` 配下へ散らばる問題を防ぐという目的から見ると、ここも `reports_base_dir` を使うべき。

## Findings

### [Medium] `ubuntu_nightly.yml` の report 保存先がまだ `playbook_dir` 依存

- 対象:
  - `playbooks/ubuntu_nightly.yml:8`
  - `playbooks/ubuntu_nightly.yml:200`

現在、以下の定義が残っている。

```yaml
nightly_report_dir: "{{ playbook_dir }}/../reports/ubuntu-nightly"
```

`ubuntu_nightly.yml` は `ubuntu_vm_patch_policy.md` で systemd timer / 将来の Semaphore Schedule 対象として定義されており、reboot_required の確認結果や reboot 後の確認結果を report として保存する playbook である。

今回の要求仕様では初回除外スコープに「playbook本体の修正（role経由で参照するため不要のはず）」とあるが、`ubuntu_nightly.yml` は role 経由ではなく playbook 内で `nightly_report_dir` を直接定義している。そのため、Semaphore から実行した場合に `ubuntu-nightly` report だけは従来どおり実行環境依存の場所に保存される。

推奨修正:

```yaml
nightly_report_dir: "{{ reports_base_dir }}/ubuntu-nightly"
```

## 確認結果

### 要求仕様との整合

- `inventories/homelab/group_vars/all.yml` は新規作成され、以下が定義されている。

```yaml
reports_base_dir: /home/yoshi/homelab-ansible/reports
```

- 以下の role defaults は要求仕様どおり修正されている。
  - `roles/proxmox_healthcheck/defaults/main.yml`
  - `roles/proxmox_patch_dryrun/defaults/main.yml`
  - `roles/proxmox_patch_apply_node/defaults/main.yml`
  - `roles/proxmox_hw_check/defaults/main.yml`
  - `roles/radius_healthcheck/defaults/main.yml`
  - `roles/monitoring_healthcheck/defaults/main.yml`

### 残存検索

実行:

```bash
rg -n "playbook_dir.*/\\.\\./reports|reports_base_dir|/reports/" .
```

結果として、実装対象 role の defaults には `playbook_dir/../reports` は残っていない。一方で、実コードでは `playbooks/ubuntu_nightly.yml` に 2 箇所残っている。

### syntax-check

最初の実行では sandbox 上の `/home/yoshi/.ansible/tmp` が read-only のため失敗した。`ANSIBLE_LOCAL_TEMP` / `ANSIBLE_REMOTE_TEMP` を `/tmp` に指定して再実行し、以下は成功した。

```bash
ANSIBLE_LOCAL_TEMP=/tmp/ansible-local ANSIBLE_REMOTE_TEMP=/tmp/ansible-remote ansible-playbook -i inventories/homelab/hosts.yml playbooks/proxmox_healthcheck.yml --syntax-check
ANSIBLE_LOCAL_TEMP=/tmp/ansible-local ANSIBLE_REMOTE_TEMP=/tmp/ansible-remote ansible-playbook -i inventories/homelab/hosts.yml playbooks/proxmox_patch_dryrun.yml --syntax-check
ANSIBLE_LOCAL_TEMP=/tmp/ansible-local ANSIBLE_REMOTE_TEMP=/tmp/ansible-remote ansible-playbook -i inventories/homelab/hosts.yml playbooks/ubuntu_nightly.yml --syntax-check
```

## 補足

`git diff --stat` には untracked の `inventories/homelab/group_vars/all.yml` が表示されない。commit 時には、この新規ファイルを必ず含める必要がある。含めない場合、修正済み role defaults が参照する `reports_base_dir` が未定義になり、実行時に失敗する。
