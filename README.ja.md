# homelab-ansible

[English](README.md) | 日本語

このリポジトリは、個人ホームラボ環境を対象とした Ansible 自動化プロジェクトです。

AI を活用しながら、セキュリティ、運用、バックアップを継続可能な形で実現することを目指しています。

ただし、本プロジェクトの目的は、単なる Ansible Playbook 集を作ることではありません。

近年、サイバー攻撃は急速に高度化しており、ホームユーザーであっても、セキュリティ対策、証明書管理、バックアップ運用、継続的なパッチ適用が求められる時代になりました。

一方で、これらを個人で設計・実装・継続運用することは容易ではありません。

本プロジェクトでは、AI を活用することで、個人でも実現可能な運用基盤を構築することを目指しています。

## AI 活用方針

本プロジェクトでは、複数の AI を利用した実装・レビュー体制を採用しています。

- AI が実装を提案する
- 別の AI がレビューを行う
- 異なる AI 同士が相互に検証する
- 最終判断は人間が行う

AI は意思決定者ではなく、あくまで支援者です。

コード品質、セキュリティ、運用方針については、人間が責任を持って判断します。

また、本リポジトリでは、一般的な Issue / Pull Request ベースの運用ではなく、要求仕様（requirement）、実装記録（implement）、レビュー結果（review）をドキュメントとして保存しています。

これにより、AI との対話、設計判断、レビュー指摘、修正の経緯を追跡可能な形で管理しています。

Git に不慣れなユーザーであっても、要求仕様、実装、レビューの履歴を追うことで、AI を活用しながらどのように設計品質やコード品質を高めていくのか、そのプロセスを確認できる構成にしています。

## AI による運用支援

本プロジェクトでは、パッチ管理にも AI を活用しています。

個人環境では、パッチ適用は属人的な管理になりがちです。

そのため、本プロジェクトでは、更新候補の分類や適用可否の判断基準をあらかじめ人間が定義し、その基準に沿って AI が変更内容やセキュリティ影響を整理します。

AI が勝手に更新を判断することはありません。

人間が決めた運用ルールに従い、AI は以下の情報を整理します。

- 緊急度
- リスク
- 影響範囲
- 適用判断材料

これにより、運用者が判断しやすい形で情報を提示することを目的としています。

## このリポジトリの構成

```text
homelab-ansible/
├── inventories/         # Inventory 定義・変数
├── playbooks/           # Playbook
├── roles/               # 共通ロール
├── reports/             # レポート出力
├── scripts/             # 補助スクリプト
├── docs/
│   └── ai/
│       ├── prompts/     # AI向け設計・実装ポリシー
│       └── reviews/     # 要求仕様・実装・レビュー履歴
├── README.md            # English
├── README.ja.md         # 日本語
└── LICENSE
```

`docs/ai/reviews` には、要求仕様（requirement）、実装記録（implement）、
レビュー結果（review）が保存されています。

本プロジェクトでは、一般的な Issue / Pull Request ベースの管理だけではなく、
AIとの対話や設計判断の履歴もドキュメントとして残しています。

## このリポジトリで扱うテーマ

本プロジェクトでは、今後ホームユーザーにも求められると考えられる技術を積極的に採用しています。

- WPA3 Enterprise
- FreeRADIUS
- EAP-TLS
- SSL/TLS 証明書管理
- パスワードレス認証
- VPN 証明書認証
- 自動パッチ適用
- バックアップ検証
- 継続的なヘルスチェック
- インフラ自動化

このリポジトリ自体は Ansible に関するものですが、ブログ記事と併せて、多くのホームユーザーに課題認識を持ってもらい、セキュリティ意識および IT スキルの向上につなげることを目的としています。

## Why this repository exists

本プロジェクトは、AI を利用してコードを生成すること自体を目的としていません。

ホームユーザーが直面するセキュリティ課題や運用課題に対し、AI を活用することで継続可能な運用を実現することを目的としています。

AI は運用者の代わりではありません。

AI を活用しながら、人間が判断し、人間が責任を持つ。

その実践例として、このリポジトリを公開しています。
